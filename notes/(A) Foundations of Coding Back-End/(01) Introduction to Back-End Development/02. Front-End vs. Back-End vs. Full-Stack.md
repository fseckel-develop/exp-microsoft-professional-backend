## Front-End

- Runs directly on the user’s device (e.g. browser, mobile app or desktop) for interaction
- Includes all user-facing elements: text, images, buttons, forms, layout, colours, fonts, ...
- Example: An online store’s interface is built with front-end code

#### Limitations of Front-End

- User devices may not be powerful enough for complex or resource-intensive tasks 
  (e.g. real-time video streaming or processing secure payments)
- Front-end code is accessible to users, visible and can be modified, creating security risks
- Cannot directly interact with databases or server resources due to sandboxing
  (running in a secure, limited environment, like a web browser)

---
## Servers

- Powerful computers that store and secure data as well as run complex application logic
  (therefore picking up where the front-end leaves off with its limitations)
- Handle requests from users, process data, interact with databases, and send response
- Example: Typing a URL sends a request to the server, which returns the page content

---
## Back-End

- Part of the application that runs on the server and manages core application operations
- Handles data processing, business logic, authentication, and communication with databases
- Ensures smooth, secure, and efficient functioning of the application

---
## Full-Stack Development

- Involves building both the front-end and the back-end (of a web application)
- Requires knowledge of:
    - Designing interactive user interfaces
    - Writing server-side code to process and manage data
    - Ensuring seamless communication between client and server
- Example: Processing a form:
    - Front-end captures the input correctly
    - Back-end validates, stores, and processes it

---
## Key Interactions Between Front-End & Back-End

#### Middleware

- Sits between front-end and back-end, managing communication in between
- Manages, validates, and transforms data as it moves through the system
- Example: Checking login credentials before a purchase

#### Application Programming Interfaces (APIs)

- Bridges that allow different software systems to communicate with each other
- Define rules and protocols for accessing data or services between different software systems
- Example: Processing credit card payments by communicating with external payment services

#### Static Content Serving

- Delivers unchanging assets (images, CSS, product descriptions) quickly from the server
- Improves load times and ensures users can view content instantly
- Example: Product images loading immediately on an online store

---
## Summary

- Front-end handles everything users see and interact with
- Back-end manages data, logic, and secure operations behind the scenes
- Full-stack developers bridge both areas to build complete, efficient applications
