## What is Back-End Development?

- “Kitchen” of an application: runs behind the scenes, powers everything the user experiences
  (while the front-end is only what the user sees and interacts with, like the restaurants front)
- Involves building and managing the systems that power apps and websites:
    - Data processing
    - Storage
    - Application logic
- Focuses on servers, databases, and APIs that make apps functional and interactive

---
## Core Components of Back-End Development

#### Servers

- Powerful computers that store data and run application software
- Handle requests, process logic, manage authentication, and communicate with databases
- All of this involves **Server-Side Scripting**:
    - Code executed on the server (not the browser)
    - Generates dynamic pages, processes user input, and manages app logic
    - Example: Login credentials being validated on a social media site

#### Databases

- Digital storage systems for all application data (users, products, transactions, etc.)
- Ensure data is organised, retrievable, and efficiently managed
- All of this involves **Database Management**:
    - Ensures that data is quickly accessed, updated, and retrieved whenever needed
    - Example: Searching for product triggers database lookup; purchasing updates inventory
- Without databases: application would struggle to handle large amount of data, 
  leading to slower performance, less reliable data access, and potential data loss

#### Application Programming Interfaces (APIs)

- Bridges that allow different software systems to communicate
- Define rules and protocols for accessing data or services between different software systems
- Example: Processing credit card payments by communicating with external payment services
- All of this involves **API Integration**:
    - Connecting two or multiple systems using APIs to exchange data or perform actions
    - Expands capabilities of applications without building everything from scratch

---
## Core Principles of Back-End Development

#### Performance Optimisation

- Speed and efficiency of how the site or application responds to user actions
- Ensures a smooth user experience, essential for retaining users and maintaining satisfaction
- Achieved through:
    - Efficient code
    - Minimising resource usage and maximising response speed
    - Regular monitoring and management of server resources

#### Scalability

- Ability of a system to grow and handle more users, data, and operations
- Example: An e-commerce platform scaling during high-traffic sales events
- Achieved through:
    - Expanding server resources
    - Designing systems that grow cleanly with demand

#### Data Integrity and Security

- Protects sensitive data, maintains user trust, and meets legal requirements
- Achieved through:
    - Secure data handling (encryption in transit & at rest)
    - Access control for sensitive data and functionalities
- Example: Protecting credit card and customer details

#### Error Handling and Reliability

- Ability of a system to function correctly and consistently over time
- Example: An online store processing orders without downtime even during peak demand
- Achieved through:
    - Graceful error handling to prevent system crashes
    - Comprehensive logging to capture errors and operational data for system diagnosis
    - Building resilient system behaviour

---
## Summary

- Back-end development centers around servers, databases, and APIs
- Strong back-end systems rely on four principles:
    - **Performance**
    - **Scalability**
    - **Security**
    - **Reliability**
- Mastering these ensures efficient, stable, and secure applications
