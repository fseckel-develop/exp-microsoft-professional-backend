## Overview

- Back-end engineers require a broad set of technical skills to build, maintain, and optimise server-side systems, especially managing databases and optimising server performance

- Core skill areas include:
    - Server-side programming languages
    - Frameworks
    - Libraries
    - Database management systems
    - Tools & Technologies (Version control, containerisation, CI/CD)

---
## Server-Side Programming Languages

- Foundation of back-end development; used to create application logic and handle data flow
- Enable interaction with computers through structured instructions

- Used for:
    - Handling logic
    - Managing database operations
    - Configuring server behaviour

- Common languages:
    - **C#**
    - **JavaScript**
    - **Python**

- Essential for processing user inputs and keeping applications running smoothly

---
## Frameworks

- Pre-written structures that streamline and organise application development
- Provide standardised patterns, reusable components, and built-in tools
- Improve development speed, consistency, and maintainability

- Popular examples:
    - **ASP.NET** (C#)
    - **Django** (Python)

- Act like a “pre-built Lego set” that guides how pieces fit together to create a particular model

---
## Libraries

- Collections of pre-written code used to perform common tasks
- Offer modular tools (functions, classes, utilities) that can be used to solve specific problems
- Increase flexibility and allow to add custom functionality without reinventing solutions

- Examples:
    - **NumPy** (Python)
    - **jQuery** (JavaScript)

- Imagine libraries as a “box of assorted Lego pieces” used to build anything you want

---
## Database Management Systems (DBMS)

- Software tools for storing, retrieving, organising, and securing data in a structured way

- Ensure data is:
    - Accessible
    - Accurate
    - Secure
    - Consistently maintained

- Critical skills include:
    - Designing and structuring databases
    - Writing and optimising queries
    - Ensuring data integrity
    - Performance tuning

- Example: Updating inventory and order history instantly when customer places an order.

---
## Tools and Technologies

#### Version Control Systems (VCS)

- Manage and track code changes over time, allowing multiple developers to collaborate
- Prevent conflicts (like overwriting another's work) and maintain codebase stability
- Enable safe experimentation through branching and merging
- Essential tool: **Git**

#### Containerisation Tools

- Containers are lightweight, portable units that package an application and all dependencies (like libraries, frameworks, configurations) into a single, consistent environment
- Ensure consistent behaviour across development, testing, and production 
- Simplify deployment and scaling

- Common Containerisation Tools:
    - **Docker**
    - **Kubernetes**

#### Continuous Integration / Continuous Deployment (CI/CD)

- Automate several essential stages of the software development workflow

- **Continuous Integration (CI)**:
    - Automatically integrates code changes into the main codebase on a frequent basis
    - Reduces merge conflicts, minimises integration errors, and ensures code stability
    - Helps developers detect issues early by running automated tests during integration

- **Continuous Deployment (CD)**:
    - Automates releasing updates to production once code passes testing
    - Ensures new features and bug fixes reach users quickly and reliably
    - Reduces manual deployment effort and lowers the risk of deployment errors

- **Importance for Back-End Developers**:
    - Enables smooth, efficient development pipelines
    - Ensures fast delivery of updates, improved reliability, and consistent code quality

- Common CI/CD Tools:
    - **Jenkins**
    - **GitLab CI**

---
## Summary

- Back-end engineering requires strong knowledge of server-side languages, frameworks, libraries, database systems, and modern tooling
- Mastering these skills enables developers to build scalable, efficient, and reliable server-side applications
- Effective use of these technologies supports smoother workflows, consistent deployments, and high-performing systems
