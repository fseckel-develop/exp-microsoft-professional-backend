## Overview

- Back-end project planning involves four key phases:
	- Defining requirements
	- Setting objectives
	- Creating timelines
	- Identifying resources
- Proper planning aligns expectations, guides development, and increases the likelihood of successful project execution
- The project **scope** acts as the boundary that shapes all planning decisions

---
## Phases of a Back-End Development Project

#### Defining Project Requirements

- Establish _what_ needs to be built, _who_ it serves, and _why_ it matters
- Requires collaborating with stakeholders to gather detailed needs and expectations
- Ensures the final product is aligned with user needs and client goals
- **Example:**
    - Improve a library’s catalog system with a new search function (**What**)
    - Target users: library staff and patrons (**Who**)
    - Purpose: faster, more accurate search and data retrieval (**Why**)

#### Setting Project Objectives

- Shift from defining _**what**_ to determining _**how**_ the goals will be achieved
- Objectives must be **SMART**:
    - Specific
    - Measurable
    - Achievable
    - Relevant
    - Time-bound
- Clear objectives guide decision-making and resource allocation
- **Example:**
    - “Develop new search functionality that reduces search time by 50% within six months.”

#### Creating Timelines

- Break down tasks and milestones from start to finish
- Establish due dates and create expectations across teams
- Helps track progress and maintain a clear delivery schedule
- **Example:**
    - 1 month design
    - 3 months development
    - 2 months testing & deployment (total 6 months)

#### Identifying Resources

- Determine the tools, technologies, and human capital needed for completion
- Includes selecting infrastructure, team size, and budgeting for necessary tools
- Proper resource allocation reduces risks like shortages and cost overruns
- **Example:**
    - Choose server architecture, assign back-end developers and database administrators, budget for cloud infrastructure and optimisation tools

---
## Role of Project Scope

- Defines what is _included_ and _excluded_ in the project
- Guides all planning phases by setting realistic boundaries
- Ensures requirements, objectives, timelines, and resources all align with what the project is meant to accomplish

---
## Summary

- Effective back-end project planning relies on clear requirements, measurable objectives, realistic timelines, and adequate resources
- When guided by a well-defined scope, these phases create a solid foundation for successful execution and a smooth development process
