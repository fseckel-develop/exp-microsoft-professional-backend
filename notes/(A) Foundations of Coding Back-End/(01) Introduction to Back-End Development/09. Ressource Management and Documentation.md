## Overview

- Successful back-end projects depend on proper **resource management**, **communication**, and **comprehensive documentation**
- These practices prevent delays, maintain project integrity, and ensure smooth collaboration

---
## Resource Management

#### Resource Identification

- Understand project **scope** and **objectives** before identifying resources
- Review project specifications to determine required systems, functionalities, tools, and expertise
- Determine required developer skills, tools, technologies, and infrastructure
- Evaluate existing resources to identify gaps in skills, tools, or systems
- **Example:**
    - Rocket ship back-end → navigation controls, fuel systems, communication systems
    - Requires Python expertise, Git, simulation tools, and specialised infrastructure

#### Resource Allocation

- Assign the right people, tools, and technologies to tasks at the right time
- Clearly define team member responsibilities
- Ensure all team members have access to necessary tools (e.g., software licenses)
- Effective allocation prevents delays, reduces waste, and supports efficient workflow

---
## Communication

#### Importance

- Clear, consistent communication is essential for complex back-end systems
- Ensures alignment, quick issue resolution, and smooth collaboration

#### Communication Strategies

- **Use appropriate communication channels:**
    - Chat → quick updates
    - Email → detailed information
    - Project platforms → structured project tracking
    
- **Schedule regular meetings:**
    - Daily stand-ups, weekly updates, milestone reviews
    - Helps monitor progress and address issues quickly
    
- **Establish feedback loops:**
    - Encourage constructive input and continuous improvement
    - Helps identify problems early and share insights across the team

---
## Documentation

#### Purpose

- Keeps everyone aligned, maintains integrity, supports onboarding, and preserves decisions
- Ensures continuity even when team members join late or roles shift

#### Types of Documentation

- **Codebase:** source code and repository structure
- **Project Plan:** overall timeline + high-level milestones
- **Requirements Document:** detailed description of what the project must achieve
- **Progress Reports:** track ongoing development and deliverables
- **Technical Documentation:** system architecture, API specs, workflows, integrations

#### Storage

- Documentation should be stored in a centralised, accessible location
- Cloud-based tools provide a _single source of truth_ for all stakeholders

---
## Summary

- Identifying and allocating resources ensures the project runs efficiently and avoids shortages
- Strong communication strategies support coordination, clarity, and timely problem-solving
- Thorough documentation maintains transparency, consistency, and long-term project health
