## Overview

- Integrating project planning tools with development workflows creates a cohesive and efficient development process.
- Integration aligns planning, coding, testing, and deployment, improving collaboration and accuracy.

---
## What Is Integration?

- Integration = connecting different tools so they communicate and work together
- Ensures project management and development execution remain synchronised

---
## Project Planning Tools

- Tools like **Jira**, **Trello**, and **Asana** support:
    - Task management
    - Progress tracking
    - Team communication
    - Deadline and workflow visibility
- Integration allows these tools to directly interact with development environments and code repositories

---
## Development Tools

#### Integrated Development Environments (IDEs)

- Primary workspace for developers to write, test, and debug code
- Examples: **VS Code**, **Eclipse**, **Xcode**
- Integration example:
    - Jira + VS Code → manage tasks inside the IDE with automatic syncing to Jira

#### Version Control Systems (VCS)

- Track and manage code changes across multiple developers
- Example: **Git/GitHub**
- Integration benefits:
    - Linking commits to task updates
    - Automatically updating Jira/Trello/Asana task status via code commits
    - Real-time visibility into development progress

---
## Steps for Integrating Tools

#### 1. Define the Workflow

- Outline key stages: planning → development → testing → deployment
- Identify where integration brings most value (e.g., linking commits, updating task statuses)

#### 2. Set Up Integrations

- Three common integration approaches
- **Native Integrations**
	- Built-in connections requiring no extra tools or code
	- e.g. Jira’s built-in GitHub/Bitbucket integrations
- **Middleware Tools**
	- Third-party connectors bridging incompatible systems
	- Examples: Microsoft BizTalk, IBM WebSphere, Flow-based platforms
- **Custom APIs**
	- Custom-coded integrations for maximum control and flexibility

#### 3. Configure Automations

- Enable automations such as:
    - Commits updating task status
    - Pull requests triggering review workflows
    - Successful builds updating completion states in the planning tool

---
## Benefits of Tool Integration

- **Enhanced productivity**
    - Reduces manual updates and repetitive administrative tasks
    - Developers spend more time coding

- **Improved project tracking**
    - Centralised, real-time view of project status
    - Faster identification of bottlenecks

- **Better collaboration**
    - Teams share updates automatically
    - Everyone stays aligned and informed

---
## Summary

- Integrating project planning tools with development tools streamlines transitions between planning and coding.
- Leads to more efficient workflows, stronger collaboration, and more accurate project execution.
