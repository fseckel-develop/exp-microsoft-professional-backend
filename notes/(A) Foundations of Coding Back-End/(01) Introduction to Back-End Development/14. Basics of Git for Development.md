## Overview

- Git enables teams to track changes, collaborate simultaneously, and experiment safely —similar to chefs testing variations without altering the original recipe
- Git preserves code integrity while supporting flexible, parallel development

---
## What Git Provides

- Tracks all code changes over time
- Allows multiple developers to work in parallel
- Enables easy rollbacks to previous versions
- Supports feature development without affecting the main codebase

#### Git vs. GitHub

- **Git** → distributed version control system used locally to manage project history
- **GitHub** → cloud-based platform for storing repositories, sharing work, collaborating, and contributing to team or open-source projects

#### Repositories

- A repository (repo) contains all project files and their entire change history
- Repos store snapshots of the codebase, enabling version tracking and rollbacks

#### Branching

- Branching = creating a separate copy of the main code so changes can be made safely
- Developers can:
    - Add features
    - Fix bugs
    - Experiment
    - Test changes
- Once stable, branches are merged back into the main branch.

---
## Essential Git Commands

#### >> `git clone`

- Creates a **local copy** of a remote repository
- Used to start working on a project on your own machine
- Example: `git clone <repository-url>`

#### >> `git branch`

- Creates a new branch for features or fixes
- Keeps work isolated from the main codebase
- Example: `git branch feature-authentication`

#### >> `git commit`

- Saves a snapshot of changes
- Includes a descriptive commit message explaining what changed
- Example: `git commit -m "Initialise user registration form"`

#### >> `git push`

- Uploads your local commits to the remote repository (e.g., GitHub)
- Makes your updates available to teammates
- Example: `git push origin feature-authentication`

#### >> `git pull`

- Fetches and merges the latest changes from the remote repository into your local branch
- Ensures you’re working with the most up-to-date version
- Example: `git pull origin main`

---
## Typical Git Workflow

1. Clone the repository
2. Create a new feature branch
3. Write code
4. Commit changes with meaningful messages
5. Push branch to remote (GitHub)
6. Pull updates to stay in sync
7. Create new branches for new features as needed

---
## Summary

- Git is fundamental for back-end development, enabling:
    - Reliable version tracking
    - Safe experimentation through branching
    - Efficient collaboration
    - Clean and maintainable project history
- Mastering Git commands significantly boosts team productivity and workflow effectiveness.
