## Overview

- Before writing code, developers must **design the program’s logical flow**
- Two common tools for planning logic: **flowcharts** and **pseudocode**
- These tools help **visualise**, **communicate**, and **translate logic** into code

---
## Flowcharts

- **Definition**: Diagrams representing processes using **shapes** and **arrows** to show application logic and user paths

- **Purpose**: Plan logic before coding, simplify understanding of complex processes

- **Example: E-commerce checkout flow**
    - **Start**: _Oval shape_ marking the beginning
    
    - **Process steps**: _Rectangles_ for actions such as:
        - Select Products
        - Add to Cart
        - View Cart
        - Proceed to Checkout
        - Enter Shipping Details
    
    - **Decision point**: _Diamond shape_ for multiple paths (e.g. select payment method)
    
    - **End**: Oval shape marking completion of the process

- **Benefits**:
    - Visualises the order of events and user paths
    - Arrows indicate direction, which can translate to **logical operators** in code

---
## Pseudocode

- **Definition**: High-level, readable description of program logic in **natural language**, not programming syntax

- **Purpose**: Bridges the gap between **conceptual design** and **actual code implementation**

- **Example: Add products to cart step**
    
    1. Start function AddToCart(productID, quantity)
    2. Retrieve product details from database
    3. If product available:
        - Option A: Add product to cart
        - Option B: Update inventory
    4. Confirm addition to cart
    5. If product not available: Show error message
	6. End function

- **Benefits**:
    - Outlines steps and conditions clearly
    - Provides a roadmap for coding logic efficiently

---
## Summary

- **Flowcharts** help **visualise program flow**, map user actions, and identify decision points
- **Pseudocode** translates visual logic into a **step-by-step textual plan** ready for coding
- Combining flowcharts and pseudocode enables developers to **plan, communicate, and implement logical processes effectively**, ensuring smooth coding and fewer errors
