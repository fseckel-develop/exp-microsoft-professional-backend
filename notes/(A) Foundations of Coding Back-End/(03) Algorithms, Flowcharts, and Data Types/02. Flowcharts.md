
- Flowcharts visually represent **processes**, making complex workflows easier to understand, design, communicate, and debug
- Use **standardised symbols** connected by arrows to show logical flow from start to finish

---
### Core Flowchart Symbols

- **Ovals → Start/End points**
    - Indicate where a process begins and ends.

- **Rectangles → Process steps**
    - Represent actions, operations, or tasks.

- **Diamonds → Decision points**
    - Represent yes/no or true/false branching.

- **Arrows → Flow direction**
    - Show sequence and movement through the process.

---
### How to Create an Effective Flowchart

- **Define the process**
    - Identify the start, key steps, decisions, and end.

- **Break down into clear steps**
    - Use rectangles for actions and diamonds for decisions.
    - Keep steps necessary, logical, and sequential.

- **Use standard symbols consistently**
    - Oval → start
    - Rectangles → actions
    - Diamonds → decisions
    - Arrows → directional flow

- **Review for accuracy**
    - Ensure steps are complete, clear, and in the correct order.    

---
### Example: Login Process Flow

- Start (oval)
	- Display login form (rectangle)
		- Enter credentials (rectangle)
			- Submit form (rectangle)
				- Validate credentials? (diamond)
				    - Yes → log in
				    - No  → create account → loop back to credential validation
						- End (oval)

---
### Analysing Flowcharts for Accuracy

- **Check each step**
    - Ensure all necessary actions are present and correctly ordered

- **Verify decision points**
    - Ensure both outcomes (yes/no) are accounted for and lead to correct next steps

- **Validate flow of control**
    - Arrows must connect logically; no dead ends or broken loops

- **Confirm start and end points**
    - Must be clearly defined and logically placed

---
### Common Flowchart Mistakes

- **Missing steps**
    - Skipping required actions leads to incomplete logic

- **Incorrect sequence**
    - Steps placed out of order disrupt process clarity

- **Ambiguous decision points**
    - Vague questions or unclear criteria cause confusion

- **Poorly defined start/end points**
    - Unclear entry or exit makes flow hard to interpret

---
### Conclusion

- Flowcharts provide **clear, visual structure** for understanding, designing, and debugging
- Using standard symbols, verifying logical flow, and avoiding common mistakes ensures reliable and easy-to-follow diagrams
