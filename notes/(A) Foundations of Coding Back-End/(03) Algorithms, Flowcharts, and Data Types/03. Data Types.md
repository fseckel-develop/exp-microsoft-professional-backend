
- Data types are the _ingredients_ of programming; variables are the _containers_ that hold them
- Data types determine **what values can be stored**, **what operations are allowed**, and **how data is stored in memory**
- Understanding them prevents errors, improves reliability, and supports structured decision-making in programs

---
## Core Data Types

- **Integers (int)**
    - Whole numbers, positive or negative
    - Used for counting, indexing, arithmetic

- **Doubles (double / float)**
    - Decimal / floating-point numbers
    - Used for precision calculations (finance, science, measurements)

- **Strings (string)**
    - Text data: letters, symbols, numbers-as-text
    - Used for messages, names, input, formatting
    - Supports operations like concatenation

- **Booleans (bool)**
    - True/False values
    - Crucial for conditions, branching, control flow (if/else, loops)

---
## Data Types Working Together

- Programs often combine multiple types, e.g., a weather app:
    - int temperature = 25
    - double humidity = 75.5
    - string description = “Sunny”
    - bool stormWarning = false

---
## Type Safety

- **Type-Safe Languages**
    - Enforce strict rules (e.g., C#, Java)
    - Reduce bugs, ensure consistency, improve security

- **Non-Type-Safe Languages**
    - Flexible, allow mixed types (e.g., JavaScript, Python)
    - Faster prototyping but higher risk of hidden bugs

---
## Identifying Data Types

- Use functions like **GetType()** to inspect a variable’s type
- Allows debugging, validation, and safe handling of unknown or user-provided input

---
## Checking Data Type Compatibility

- Type checks determine whether a variable matches the required type
- Conditional logic (if/else) ensures variables are used correctly before operations are applied

---
## Data Type Conversion

- Converting a value from one type to another (e.g., string → int)
- Necessary when input formats differ from operation requirements

#### Implicit Conversion

- Conversion happens automatically by the language
- Example: int + double → double
- Convenient but risky if behaviour is unexpected

#### Explicit Conversion

- Developer manually specifies conversion
- Two common methods:
    - **Casting** – For compatible numeric types (double → int)
    - **Parsing** – Convert text → number (string “123” → int 123)

#### Validating Conversions

- Methods like **TryParse()** safely test if conversion is possible
- Prevents crashes and handles invalid input gracefully

#### Practical Uses of Conversion

- User input handling (strings → ints)
- Financial calculations (double precision)
- Display formatting (numbers → formatted strings)
- Data interoperability across systems
