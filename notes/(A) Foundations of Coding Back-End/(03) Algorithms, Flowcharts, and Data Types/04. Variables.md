
- Named storage locations in memory used to store, retrieve, and manipulate data during a program’s execution
- They allow programs to remember information (customer name, age, price, purchase date)
- Variables support flexible program behaviour by allowing data to change or remain constant depending on how they are declared

---
## Why Variables Matter

- Enable accurate calculations by ensuring data is stored in the correct format
- Prevent errors caused by incorrect types
- Provide structure, clarity, and flexibility
- Essential for debugging, maintenance, and long-term stability

---
## Naming Best Practices

- Use descriptive names (userAge, totalAmount, firstName)
- Avoid single-letter names
- Follow camelCase in C#
- Choose names that align with their scope

---
## Variable Scope

- Global scope: accessible anywhere in the program
- Local scope: accessible only within a specific function or block
- Helps control access and reduce unintended modifications

---
## Type Safety (C# vs. JavaScript)

**C#**
- Type-safe
- Variables must be explicitly typed
- Prevents mismatched type errors

**JavaScript**
- Loosely typed
- Variables can store any type
- Uses flexible keywords (var, let, const)

---
## Variable Declaration C# Syntax

- int age = 25;
- double price = 19.99;
- string name = "Alice";
- bool isStudent = true;
- int[] numbers = {1,2,3};

(**JavaScript** equivalents use var, therefore loosely typed)

---
## Mutable vs. Immutable Variables in C# 

- **Mutable Variables**
	- Can be changed after declaration
	- Default behaviour in C# for standard types
	- Example:    int score = 10; → score = 20;

- **Immutable Variables**
	- const → must be assigned at declaration
	- readonly → can be assigned once (declaration or constructor)
