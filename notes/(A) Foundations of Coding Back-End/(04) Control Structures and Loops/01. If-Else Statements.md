
- Enable decision-making in programs based on whether a condition is **true or false**
- Allow code to follow different paths (like a **fork in the road**)
- Essential for binary choices (e.g., allowed/denied, pass/fail, yes/no)

---
## How If-Else Works

- Program evaluates a **condition**
- If the condition is **true**, one code block runs
- If the condition is **false**, an alternative block runs
- Prevents code from running in a straight line by adding logical control

---
## Basic C# Syntax

- if (condition) { ... } else { ... }
- Conditions must evaluate to **true or false**
- Curly braces define the code block executed for each outcome

---
## Binary Decisions in Practice

- Ideal for scenarios with only **two possible outcomes**
- Example: Determine a student’s result:
    - grade ≥ 50 → “Passed”
    - grade < 50 → “Failed”
- Example: Check whether a user meets a required age (18+)
	- If age >= 18 → print “Access granted."
	- Else → print “Access denied.”

---
## Advanced If-Else Structures

#### Nested If-Else

- If-else statements placed **inside** other if-else blocks
- Used for **dependent** conditions
- Example: 
  Discounts based on purchase amount → then check membership → apply correct discount

#### Chained If-Else (Else-If Ladder)

- A sequence of independent conditions checked one by one
- Used when conditions are **mutually exclusive**
- Example: Shipping cost based on location → local → domestic → international

---
## Avoiding Spaghetti Code

- Keep conditions simple and readable
- Break complex logic into functions
- Add comments to clarify decision paths

---
## Key Takeaways

- If-else statements are a core control structure for guiding program flow
- Useful for simple and advanced decision-making
- Nested and chained versions help manage more complex logic
- Clean structure and planning prevent messy, unmaintainable code
