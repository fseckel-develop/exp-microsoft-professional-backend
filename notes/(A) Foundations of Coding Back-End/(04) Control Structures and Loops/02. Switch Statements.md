
- Used to evaluate a variable and execute different code depending on its value
- More readable and organised than multiple chained if-else statements
- Ideal for scenarios with several possible outcomes

---
## Basic Structure & C# Syntax

- Begins with the switch keyword followed by a variable in parentheses
- Contains **cases**, each representing a possible value of the variable
- A **break** statement ends each case to prevent fall-through
- A **default** case handles unmatched values
- switch (variable) {case value1: ... break; case value2: ... break; ... default: ... break; }

---
## Default Case

- Runs when no case matches
- Prevents unexpected behaviour or program crashes
- Does not need to be at the end of the switch structure, though this is common

---
## Switches in Practices

- Handling multiple discrete values (e.g., menu options, commands, categories)
- Replacing long if-else chains to improve readability
- Grading systems, vending machines, input routing, etc.
- Example: Vending Machine
	- Variable button is evaluated
	- Cases: "Water", "Soda", "Coffee"
	- Each case prints a message like “Dispensing water”
	- Default prints “Invalid option”

---
## Multi-Case Actions (Shared Cases)

- Multiple cases can share one action
- Eliminates redundant repeated code
- Example:
    - case "Monday":
    - case "Wednesday":
    - Both run the same action (e.g., print “Eat cereal”)

---
## Pattern Matching in Switch Statements

- Allows matching patterns instead of exact values
- Example:
    - Input contains “jazz” → output “jazz music”
    - Input contains “rock” → output “rock music”
- Useful for analysing text, input categories, or complex conditions
- Example:
	- Input is larger than 95 → Syntax: case int n when (n > 95): ... break;
	- Input is larger than 85 → Syntax: case int n when (n > 85): ... break;
	- ...

---
## Benefits of Complex Switch Statements

- Cleaner handling of multiple related conditions
- Better organisation in scenarios with shared logic
- Flexibility with pattern matching and fallback behaviour
