## For Loops

- Used when the number of repetitions is known in advance
- Structure includes: **initialisation → condition → increment → code block**
- Ideal for tasks like processing arrays or running fixed-count operations
- Executes code repeatedly until the condition becomes false
- Syntax: for (int i = 0; i < 5; i++) { ... }

---
## While Loops

- Used when the number of repetitions is **unknown**
- Repeats **as long as a condition remains true**
- Checks the condition **before** each iteration
- Ideal for reading input, waiting for events, or monitoring changes
- Syntax: while (condition) { ... }

---
## Do-While Loops

- Guarantees the loop body runs **at least once**
- Condition is checked **after** the code block executes
- Useful for input validation or initial mandatory operations
- Syntax: do { ... } while (condition);

---
## Comparing the Loops

- **For Loop:** best when iteration count is known
- **While Loop:** best when repetition depends on changing conditions
- **Do-While Loop:** best when the action must run at least once

---
## Practical Applications of Loops

- **Iterating Through Arrays**:
	- For loops are ideal for indexing through arrays or lists
	- Each iteration processes one element, such as printing it or updating its value
	- Example concept: looping through an array and increasing each price by 5%

- **Generating Sequences**
	- While loops repeat until a condition becomes false
	- Suitable for tasks like generating numbers from 1 to 10
	- The loop continues as long as the counter meets the loop condition

- **Ensuring At Least One Iteration**:
	- Do-while loops always run once before checking any condition
	- Useful for tasks where one action (like prompting user input) must occur at least once

---
## Combining Loops with Control Structures

- Loops can be paired with control structures (if-else or switch) to process data and make decisions repeatedly
- Common use case: evaluating each element in a collection and deciding what action to take

#### **Using Loops with If-Else Statements**

- If-else statements inside loops allow dynamic decision-making for each iteration
- Example concept: checking whether each number in an array is even or odd
- The modulo operator (%) returns the remainder of a division:
    - If number % 2 == 0 → even
    - Otherwise → odd
- When iterating through an array, each value can be evaluated depending on this check.

#### **Using Loops with Switch Statements**

- Switch statements inside loops efficiently handle multiple possible values.
- Example: iterating through weather forecasts and displaying a different message for each one
- For each element in the array:
    - Case “Sunny and warm” → message about sunglasses
    - Case “Rainy with thunderstorms” → message about umbrella
    - Case “Cloudy and cold” → message about warm clothing

## Loop Optimisation

- Inefficient loops slow programs, especially with large datasets.
- Optimisation improves speed, reduces resource usage, and increases scalability.

- **Early Termination with break**:
	- When a desired condition is met, the loop can exit early
	- This avoids unnecessary iterations and improves performance

- **Avoiding Nested Loops**:
	- Nested loops multiply the number of iterations and can degrade performance.
	- Where possible:    
	    - Replace nested loops with separate loops
	    - Use more efficient data structures
	    - Reduce redundant comparisons
