
- Functions are reusable blocks of code that take input, process it, and return an output
- They make programs more efficient, easier to manage, and quicker to update
- Developers use functions to avoid repeating code and to handle common or complex tasks in a clean, organised way

---
## Why Functions Matter

- **Reusability:** A function can be used anywhere in the program without rewriting logic
- **Modularity:** Breaks large problems into small, manageable pieces
- **Maintainability:** Updating the function updates behaviour everywhere it’s used
- **Consistency:** Ensures the same logic runs the same way across the entire application

---
## Core Structure of a Function

- **Function declaration:** Name and purpose of the function (e.g., calculateTotal, calculateArea)
- **Parameters:** The input values the function processes (e.g., items, length, width)
- **Function body:** Logic that performs the task (e.g., looping through items, multiplying values)
- **Return value:** The final result the function sends back to the caller

---
## How Functions Work in a Program

- A function is activated by **calling it** using its name followed by parentheses
- If parameters exist, they are placed inside the parentheses (e.g., calculateTotal(items))
- When called, the program enters the function body, executes its logic, and returns a value if defined
- The returned value can be stored, displayed, or used directly in further operations

---
## Real-World Example

- A retail site can use a function such as calculateTotal() to:
    - Receive the list of items
    - Loop through prices
    - Compute the total
    - Return the final cost for display on the cart, checkout, or summary pages
- Updating the calculation logic only requires editing the function once.
