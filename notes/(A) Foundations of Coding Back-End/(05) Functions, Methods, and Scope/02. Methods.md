
- Methods are reusable blocks of code **associated with a class**, making them a core part of object-oriented programming (OOP)
- While **all methods are functions**, not all functions are methods; functions can exist independently, whereas methods belong to objects and can use or modify their data

---
## What Methods Are

- Methods behave like functions but operate **within the context of a class**
- They can access **member variables** (the class’s internal data)
- They define the **behaviour** of objects—how they act, respond, or change

---
## Methods vs. Functions

- Functions → standalone tools usable anywhere in code
- Methods → tied to a class and work with the class’s data
- Functions offer general-purpose reuse; methods offer behaviour specific to a class

---
## Why Methods Matter

- Enable OOP behaviour by coupling data + functionality
- Improve code organisation by grouping related logic inside classes
- Create predictable, maintainable, scalable application structures

---
## Method Structure in C# 

- **Access modifier:** controls visibility (public, private, protected, internal)
- **Return type:** describes what value the method returns (string, int, double, void)
- **Method name:** descriptive identifier explaining its purpose
- **Parameters:** input values passed in to operate with
- **Method body:** code that performs the task and (optionally) returns a result
- Example:
	- public int AddNumbers(int a, int b) → adds two integers and returns an int.

---
## Calling Methods in C# 

- Calling methods lets you reuse code and keep programs organised
- A method has a name, return type, and parameters; 
  you call it like: int result = AddNumbers(5, 10);
- Methods can return values using return, e.g.: public int GetNumber() { return 42; }
- Arguments must match the method’s parameters, e.g.: int sum = CalculateSum(5, 10);
- Common examples:
    - Display a message: DisplayWelcomeMessage();
    - Calculate a sum: CalculateSum(a, b);
    - Validate age: IsUserOldEnough(age);

---
## Common Use Cases

- **Calculations:** repeating math logic (Add, CalculateDiscount, ComputeAverage)
- **Event handling:** responding to interactions (Button_Click, OnPlayerMove)
- **Data manipulation:** sorting, formatting, processing lists or collections
- Methods support cleaner, reusable, consistent logic across an application
