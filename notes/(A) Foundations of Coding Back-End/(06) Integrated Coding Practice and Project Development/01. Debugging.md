## Introduction

- Debugging is the process of identifying, reproducing, isolating, and fixing errors in a program
- Helps ensure applications run reliably and as intended

### Understanding Error Messages

- Error messages provide clues such as error type, file, and line number
- Example: _NullReferenceException_ indicates an object wasn’t initialised before use
- Use error details (line numbers, stack traces) as starting point when searching for the cause

### Reproducing & Isolating Errors

- Reproduce errors consistently to confirm they aren’t random
- Isolate the problem using tools like breakpoints, watches, or print statements to narrow down faulty logic

### Fixing Errors

- Analyse surrounding code to determine what caused the issue
- Look for incorrect logic, wrong assumptions, or unexpected input
- Apply fixes, then retest to confirm the issue is resolved

---
## Common Error Types

**Logic Errors**:
- Program runs but produces incorrect results
- Caused by faulty logic, incorrect formulas, or off-by-one mistakes
- Best found through flowchart comparison and test cases

**Syntax Errors**:
- Violations of language rules (missing semicolon, wrong variable name)
- Compiler or interpreter usually catches them early

**Runtime Errors**:
- Occur only during execution (e.g., divide by zero, out-of-range access)
- Found by thorough testing, including edge cases

---
## Debugging Tools & Techniques

**Watches**:
- Track variable values in real time within an IDE
- Useful for spotting sudden or incorrect value changes

**Logging**:
- Records program behaviour at key points, stored for later review
- Helps trace complex processes across long execution flows

**Print Statements**:
- Quick, simple way to display runtime values or code paths
- Ideal for fast, small-scale debugging

**Breakpoints**:
- Pause execution at specific lines to inspect variables and state
- Allow step-by-step execution to isolate complex bugs

### Combining Techniques

- Use multiple tools together for the best results
- Example workflow: 
  log overall behaviour → add print statements for specifics 
  → set breakpoints for detailed inspection → monitor variables with watches.

---
## Why Debugging Matters

- Leads to more reliable, maintainable, and predictable programs
- Builds understanding of program flow and strengthens problem-solving skills
