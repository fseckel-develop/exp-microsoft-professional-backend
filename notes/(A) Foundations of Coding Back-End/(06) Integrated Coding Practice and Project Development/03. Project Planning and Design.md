
- Successful software development begins with **careful planning and structured design**
- Ensures code is organised, efficient, scalable, and aligned with project goals

---
## Define Project Requirements

- Requirements describe what the project must accomplish to succeed
- Two categories:
    - **Functional requirements** 
      → describe what the app _does_ (e.g., add, review, complete tasks)
    - **Non-functional requirements** 
      → describe _how_ the app performs (usability, speed, performance limits)

---
## Set Clear Objectives

- Objectives are **measurable results** within a defined timeline
- Examples for a task app:
    - Build core task features within three months
    - Implement data storage within four months
    - Fix 95% of critical bugs before release
- Helps with tracking progress and allocating resources effectively

---
## Create a Design Outline

- Converts requirements + objectives into a **step-by-step development blueprint**
- Breaks the project into modules such as:
    - Viewing tasks
    - Adding tasks
    - Marking tasks complete
- Ensures no essential features are overlooked

---
## Plan Coding Components

- Identify the programming elements required to build each module
- Key components include:
    - **Variables** → store task data (strings in lists or arrays)
    - **Methods** → implement reusable actions (view, add, complete task)
    - **If/else statements** → handle conditions (e.g., empty input fields)
    - **Loops** → repeat actions (displaying all tasks)
