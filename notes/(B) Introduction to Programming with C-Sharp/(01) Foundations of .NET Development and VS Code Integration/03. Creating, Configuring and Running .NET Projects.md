## Creating a .NET Project

- Open Command Palette → **Ctrl+Shift+P / Cmd+Shift+P**
- Run **“.NET: New Project”** from the palette
- Choose a template (Console App recommended for beginners)
- Select a folder for the project
- Enter a project name → VS Code creates the directory
- Confirm **“Trust the authors”**
- View generated files in Explorer (e.g., _Program.cs_)
- Run the project → **dotnet run** → outputs _Hello World_

---
## Configuring a .NET Project

#### Understanding the .csproj File:

- Acts as the project’s blueprint for build, run, test, and configuration
- Written in **XML**
    - Organised with nested tags
    - Tags may include attributes such as Sdk="Microsoft.NET.Sdk"
- Specifies **Target Framework** (e.g., net8.0)
- Scaffolding tools like **C# Dev Kit** generate this file automatically

#### Adding NuGet Packages

- Packages = reusable external code (added with **NuGet**)
- Add packages via Command Palette → **“NuGet: Add Package”**
- Example: _Pastel_ package added to colorise console text
- Package reference appears in _.csproj_ under < ItemGroup >
- Add using Pastel; to import the library in code
- Use IntelliSense to access functionality (e.g., Color.LightGreen)

## Building and Running a .NET Project

#### Running the Application

- Run via terminal → **dotnet run**
- Or via VS Code GUI → **Run → Run and Debug**
- First run may take longer (initial build process)

#### What Happens When Code Runs

- C# is a **compiled language**
- dotnet run → compiles (builds) → then executes
- Output files stored in the **bin/** folder

#### Building Only (Without Running)

- Build via Command Palette → **“.NET: Build”**
- Or via terminal → **dotnet build**
- Useful when preparing code for distribution or validating compilation

#### Cleaning Build Outputs

- Remove compiled artefacts with → **dotnet clean**
- Cleans **bin/** (compiled binaries)
- Does **not** affect source files
- Helpful when binaries get corrupted or outdated
