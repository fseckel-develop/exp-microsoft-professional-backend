## Overview

- Key concepts in logical thinking for programming
- Applied in software development to:
	- Debug code
	- Optimise performance
	- Design new features effectively
- Main approaches:
	- Deductive reasoning
	- Inductive reasoning
	- Problem decomposition

---
## Deductive Reasoning (general → specific)

- Drawing specific conclusions from general premises
- Common Use Case: **Debugging a Function**
	- compare expected and actual outcomes to identify the problem
	- Apply general principles to hypothesise the source of the error
	- Test hypothesis by modifying code and verifying resolution

---
## Inductive Reasoning (specific → general)

- Forming general conclusions from specific instances or observations
- Use Case: **Optimising Algorithms**
	- Analyse performance to identify patterns or repeated operations that slow execution
	- Refactor code to remove redundancies and improve efficiency
	- Validate changes by testing improved performance under various conditions

---
## Problem Decomposition

- Breaking complex problems into smaller, manageable parts
- Two main approaches:

	- **Top-Down Approach**
		- Start with a high-level overview and break it into smaller components or tasks
		- Ideal when overall structure is known, e.g., developing a feature that follows established patterns

	- **Bottom-Up Approach**
		- Begin with basic elements or functions and combine them to build a complete system
		- Suitable when details are unclear or development requires flexibility, e.g., building a new feature from scratch
