## Overview

- Debugging is finding and fixing errors or bugs in a computer program
- Critical skill for developers to ensure smooth and effective code execution
- Involves correcting mistakes, understanding causes to improve software quality and reliability

---
## Common Types of Errors in Programming

- **Syntax errors**
	- Occur when code violates programming language rules (e.g., missing semicolons, unmatched brackets)
	- Usually caught by the compiler, making them easier to detect and correct

- **Runtime errors**
	- Happen while the program is running rather than during compilation
	- Often cause crashes or unexpected behaviour
	- Example: division by zero, encountered only during execution

- **Logical errors**
	- Mistakes in code logic that produce incorrect results without crashing
	- Example: using a wrong formula that outputs incorrect results even if the program runs
	- Often hardest to detect because program appears functional

---
## Key Debugging Techniques

- **Breakpoints**
	- Pause code execution at specific points
	- Allow inspection of program state and behaviour at that moment

- **Inspecting variables**
	- Check variable values at different code points
	- Helps verify correct data manipulation

- **Stepping through code**
	- Execute code line by line to closely observe behaviour
	- Pinpoints exact line where error occurs

- **Printing error messages**
	- Insert messages in code to track execution flow
	- Helps identify where and why code fails, especially for unexpected behaviour or crashes

---
## Using Debugging Tools

- Visual Studio Code Debugger is a highly effective tool
- Features include setting breakpoints, inspecting variables, and stepping through code
- Enables quick identification and fixing of errors, improving software quality and reliability
- Set breakpoints by clicking next to the line number where program should pause
