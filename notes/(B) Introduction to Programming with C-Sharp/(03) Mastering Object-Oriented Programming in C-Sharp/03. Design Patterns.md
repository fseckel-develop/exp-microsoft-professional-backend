
- Design patterns are **reusable solutions** to common software design problems
- They offer **standardised, proven approaches** that improve readability, maintainability, and scalability
- Following patterns aligns code with **industry best practices** and supports consistent team collaboration

---
## Categories of Design Patterns

### Creational Patterns

Focus on **how objects are created**, enabling flexibility and preventing unnecessary complexity.

- **Singleton Pattern**
    - Ensures **only one instance** of a class exists
    - Provides a **global access point** to shared resources (e.g., configuration, database connection)
    - Uses a **private constructor** and a static access method to control instance creation
    
- **Factory Pattern**
    - Separates **object creation** from object usage
    - A factory decides **which specific object type to create** based on input
    - Supports flexibility when adding or modifying object types

### Structural Patterns

Deal with **how classes and objects are composed** to form larger systems.

- **Adapter Pattern**
    - Allows **incompatible interfaces** to work together
    - Acts as a **bridge** so components can interact without modifying their internal logic
    - Reduces system impact when a component changes — only adapter must be updated

### Behavioural Patterns

Define **how objects interact and communicate**.

- **Observer Pattern**
    - Enables one subject to notify multiple observers of state changes
    - Useful for real-time updates, such as **UI refreshes when data changes**
    - Promotes loose coupling by separating data producers from data consumers

---
## Benefits of Using Design Patterns

- Improve code **readability and structure**
- Increase **reusability** and reduce repeated logic
- Support **scalable system architecture** as applications grow
- Simplify **maintenance and troubleshooting** by following known, predictable structures
- Make systems easier to **extend or modify** without major rewrites

---
## Practical Examples (Conceptual)

- **Singleton:** A database connection manager with only one active instance in the application
- **Factory:** A notification system that produces Email or SMS objects depending on user input
- **Adapter:** Connecting a class that provides width to a system expecting radius
- **Observer:** A weather station notifying phone + desktop displays when temperature changes
