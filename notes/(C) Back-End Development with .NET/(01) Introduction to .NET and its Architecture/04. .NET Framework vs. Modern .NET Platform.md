## Overview

- The .NET Framework has powered Windows development for nearly 20 years, 
  but modern .NET has replaced it as the preferred platform.
- Understanding their differences is key to choosing the right technology.

---
## Legacy .NET Framework

- **Core Components**:
	
	- **Common Language Runtime (CLR)**
	    - Enables cross-language development via Common Intermediate Language (CIL).
	    - Allows seamless integration of code written in different .NET languages.
	
	- **Base Class Library (BCL)**
	    - Provides standardised classes for file I/O, collections, and data manipulation.
	    - Ensures consistent tools across all .NET-supported languages.

- **Limitations of the .NET Framework**:
	
	- **Windows-only**
	    Cannot run natively on macOS or Linux.
	
	- **Lower performance**
	    Lacks modern runtime optimisations; slower startup and larger deployments.
	
	- **Monolithic architecture**
	    Applications include unnecessary components, increasing size and complexity.
	
	- **Not recommended for new projects**
	    Microsoft now directs developers to modern .NET.

- **Why .NET Framework Still Matters**:
	
	- **Critical for legacy enterprise applications**
	    Many organisations maintain long-standing Windows apps using WinForms or WPF.
	
	- **Long-term Microsoft support**
	    Ongoing security patches ensure stability for businesses relying on it.
	
	- **Deep Windows integration**
	    Ideal for apps requiring Windows-specific capabilities.

---
## Modern .NET Platform (from .NET 5 to .NET 8)

- **Unified Platform**
	- Combines **.NET Core**, **.NET Framework (parts)**, and **Xamarin** into a single ecosystem.
	- Enables consistent development for mobile, cloud, desktop, web, and IoT.

- **Cross-Platform Capabilities**
	- Runs on **Windows, macOS, Linux, iOS, Android**, 
	  and even in **web browsers** via WebAssembly.
	- “Write once, run anywhere” reduces duplication and accelerates delivery.
	- Same tools, languages, and APIs across all operating systems.

- **Open-Source and Community-Driven**
	- Fully open-source with global contributions.
	- Faster innovation and a vibrant ecosystem of tools and libraries.
	- Developers can contribute directly to the platform’s future.

- **Cloud-Ready Design**
	- Strong integration with cloud platforms—especially **Microsoft Azure**.
	- Supports modern DevOps practices: 
	  automated testing, CI/CD pipelines, containerised workloads.
	- Built for handling large-scale, high-demand applications.

- **Flexible Language Support**
	- Supports multiple programming languages such as **C# and F#**.
	- Developers can choose the paradigm best suited to their project.

---
## Key Takeaway

- **.NET Framework** remains vital for legacy, Windows-specific applications.
- **Modern .NET** is the future: 
  cross-platform, open-source, cloud-ready, and unified across all application types.
- Knowing both helps developers maintain older systems while building next-gen solutions.
