## What Packages Are

- A _package_ is a bundle of libraries, config files, and metadata.
- It simplifies code sharing, reuse, and dependency management.
- Packages save development time by providing ready-made, optimised solutions.
- They are maintained by communities or organisations that offer updates and patches.

---
## Why Packages Matter

- Reduce development overhead by avoiding reinventing common components.    
- Often follow industry best practices for security and performance.
- Easy updating ensures bug fixes and improvements are readily accessible.

---
## What NuGet Is

- NuGet is the official package manager for .NET.
- Provides access to a massive repository of community and organisational packages.
- Simplifies installation, updating, configuration, and removal of libraries.

---
## Installing, Updating and Removing Packages

**VS Code**:
- Open VS Code and load the project.
- Use the command palette → Add NuGet Package (or Update NuGet Package)
- Search for the package (e.g., _Newtonsoft.Json_).
- Select the project and choose a version.
- Removing a Package: Command palette → Remove NuGet Package.

**Command Line Interface**:
- Navigate to the project folder.
- Adding a Package:      dotnet add package < PackageName > 
- Updating a Package:   dotnet add package < PackageName > --version < Version >)
- The package appears in the project dependencies.
- Removing a Package:  dotnet remove package < PackageName >

---
## Dependencies

- A dependency is an external module your code needs.
- Packages may depend on other packages.
- NuGet auto-resolves and installs all required dependencies.

---
## Semantic Versioning (SemVer)

- Format: **Major.Minor.Patch**
- **Major** → breaking changes
- **Minor** → new features, non-breaking
- **Patch** → bug fixes, stability improvements
- NuGet uses SemVer to ensure predictable, safe version updates.

---
### Version Control in NuGet

- **Pinning a version** (e.g., 10.4.31) prevents automatic updates.
- **Version ranges** allow flexibility:
    - Example: (1.0,2.0) → Accept versions ≥1.0.0 and <2.0.0
- NuGet resolves conflicting versions by picking the most compatible one.

---
## Key Takeaways

- NuGet streamlines dependency management in .NET.
- Reduces manual work, prevents conflicts, and ensures reliable updates.
- Knowing package handling and versioning boosts project stability and developer productivity.
