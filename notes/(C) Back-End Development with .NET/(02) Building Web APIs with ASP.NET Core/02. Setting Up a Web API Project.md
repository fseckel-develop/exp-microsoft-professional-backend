## Introduction

- APIs keep applications connected and functional.
- Building efficient APIs is essential for responsive and scalable software.
- ASP.NET Core provides the tools to build modern, fast web APIs.

	Console: `dotnet new webapi -o <projectName>`

---
## Program.cs

- Serves as the **entry point** of the application.
- Configures the **host**, **services**, and **application setup**.
- Sets up essential components like middleware and dependency injection.
- The **host** controls the app lifecycle: startup, shutdown, config loading.

---
## Startup.cs

- Acts as the **application manager**.
- Configures **middleware**, **services**, and **request handling**.
- Defines how requests move through the middleware pipeline.
- Sets up **routing**, **security**, and other core behaviours.

---
## Controllers

- Located in the _Controllers_ folder.
- Bridge between **incoming requests** and **application logic/data**.
- Each controller handles a specific domain (e.g., products).
- Examples include: listing products, fetching details, adding, updating, deleting.

---
## Endpoints

- Defined as **methods inside controllers**.
- Each method maps to a **unique URL** for client interaction.
- Endpoints allow clients to retrieve data, send data, or trigger actions.
- Represent the “address” where requests are sent.

---
## Routing

- Determines **which controller method** handles an incoming request.
- Convention-based routing maps requests automatically by controller/action names.
- Example: /api/products → routed to the ProductsController’s GET method.
- Routing configuration lives in _Startup.cs_.

---
## Attribute Routing

- Allows **custom, explicit route definitions** in controllers.
- Example route: `[HttpGet("products/{id}")]`
- Creates clean, descriptive URLs like /api/products/5.
- Gives full control over URL structure.

---
## HTTP Methods

- **GET** → Retrieve data.
- **POST** → Create new data.
- **PUT** → Update existing data.
- **DELETE** → Remove data.

---
## Testing the API

- Run the project to test GET endpoints in the browser.
- Use external tools (e.g., Postman) to test POST, PUT, DELETE.
- Testing ensures correctness before deployment.

---
## Summary

- program.cs and startup.cs establish the **foundation** of your API.
- Controllers define **logic**, endpoints define **interactions**, routing defines **behaviour**.
- You now have the building blocks to create and test a simple ASP.NET Core Web API.
