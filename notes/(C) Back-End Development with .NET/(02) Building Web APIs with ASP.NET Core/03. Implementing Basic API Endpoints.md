## Creating a Basic Web Application

- Start a new empty ASP.NET Core project in Visual Studio Code.
- The “ASP.NET Core Empty” template provides a minimal setup ideal for learning basic APIs.
- The project generates only a few lines in _Program.cs_, 
  but behind the scenes .NET runs thousands of lines to initialise the environment.

---
## Understanding Program.cs

- Uses WebApplication.CreateBuilder() to configure the app before it runs.
- The builder creates an application object, which becomes your running web server.
- A web app simply listens for incoming requests and responds based on defined logic.

---
## URLs, Domains, and Paths

- URLs act like addresses for your server, telling clients where to send requests.
- The root path is /, similar to the building’s front lobby.
- Additional paths (e.g., /downloads) behave like separate rooms in the building.

---
## Defining API Endpoints (MapGet, MapPost, etc.)

- MapGet, MapPost, MapPut, and MapDelete define what happens 
  when a request hits a specific path using a specific HTTP method.
- Example logic: root path returns a simple text response; /downloads returns another string.
- Methods correspond to intent:
    - GET        → retrieve data
    - POST     → send/create data
    - PUT        → update data
    - DELETE  → remove data

---
## Running and Testing the Server

- Running the application starts a **development server** on localhost using a port number.
- Browsers always perform **GET** requests, 
  so other methods require tools like REST Client, Postman, or cURL.
- After changing endpoints, always restart server — otherwise new routes or logic will not load.

---
## Status Codes and Responses

- The server returns HTTP status codes, such as:
    - **200 OK** → request succeeded
    - **404 Not Found** → no matching route
    - **405 Method Not Allowed** → method exists but is not mapped for that route
- Responses include status, headers, protocol, and body.

---
## Expanding to Multiple Methods

- Each path can support multiple HTTP methods by mapping each one separately.
- ASP.NET Core allows clean endpoint definitions with minimal syntax.

---
## What an API Really Is

- The API is the part of your web application that defines available paths and how they behave.
- It provides an interface for other systems to communicate with your app programmatically.
