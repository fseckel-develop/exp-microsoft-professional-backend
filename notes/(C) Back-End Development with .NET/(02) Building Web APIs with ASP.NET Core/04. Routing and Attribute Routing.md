## What Routing Does

- Directs incoming HTTP requests to the correct handler based on URL, method, and parameters
- Ensures actions like data retrieval or form submission reach the proper function
- Works like mail addressing: every request is delivered to the correct destination

---
## How Basic Routing Works

- Defined in _Program.cs_ using methods such as 
  app.MapGet, app.MapPost, app.MapPut, app.MapDelete
- Each map-call specifies a route pattern and a handler function
- Patterns match the path portion of a URL (e.g., /users)
- Handler functions can return strings, JSON, or any response type

---
## Components of URL Matching

- Protocol: https
- Domain: localhost
- Port: e.g., 5124
- Path: e.g., /hello → the part that routing matches

---
## Route Handlers

- Are _request delegates_ that run when the route is matched
- Can be anonymous (lambda) functions for simple logic
- Can also be predefined methods when logic needs organisation or reuse

---
## Static and Dynamic Routes

- Static routes: match exact paths (e.g., /users, /admin)
- Dynamic routes: use parameters like /users/{id} to handle many URLs
- Route parameters: pass values from the path directly into the handler

---
## Optional Parameters

- Add ? to make a value optional (e.g., /users/{id?})
- Handlers can provide default values when nothing is supplied

---
## Route Constraints

- Restrict what values a parameter may accept
- Examples:
    - Type constraints: {id:int}
    - Range or length constraints
    - Full pattern matching with regular expressions
- Invalid inputs (e.g., /users/abc for an int route) are rejected

---
## Attribute Routing

- Routes are defined directly on controller actions using attributes
- Keeps routing logic close to the code being executed
- Ideal for more complex or large applications

---
## Advanced Routing Techniques

- Route constraints: validate numeric ranges, string lengths, etc.
- Catch-all routes: {*filePath} captures remaining URL segments
- Query parameters: allow flexible data input without changing the path 
  (e.g., /search?q=aspnet&page=2)
- Combined patterns: mix static segments, parameters, optional values, 
  and constraints for complex scenarios
