## Importance of Error Handling and Logging

- Essential for building stable and reliable ASP.NET Core applications
- Improves user experience by preventing crashes and showing helpful messages
- Supports troubleshooting by capturing detailed information about failures
- Enhances security, performance monitoring, and overall app health

---
## Key Error Handling Techniques

- **Try-Catch Blocks**:
	- Handle known exceptions without stopping the application
	- Allow custom responses for predictable issues (e.g., divide-by-zero errors)
	- Prevent crashes by catching and managing specific exceptions
	- Promote clearer debugging by avoiding overly generic exception handling

- **Global Exception Handling**:
	- Implemented through middleware to catch unhandled exceptions centrally
	- Ensures consistent logging and user-friendly messages across the app
	- Useful for failures from external APIs or unpredictable runtime issues
	- Helps maintain stability by preventing unhandled crashes

---
## Logging Best Practices

- **Purpose of Logging**:
	- Records application events and behaviour for diagnostics
	- Assists with performance monitoring and security auditing
	- Enables early detection of issues through real-time or historical logs

- **Configuring Log Levels**:
	- Log levels control how much detail is recorded
	- Production: minimal (errors, warnings)
	- Development: detailed (info, debug)

- **Consistent Log Formatting**:
	- Improves readability and searchability
	- Should include timestamps, log levels, identifiers, and context

- **Using Log Scopes**:
	- Groups related log entries for multi-step processes
	- Helps identify issues in complex workflows like requests or transactions
	- Provides structured context to simplify debugging

---
## Configuring Logging in ASP.NET Core

- Can be set through **appsettings.json** or **Program.cs**
- Define output targets (console, file, external services, cloud)
- Set log levels for both the app and external libraries
- Supports third-party frameworks like Serilog and NLog for richer features
- Proper setup ensures logs are accessible for monitoring and long-term diagnostics
