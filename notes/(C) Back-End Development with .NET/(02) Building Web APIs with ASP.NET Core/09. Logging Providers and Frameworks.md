## Purpose of Logging Providers

- Capture and organize essential information about application behavior
- Adapt logging to different environments (development vs. production)
- Enable effective monitoring, diagnosing, and maintaining application health
- Support system stability, debugging, and long-term analysis

---
## Built-In Logging Providers in ASP.NET Core

- **Console Provider**:
	- Sends log messages directly to the console window
	- Extremely helpful during development
	- Provides immediate feedback for quick debugging
	- Allows filtering by log severity (Error, Warning, Info, etc.)

- **Debug Provider**:
	- Sends logs to the active debugging output
	- Useful while stepping through code in an IDE
	- Helps match log messages with execution flow
	- Great for catching issues early during development

- **Event Log Provider (Windows Only)**:
	- Writes logs to the Windows Event Log
	- Ideal for production environments on Windows
	- Supports structured, secure, and system-level logging
	- Allows system administrators to search, filter, and analyze logs with built-in tools
	- Not available on Linux or macOS

---
## When to Use Built-In vs Third-Party Providers

- Built-in providers are simple, require minimal setup, and work well in development
- For production—especially cross-platform—more advanced tools are often needed
- Third-party frameworks enable structured logging, cloud integration, and better scalability

---
## Third-Party Logging Frameworks

- **Serilog**:
	- Excellent for structured, context-rich logging
	- Works well with modern log pipelines and cloud platforms
	- Ideal for apps requiring detailed insights and scalability
	- Logs properties like user IDs, transaction durations, device details, etc.
	- Enables fast searching and analytics for debugging and performance monitoring

- **NLog**:
	- High-performance, flexible, and easy to configure
	- Suitable for high-throughput applications
	- Supports many output targets (files, databases, cloud services)
	- Works well with centralised log management tools like ElasticSearch or Logstash
	- Focuses on speed and customisation rather than property-based querying

---
## Why Logging Providers Matter

- Essential for monitoring and maintaining application health
- Help identify recurring issues and performance bottlenecks    
- Improve debugging through context-rich, searchable data
- Support production stability and security
- Enable smooth operations in both development and live environments
