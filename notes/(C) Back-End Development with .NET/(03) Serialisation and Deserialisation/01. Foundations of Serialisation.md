## What Serialization Is

- Converts objects into storable or transmittable formats
- Enables data sharing across files, networks, and systems
- Maintains structure and state during transfer
 
---
## Why Serialisation Matters

- Simplifies saving and loading complex data
- Improves communication in distributed systems
- Supports multiple formats like JSON, XML, and binary
- Allows state persistence between sessions
- Ensures interoperability across platforms

---
## Core Benefits

- **Data Storage**:
	- Saves objects to files or databases
	- Supports readable (JSON/XML) or compact (binary) formats
	- Useful for settings, preferences, and save states

- **Data Transmission**:
	- Allows objects to be sent over networks
	- Standard for web APIs and client–server communication
	- Enables smooth system-to-system data exchange

- **Persistence**:
	- Retains object state across application restarts
	- Common in configuration files and user progress tracking

- **Interoperability**:
	- Uses standard formats for cross-technology compatibility
	- Facilitates data exchange between different systems

---
## Serialisation Formats

- **Binary Serialisation**:
	- Compact, high-performance format
	- Not human readable
	- Best for internal or performance-critical scenarios

- **XML Serialisation**:
	- Human-readable, structured, and verbose
	- Common in enterprise and legacy systems
	- Ideal for complex data models

- **JSON Serialisation**:
	- Lightweight and widely supported
	- Standard for modern APIs
	- Easy to read and efficient

---
## **Practical Use Cases**

- **Data Persistence**:
	- Stores user settings, game progress, and application configurations
	- JSON/XML for readability, binary for efficiency

- **Network Communication**:
	- Sends serialised objects between client and server
	- Used in web APIs (e.g., sending order details as JSON)

- **Caching**:
	- Stores frequently accessed data for fast retrieval
	- Reduces expensive operations like database queries
	- Ideal for performance-heavy systems such as reporting tools

---
## Summary

- Serialisation enables efficient storage, transmission, and sharing of object data
- JSON, XML, and binary each offer different advantages depending on 
  readability, performance, and complexity
- Common use cases include saving data, communicating across networks, 
  and caching frequently used information
