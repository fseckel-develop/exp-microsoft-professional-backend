## What Deserialisation Is

- Deserialisation is the reverse of serialisation, converting stored
  or transmitted data back into usable .NET objects.
- Applications rely on deserialisation to restore saved information 
  such as user preferences, profiles, or app settings.
- When an app saves data, it serialises it into a storable format; when reopened, deserialisation reconstructs that data so the app can use it again.

---
## How Deserialisation Works in .NET

- .NET deserialisation restores previously saved objects, 
  such as user profiles when logging back into an application.
- It converts external data — like JSON received from a web API — 
  into objects your application can work with.
- Deserialisation is essential for processing information retrieved from files, 
  network responses, or configuration systems.

---
## Common Data Formats

- **JSON**:
	- A simple, text-based format widely used for web communication.
	- Automatically maps JSON fields to object properties if they match.
	- Ideal for exchanging lightweight data like user details and API responses.

- **XML**:
	- A structured format using tags, suitable for complex or hierarchical data.
	- Often used for configuration settings or detailed structured information.
	- Requires reading data streams and converting them using an XmlSerializer.

- **Binary**:
	- A compact, efficient format used for large datasets or performance-critical scenarios.
	- Faster to process but requires careful handling for security.

---
## Real-World Use Example

- A web app saves user preferences in JSON, XML, or binary when closed.
- When reopened, the saved data is deserialised back into meaningful settings.
- This allows the app to restore user states, preferences, and previously used data.

---
## Implementation Techniques in .NET

- **JSON Deserialisation**:
	- .NET provides global JSON options to control naming conventions and behavior.
	- Custom route-level options can override defaults for specific endpoints.
	- Supports handling unmapped or unexpected fields to reduce errors.

- **XML Deserialisation**:
	- Requires reading and parsing the incoming data stream.
	- Converts XML into objects using XmlSerializer.
	- Useful when JSON isn’t the right fit, especially for structured configurations.

---
## Key Considerations

- **Data Validation & Error Handling**:
	- Handles missing or invalid data through defaults or error responses.
	- Ensures that incoming data aligns with expected object types and structure.

- **Security**:
	- External data must be validated to avoid harmful or unexpected objects.
	- .NET safe deserialisation practices to protect applications from unsafe data inputs.
