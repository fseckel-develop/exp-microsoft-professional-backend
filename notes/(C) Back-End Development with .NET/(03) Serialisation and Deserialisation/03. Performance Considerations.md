## Impact of Serialisation on Performance

- Serialisation converts objects into storable or transferable formats but can affect overall application performance.
- CPU usage increases with complex or deeply nested objects that require more processing.
- Memory usage rises when serialising large or detailed objects, such as user profiles with multiple nested elements.
- I/O operations introduce latency when writing or sending serialised data, especially large files or when using slow storage or networks.
- Network transmission can further slow performance due to bandwidth limitations or unstable connections.

---
## Influence of Data Formats

- **Binary**:
    - Fast, compact, and efficient.
    - Not human-readable and harder to debug.

- **XML**:
    - Human-readable and structured.
    - Slower performance and larger file sizes.

- **JSON**:
    - Balanced between readability and speed.
    - Faster than XML and easier to inspect than binary.

---
## Complexity & Data Size

- More complex objects take longer to serialise due to nested layers and additional attributes.
- Larger datasets significantly increase serialisation and deserialisation time.
- Example: Serialising 10,000 users takes far longer than 100 users, even if objects are simple.

---
## Serialisation Settings

- Settings like ignoring null values or enabling compression can improve performance by shrinking output.
- Pretty-printing enhances readability but increases file size and slows down serialisation.
- Extra metadata or added formatting can reduce efficiency depending on use case.

---
## Network Latency

- Serialised data transmitted over networks is impacted by network speed and stability.
- Larger and more complex serialised files take longer to send, especially on slow connections.

---
## Optimisation Techniques

- **Choosing the Right Format**:
	- Select binary or JSON for faster serialisation depending on readability/debugging needs.
	- JSON is ideal for APIs and web applications; 
	  binary is better for compact and high-performance scenarios.

- **Simplifying Data Structures**:
	- Remove unnecessary nesting and only serialise essential data fields.
	- Example: Serialise only a customer’s name and recent purchases instead of full profile.

- **Using Efficient Libraries**:
	- Prefer **System.Text.Json** for high-performance JSON operations in .NET Core.
	- Avoid slower legacy libraries unless specific features are needed.

- **Minimising Serialisation Scope**:
	- Serialise only what your application truly needs.
	- Create smaller DTOs (data transfer objects) when only partial data is required.
