## Security Risks in Serialization

- Serialisation enables data storage and transfer but can introduce vulnerabilities 
  if not handled securely.
- Deserialisation attacks occur when attackers send harmful or unexpected data 
  that executes malicious code.
- Data tampering happens when serialised data is intercepted and altered during 
  transmission, compromising integrity.
- Sensitive information exposure occurs when confidential data is serialised, 
  logged, or transmitted insecurely.
- Real-world failures, such as the 2015 Java deserialisation vulnerability, 
  show how widespread and severe these risks can be.

---
## Deserialisation Attacks

- Attackers exploit weak validation by sending manipulated data into deserialisation pipelines.
- Malicious payloads can trigger unauthorised behaviour, including remote code execution.
- Example scenario: 
  A system accepting user-uploaded files without validation risks executing harmful content.

---
## Data Tampering

- Serialised data sent across networks can be intercepted if not protected.
- Attackers may modify packets to change values, such as altering transaction amounts.
- Lack of encryption or integrity checks increases vulnerability to corruption and fraud.

---
## Sensitive Data Exposure

- Including passwords, tokens, or personal details in serialised objects can unintentionally 
  leak sensitive information.
- Storing serialised data in logs, caches, or unsecured channels exposes it to attackers.
- Poor handling can lead to identity theft, data breaches, and unauthorized access.

---
## Best Practices for Secure Serialisation

- **Input Validation & Sanitisation**:
	- Validate serialised data before deserialisation to ensure expected formats (JSON etc.).
	- Sanitise incoming data to remove harmful or unexpected elements.
	- Reduces risk of deserialisation attacks and code execution exploits.

- **Use Secure & Updated Libraries**:
	- Choose serialisation libraries with strong security features.
	- Keep libraries patched to avoid known vulnerabilities.
	- Prevents exploitation of outdated components.

- **Avoid Untrusted Data**:
	- Never deserialise data received from anonymous or unauthenticated sources.
	- Treat external input as hostile unless proven otherwise.
	- Minimises exposure to malicious payloads.

- **Access Control Measures**:
	- Use role-based access control (RBAC) to limit who can view or modify serialized data.
	- Ensures only authorised users can interact with sensitive information.

- **Encryption & Integrity Checks**
	- Encrypt sensitive data before serialisation to protect confidentiality.
	- Use hashing or digital signatures to detect tampering during transmission.
	- Ensures data remains authentic and unreadable without proper keys.
