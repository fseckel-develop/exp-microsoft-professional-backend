## What Middleware Is

- Sits between front-end and back-end, helping systems communicate and manage data.
- It processes HTTP requests and responses as they move through the application.
- Each middleware component handles one task, then passes the request along to the next.
- Middleware creates a structured, modular approach to managing web application behaviour.

---
## HTTP Requests and Why They Matter

- HTTP requests are messages sent from clients to servers requesting data or actions.
- Middleware controls how these requests are handled in ASP.NET Core.
- A request moves through a pipeline of middleware until a final response is created.

---
## The Middleware Pipeline

- The pipeline works like a conveyor belt — 
  each middleware performs a task and passes the request forward.
- Example tasks include logging, authentication, routing, data retrieval, and error handling.
- The request flows step by step:
    1. Incoming request enters the pipeline.
    2. Logging records activity.
    3. Authentication checks identity.
    4. Application logic processes data.
    5. A response is created and returned to the client.
- The order of middleware is critical for security and functionality.

---
## Example of Request Flow

- A user loads a webpage → request enters the pipeline.
- Logging middleware tracks who accessed the page.
- Authentication middleware checks whether the user is signed in.
- Application logic retrieves needed data (e.g., product details).
- Response is generated and sent back to display the page.

---
## Built-In Middleware Components

- **UseAuthentication**:
    - Verifies user credentials before granting access to protected areas.
    - Prevents unauthorised users from seeing sensitive data.

- **UseRouting**:
    - Maps incoming requests to the correct controller or endpoint.
    - Works like a GPS, guiding the request to the appropriate destination.

- Additional middleware includes static file handling, error handling, authorisation, and more.

---
## Real-World Scenario (Groove Echo Example)

- User requests a product page → request enters the pipeline.
- Logging middleware records the visit.
- Authentication middleware checks login status.
- Application logic retrieves product data from the database.
- Response is returned showing the product page to the user.

---
## Core Benefits

- Keeps applications modular and easier to maintain.
- Enhances security by enforcing authentication early in the pipeline.
- Improves performance through structured request handling.
- Allows developers to customise and extend the request-processing flow.
