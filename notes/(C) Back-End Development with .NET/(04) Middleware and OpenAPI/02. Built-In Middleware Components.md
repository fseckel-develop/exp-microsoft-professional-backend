## Exception Handling Middleware

- Detects and manages errors during application execution.
- Redirects users to a friendly error page instead of exposing technical details.
- UseExceptionHandler is used in production; 
  UseDeveloperExceptionPage is used in development.
- Environment checks (env.IsDevelopment) determine which behaviour to apply.

---
## Authentication Middleware

- Verifies the identity of a user before granting access to protected areas.
- Added to the pipeline using app.UseAuthentication().
- Ensures only authenticated users can reach secured parts of the app.

---
## Authorisation Middleware

- Determines what authenticated users are allowed to do.
- Implemented with app.UseAuthorization().
- Enforces defined permission rules and prevents unauthorised access.

---
## Logging Middleware

- Records request details, errors, and user activities for monitoring and troubleshooting.
- Enabled via app.UseHttpLogging().
- Requires setup with builder.Services.AddHttpLogging() and configuration in _appsettings.json_.
- Displays HTTP logs in the console for every incoming request.

## Role of Startup.cs

- Older ASP.NET Core versions (2.x–5.x) required a **Startup.cs** file.
- Central file where middleware is registered and configured.
- Controls request flow, available services, and overall application behaviour.
- Many conceptual lessons still teach _Startup.cs_ because:
    - It explains middleware structure clearly.
    - Many real-world enterprise projects still use it.
- The **theory remains the same**, but the **location of the code changed**.

#### BUT: New Project Structure of ASP.NET Core

- In **ASP.NET Core 6 and later**, Microsoft introduced new style called **minimal hosting model**.
- This new model **removes Startup.cs** completely.
- Everything — services, middleware, routing — is now placed in **Program.cs**.

---
## Overall Purpose

- Built-in middleware components simplify handling critical tasks like error management, security, and monitoring.
- Together they help create secure, robust, and maintainable ASP.NET Core applications.
