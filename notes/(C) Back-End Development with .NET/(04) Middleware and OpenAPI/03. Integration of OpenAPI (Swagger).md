## Purpose of OpenAPI

- Provides a standardised specification for describing APIs
- Acts as a clear blueprint outlining endpoints, request formats, and responses
- Ensures consistency so all developers understand how the API works
- Improves communication across teams by using a shared API contract

---
## Role of Swagger

- Toolset built around the OpenAPI specification
- Converts the API blueprint into an interactive, browser-based interface
- Allows developers to explore, test, and send requests without extra tools
- Reduces development and debugging time by enabling instant feedback

---
## Key Benefits of OpenAPI and Swagger

- Standardised documentation prevents confusion and misinterpretation
- Interactive testing directly from the browser speeds up development
- Enhances collaboration between front-end, back-end, and QA teams
- Automatically updates documentation when APIs change, reducing maintenance effort

---
## OpenAPI, Swagger, and Swashbuckle Relationship

- OpenAPI is the specification (the standard format)
- Swagger is the tooling that implements and visualizes the specification
- Swashbuckle is the .NET library that integrates Swagger into ASP.NET Core projects

---
## Integration Steps in ASP.NET Core

- Install required packages to enable OpenAPI and Swagger support
	- using Microsoft.AspNetCore.OpenApi;
	- using Swashbuckle.AspNetCore;
- Register API endpoint exploration and Swagger generation services
- Enable Swagger middleware in development to expose documentation and UI
- Use annotations and configuration methods to enrich endpoint descriptions

---
## Enhancing and Using Swagger UI

- Add summaries and response definitions to improve clarity
- Test API endpoints directly through the Swagger UI interface
- Hide internal or non-public endpoints to keep documentation focused

---
## Summary

- OpenAPI defines how an API is described and understood
- Swagger turns that definition into an interactive, testable experience
- Together, they simplify documentation, testing, collaboration, and long-term API maintenance
