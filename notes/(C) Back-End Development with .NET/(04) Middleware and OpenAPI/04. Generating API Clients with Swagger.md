## What an API Client or SDK is

- Generated client code is often called an SDK (Software Development Kit)
- It provides ready-made, strongly typed methods for calling API endpoints
- Front-end developers can focus on application logic instead of low-level HTTP handling
- Strong typing and IntelliSense reduce errors and improve productivity

---
## Steps to Generate an API Client

- Retrieve the Swagger (OpenAPI) JSON from a running API server
- Create or configure a client generator to parse the Swagger document
- Generate client code using NSwag (CLI, desktop app, or C# libraries)
- Add the generated client to your project like a normal dependency

---
## Creating and Configuring a Custom Client

- Configure client generation settings such as class name and namespace
- Set deserialisation options (e.g., case-insensitive JSON handling) for compatibility
- Customise output to match the structure and conventions of your API

---
## Automating and Maintaining the Client Code

- Use a client generator class (e.g., SwaggerClientGenerator) to automate generation
- Regenerate the client whenever the API changes to stay in sync
- Generated clients handle requests, responses, error handling, and deserialisation automatically

---
## Using the Generated Client

- Instantiate the generated client with a base URL and HTTP client
- Call methods like, for example BlogsAllAsync() or BlogsPOSTAsync(), directly
- No need to manually build HTTP requests or parse JSON responses
- Client methods already know required parameters and expected response types

---
## Key Benefits

- Saves development time
- Reduces bugs and inconsistencies
- Ensures client and server agree on data contracts
- Improves collaboration between front-end and back-end teams

---
## Summary

- OpenAPI defines how an API works
- Swagger exposes and visualizes that definition
- NSwag converts it into type-safe, usable client code
- Together, they make API consumption faster, safer, and easier
