
- **Why Copilot Is Useful for Debugging**:
	- Helps identify and fix bugs faster than manual searching
    - Explains errors and unexpected behaviour in simple terms
    - Reduces reliance on traditional Q&A sites
    - Works well when given clear context and error messages

- **Providing Context to Copilot**:
	- Paste the relevant code, not just one line
	- Include compiler errors, warnings, or stack traces
	- Describe what you expected vs. what actually happened
	- More context leads to more accurate fixes

- **Fixing Common Code Bugs**:
	- Missing break statements in switch cases
	- Index out of range errors caused by incorrect loop bounds
	- Incorrect conditional operators ( = instead of == )
	- Integer division issues when a decimal result is expected

- **Using Editor Clues Before Copilot**:
	- Red squiggles indicate errors that must be fixed
	- Yellow squiggles indicate warnings or potential issues
	- Hover messages often explain the problem clearly
	- Fix simple issues directly before asking Copilot

- **Debugging Workflow with Copilot**:
	- Start by pasting all related code and ask for a fix
	- Narrow down to specific sections if needed
	- Add error messages or stack traces for clarity
	- Review AI changes carefully to avoid unwanted edits

- **Finding Hidden or Advanced Bugs**:
	- Detects logic issues that still compile and run
	- Identifies concurrency and race condition problems
	- Suggests safer data structures like ConcurrentDictionary
	- Highlights design flaws, not just syntax errors

- **Debugging APIs with Copilot**:
	- Finds index validation issues in GET-by-ID routes
	- Detects unsafe list access in CRUD operations
	- Suggests better data storage patterns
	- Improves robustness without changing API contracts

- **Using Screenshots for Debugging**:
	- Screenshots can include code and stack traces
	- Copilot can extract and understand text from images
	- Useful when errors cannot be copied directly
	- Still effective without pasting raw code

- **Best Practices When Using Copilot**:
	- Start with minimal input, add detail only if needed
	- Avoid biasing Copilot toward the wrong code section
	- Always review and test suggested fixes
	- Treat Copilot as a powerful assistant, not an authority
