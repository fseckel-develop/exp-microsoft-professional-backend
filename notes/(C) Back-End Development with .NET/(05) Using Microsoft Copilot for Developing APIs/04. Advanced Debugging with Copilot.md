
- **When Copilot Can’t Solve a Bug**:
	- Some bugs are complex, involve hidden state, or visual issues
	- Sometimes external data or code causes the problem
	- You may run out of context to provide Copilot

- **Using Copilot as a Debugging Guide**:
	- Treat it like a tech support assistant
	- Ask it to **walk through the code behaviour** step by step
	- Request explanations of expected vs. actual behaviour
	- Use it to **decompose the code** logically

- **Debugging Techniques with Copilot**
	1. Add logging statements at key points
	2. Use breakpoints and the debugger
	3. Run unit tests to isolate issues
	4. Check your data and API responses
	5. Inspect headers or other unexpected outputs

- **Refactoring to Aid Debugging**:
	- Ask Copilot to refactor code for clarity and modularity
	- Modular code is easier to understand, maintain, and test
	- Refactoring can reveal hidden bugs or design flaws

- **Unit Tests and Edge Cases**
	- Copilot can generate unit tests for your code
	- Helps identify missing test cases or edge conditions
	- Review tests carefully to avoid introducing faulty tests

- **Advanced Copilot-Assisted Workflow**
	- Pass your full code and error messages for context
	- Highlight suspected sections or provide stack traces
	- Ask Copilot to walk through debugging steps in your editor
	- Include details about your environment (VS Code, extensions, profilers)

- **Other Tips**
	- Use “rubber ducking” with Copilot: explain the problem and logic out loud
	- Ask Copilot to suggest which lines to watch, breakpoints to set, and variables to inspect
	- If all else fails, let Copilot guide you step by step until you isolate the issue
