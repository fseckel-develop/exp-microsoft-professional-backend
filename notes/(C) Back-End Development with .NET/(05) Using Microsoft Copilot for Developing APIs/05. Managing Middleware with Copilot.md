
- **Why Use Copilot for Middleware**:
	- Speeds up problem-solving beyond current knowledge
	- Helps learn APIs and coding patterns faster
	- Generates functional code from specifications or problems
	- Reduces manual research and trial-and-error

- **Creating Path Counting Middleware**:
	- Tracks how often each API endpoint is accessed
	- Copilot generates middleware class, request delegate, and dictionary for path → count
	- Increments existing counts or adds new paths
	- Optionally adds counts to response headers

- **Testing Path Counting Middleware**:
	- Routes: /api/route1 and /api/route2
	- Headers confirm correct incrementing of request counts
	- Requests tracked across multiple calls

- **Extending Middleware with a Service**:
	- Service manages request counts externally
	- Methods: IncrementCount(path), GetCount(path), GetAllCounts()
	- Middleware updated to use the service
	- Route /api/counts exposes aggregated counts

- **Managing Middleware Pipeline**:
	- Problem: Handling traffic spikes causing performance issues
	- Copilot suggests: rate-limiting, caching, queues, load balancing
	- Rate-limiting chosen for simplicity
	- Explains fixed window, sliding window, token bucket, concurrency limits

- **Implementing Rate-Limiting Middleware**:
	- Built-in ASP.NET Core middleware via AddRateLimiter
	- Fixed window approach for simplicity
	- Configured permit limits and time windows
	- Verified behaviour: 503 error when limit exceeded
	- Thresholds adjustable for practical traffic

- **Combining Middleware with CRUD API**:
	- Integrated path counting and rate-limiting into CRUD API
	- Registered services in builder and confirmed middleware order
	- Verified functionality for all endpoints

- **Testing and Validation**:
	- /api/counts confirms accurate request counts
	- Rate-limiting blocks excessive requests with 503 errors
	- Adjusted thresholds for realistic use
	- Middleware works across multiple routes

- **Best Practices with Copilot**:
	- Start with clear specification or problem description
	- Review generated code carefully
	- Test middleware individually before integration
	- Use services for shared state
	- Explore additional middleware for extended functionality
