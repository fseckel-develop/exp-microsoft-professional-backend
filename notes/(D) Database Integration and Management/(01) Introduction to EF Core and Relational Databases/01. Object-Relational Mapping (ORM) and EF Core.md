## The Problem with Manual Data Management

- Managing data manually requires writing repetitive and complex SQL code.
- Developers must handle storing, updating, and retrieving data themselves.
- As applications scale, this approach becomes inefficient, error-prone, and hard to maintain.

---
## Relational Databases and SQL

- Relational databases store data in **tables** made up of:
	- **Rows** (records)
	- **Columns** (fields)
- Tables are connected through identifiers (keys).
- **SQL (Structured Query Language)** is used to:
	- Insert, retrieve, update, and delete data
	- Communicate with relational databases
- Example:
	- A `Books` table with columns like `Title`, `Author`, and `PublicationYear`
	- Each row represents a single book

---
## Object-Oriented Programming and the Data Mismatch

- In object-oriented languages like **C#**, data is represented as:
	- **Classes** and **objects**
- Example:
	- A `Book` class with properties such as `Title`, `Author`, and `PublicationYear`
- Problem:
	- Databases store data as tables
	- Code represents data as objects
- Mismatch requires manual translation using SQL, increasing complexity and risk of errors

---
## What Is Object-Relational Mapping (ORM)?

- ORM is a programming technique that:
	- Maps objects in code to records in relational database tables
- Benefits:
	- Eliminates the need to write raw SQL
	- Allows developers to interact with databases using objects
	- Automatically handles data conversion between code and database structures
- Acts as a **bridge** between object-oriented code and relational databases

---
## Entity Framework Core (EF Core)

- EF Core is:
	- An open-source ORM in the **.NET ecosystem**
- Purpose:
	- Enables interaction with relational databases using **C# objects**
	- Automatically generates SQL behind the scenes
- Example:
	- Creating a `Book` object in C# automatically saves it to the database
	- Retrieving data returns objects instead of raw query results

---
## Key Features of EF Core

- **LINQ (Language Integrated Query)**:
	- Allows writing database queries directly in C#
	- Queries are:
		- Type-safe
		- Readable
		- Integrated into application logic

- **Database Migrations**:
	- Enables incremental changes to the database schema
	- Supports:
		- Adding tables
		- Modifying columns
		- Evolving the database as the application grows

- **Change Tracking**:
	- Automatically tracks modifications made to objects
	- Simplifies database updates by detecting what has changed

---
## Advantages of Using EF Core

- **Ease of Use**
	- No need to write complex SQL queries
	- Work directly with C# objects

- **Maintainability**
	- EF Core generates SQL automatically
	- Reduces errors caused by manual SQL

- **Flexibility**
	- Supports multiple relational databases:
	    - SQL Server
	    - PostgreSQL
	    - SQLite

- **Scalability**
	- Simplifies data access as applications grow

---
## Key Takeaway

- ORMs solve the mismatch between object-oriented code and relational databases.
- EF Core provides a powerful, flexible, and maintainable ORM solution for .NET developers.
- Using EF Core reduces complexity, saves time, and improves long-term code quality.
