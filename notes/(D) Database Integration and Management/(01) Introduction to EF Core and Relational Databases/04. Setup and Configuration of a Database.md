
- Setting up a relational database involves transforming raw data into a well-structured, functional system.
- The process is divided into three main phases:
	1. Installing the DBMS
	2. Configuring the system
	3. Creating the actual database

---
## Phase 1: Installing the DBMS:

- Download the DBMS software from the provider's website.
- Select the version compatible with your operating system (Windows, macOS, or Linux).
- Run the installation file and follow prompts
- Choose installation directories and components 
  (default options are usually safe for beginners).
- Verify that the DBMS service is running after installation.

---
## Phase 2: Configuring the System:

- **User Accounts**:
	- Create an admin/root account with a strong password.
	- Optionally, create additional user accounts for staff or managers.

- **Roles and Permissions**:
	- Assign roles to define who can access the database and what actions they can perform.
	- Example: A manager can view project data but not financial records.

- **Network Security**:
	- Restrict database access to the internal network or specific servers.
	- Ensure only authorised users can connect.

- **Storage Options**:
	- Configure directories or drives where database files will be stored.
	- Set storage allocation according to expected database growth.

---
## Phase 3: Creating the Database:

- **Database Naming**:
	- Use clear, descriptive names without spaces or special characters.
	- Example: `CompanyDB`

- **Schema Design**:
	- Define a schema that maps out tables, relationships, and database elements.
	- Example tables for a company database:
		- Employees
		- Departments
		- Projects
	- Map relationships:
		- Employees belong to departments.
		- Departments manage projects.
		- Employees can work on multiple projects.

- **Table Structures**:
	- Define rows and columns for each table.
	- Example:
		- **Employees**: EmployeeID, FirstName, DepartmentID
		- **Departments**: DepartmentID, DepartmentName, CostCenter
		- **Projects**: ProjectID, ProjectName, DepartmentID

---
## Key Takeaways:

- A complete relational database setup includes:
	- Installing and verifying the DBMS
	- Configuring users, roles, network security, and storage
	- Creating a well-structured database with clear naming conventions
	- Designing schemas and tables with mapped relationships
- Proper setup ensures efficient data retrieval, management, and organization.
- A structured database forms the foundation for effective data-driven projects.
