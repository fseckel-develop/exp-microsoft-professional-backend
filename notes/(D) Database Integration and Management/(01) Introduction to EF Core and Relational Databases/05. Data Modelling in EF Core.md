## Overview

- Data modelling is the process of designing how data is structured, stored, and related within a database.
- In EF Core, data modelling ensures efficient storage, accurate relationships, and seamless interaction between application logic and the database.
- Entity classes and DbContext work together to map C# objects to relational database tables.

---
## What Is Data Modelling

- Data modelling creates a visual and logical blueprint of a database system.
- It defines:
    - Entities (tables)
    - Attributes (columns)
    - Relationships between data
- Example: Library system with tables for books, authors, and borrowers.

---
## Entities in EF Core

- An entity represents a real-world object or concept.
- In EF Core:
    - Each entity is a C# class.
    - Each property maps to a database column.
- Example:
    - A Book class with properties like Id and Title.

---
## Creating Entity Classes

- **Create Models Folder**
    - Add a folder named Models to store all entity classes.

- **Define Entity Classes**
    - Create one C# file per entity (e.g., Employee.cs).
    - Declare each entity as a public class.

- **Add Properties**
    - Properties represent table columns.
    - Example:
        - EmployeeID (Primary Key)
        - FirstName
        - LastName
        - HireDate

---
## EF Core Conventions

- EF Core automatically applies rules based on naming conventions.
- Examples:
    - EmployeeID is automatically recognised as a primary key.
    - Properties map automatically to table columns.
- Conventions reduce the need for manual configuration.

---
## Establishing Relationships

- Relationships are defined using:
    - Foreign key properties (e.g., DepartmentID)
    - Navigation properties (e.g., Department)

- Example:
    - One department has many employees.
    - Each employee belongs to one department.

- Navigation properties allow related data to be retrieved easily.

---
## Using Data Annotations

- Data Annotations are attributes applied directly to entity properties.
- Common annotations include:
    - **[Key]** — Explicitly defines a primary key.
    - **[Required]** — Makes a field mandatory.
    - **[StringLength(50)]** — Limits string length.
- These annotations help enforce data integrity with minimal code.

---
## Using Fluent API

- Fluent API provides advanced configuration through code.
- Configurations are defined in the OnModelCreating method of the DbContext.
- Allows detailed control over:
    - Field requirements
    - String lengths
    - Relationships
- Example methods:
    - .IsRequired()
    - .HasMaxLength(50)

---
## Role of DbContext

- DbContext acts as the bridge between the application and the database.
- Responsibilities include:
    - Managing database connections
    - Tracking entity changes
    - Executing CRUD operations
- DbSet properties map entities to database tables.

---
## Summary

- Data modelling in EF Core uses entities, relationships, and configuration rules to reflect real-world systems.
- Conventions simplify setup, while Data Annotations and Fluent API provide control.
- A well-designed model improves performance, maintainability, and scalability.
- Strong data modelling is the foundation of effective database-driven applications.
