## Overview of CRUD Operations

- CRUD stands for **Create, Read, Update, and Delete**
- These operations form the foundation of all database interactions
- EF Core provides built-in support for CRUD through **DbContext**
- Mastering CRUD enables effective data management in any application

---
## Role of DbContext in CRUD

- **DbContext** acts as the bridge between the application and the database
- It tracks entity changes and translates them into database operations
- Two core components are essential for CRUD:
	- **DbSet**
	- **SaveChanges**

---
## DbSet

- Represents a table in the database
- Each entity class maps to one DbSet
- Acts like a collection that supports querying and data manipulation
- Example:
	- `DbSet<Employee>` represents the **Employees** table
- Enables:
	- Adding new entities
	- Querying existing entities
	- Updating entities
	- Deleting entities

---
## SaveChanges

- Commits all tracked changes to the database
- Required for **Create, Update, and Delete** operations
- Without calling `SaveChanges`, no changes are persisted
- Ensures database state matches the application state

---
## Create Operation

- Used to add new records to the database
- Steps:
	- Create a new instance of an entity
	- Populate its properties
	- Add it to the DbSet using `Add`
	- Call `SaveChanges`
- EF Core inserts the new record into the database

---
## Read Operation

- Used to retrieve existing data from the database
- Common methods include:
	- `Find` – retrieves a record by primary key
	- `FirstOrDefault` – retrieves the first matching record
	- `ToList` – retrieves multiple records
- Supports querying and data processing

---
## Update Operation

- Used to modify existing records
- Steps:
	- Retrieve the entity using `Find` or `ToList`
	- Modify the entity’s properties
	- Call `SaveChanges`
- EF Core tracks changes automatically and updates the database

---
## Delete Operation

- Used to remove records from the database
- Steps:
	- Retrieve the entity using `Find` or `ToList`
	- Pass it to the `Remove` method
	- Call `SaveChanges`
- EF Core marks the entity for deletion and removes it when saved

---
## Key Takeaways

- CRUD operations are essential for all data-driven applications
- DbContext and DbSet simplify database interaction
- SaveChanges is required to persist modifications
- EF Core handles change tracking and SQL generation automatically
- Understanding CRUD unlocks full control over application data
