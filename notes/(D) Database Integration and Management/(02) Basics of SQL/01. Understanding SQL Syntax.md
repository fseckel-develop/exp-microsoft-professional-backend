## What Is SQL?

- Structured Query Language (SQL) is a programming language used to communicate with relational databases. 
- It allows you to retrieve, insert, update, and delete data stored in tables. 
- Even when using tools like Entity Framework Core, understanding SQL is essential for direct database management and advanced scenarios.

---
## What Is a Query?

- A query is a request sent to a database to perform an action on data. 
- Includes retrieving records, inserting new data, updating existing values, or deleting records. 
- For example, a query might retrieve book titles from a library database or filter books published after a certain year.

---
## Structure of an SQL Query

SQL queries are built using three main components:

- **Keywords**: 
  Reserved words that define actions, such as SELECT, INSERT, UPDATE, and DELETE.

- **Clauses**: 
  Parts of a query that specify conditions or data sources, such as FROM and WHERE.

- **Expressions**: 
  Combinations of values, operators, functions that evaluate conditions, used in filtering logic.

---
## Common SQL Commands

These are the most frequently used SQL commands:

- **SELECT**: Retrieves data from a table.
- **INSERT INTO**: Adds new records to a table.
- **UPDATE**: Modifies existing records.
- **DELETE**: Removes records from a table.
- **WHERE**: Filters records based on conditions.

---
## Example SQL Queries

Here are examples demonstrating how SQL commands work:

Retrieve all book titles:

```sql
SELECT bookTitle FROM books;
```

Retrieve books published after 2015:

```sql
SELECT bookTitle FROM books 
	WHERE publicationYear > 2015;
```

Insert a new book:

```sql
INSERT INTO books (bookTitle, author, publicationYear) 
	VALUES ('New Book', 'Author', 2021);
```

Update a book’s publication year:

```sql
UPDATE books 
	SET publicationYear = 2022 
	WHERE bookTitle = 'Old Book';
```

Delete a book:

```sql
DELETE FROM books 
	WHERE bookTitle = 'Obsolete Book';
```

---
## Why SQL Knowledge Matters

Understanding SQL helps you:

- Work more effectively with relational databases
- Debug and optimise data access
- Handle complex data scenarios beyond ORM abstractions

Even when frameworks simplify database interactions, SQL remains a foundational skill for efficient and reliable data management.
