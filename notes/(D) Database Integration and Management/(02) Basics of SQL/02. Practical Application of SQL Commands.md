### Purpose of This Note

- Introduces **advanced patterns** commonly used in professional database work

---
### Using SELECT in Real Scenarios

- Retrieve **all records** or **specific columns** depending on needs
- Filter results with WHERE to focus on relevant data
- Sort results using ORDER BY to make output meaningful
- Limit results with SELECT TOP (or LIMIT) when only a subset is needed
    - Useful for dashboards, summaries, or performance optimization
- Combine data from multiple tables using JOIN
    - Enables meaningful insights across related tables

---
### INSERT INTO Beyond Single Records

- Add **single rows** by specifying column values
- Insert **multiple rows at once** to improve efficiency
- Copy data between tables using INSERT INTO ... SELECT
- Common for:
	- Archiving data
	- Migrating records
	- Creating reporting tables

---
### Advanced UPDATE Operations

- Modify **single or multiple records** using conditions
- Use subqueries to update data based on values from another table
- Example use cases:
	- Updating employee departments from a departments table
	- Adjusting salaries based on role or performance data
- Ensures consistency when data dependencies exist across tables

---
### Advanced DELETE Operations

- Remove specific records using WHERE
- Delete multiple records that meet a condition
- Use subqueries to delete records linked to another table
- Important distinction:
    - DELETE removes data but **keeps the table structure**
    - DROP removes the **entire table and schema**
- DELETE operations should be used carefully, especially without conditions

---
### JOINs for Working with Related Data

- JOINs allow combining data from multiple tables into one result set
- Common join types:
    - INNER JOIN for matching records
    - LEFT JOIN to include unmatched records from one table
- Essential for normalised databases where data is spread across tables
- Enables reporting, analytics, and complex queries
