## Inserting Data with INSERT Statements

- **Usage:** Add new data into a database, either as individual records or batch inserts.

- **Basic Syntax:**

```sql
INSERT INTO tableName (columnA, columnB) 
	VALUES (‘valueA’, ‘valueB’);

-- Specify the table and columns.
-- Provide values in the same order as columns.
```

- **Single Record Example:**

```sql
INSERT INTO Customer (FirstName, LastName) 
	VALUES (‘John’, ‘Doe’);
```

- **Multiple Records Example:**

```sql
INSERT INTO Customer (FirstName, LastName) 
	VALUES (‘John’, ‘Doe’), (‘Jane’, ‘Smith’);
```

- **Foreign Key Constraints:**
    - Ensure referenced data exists in related tables.
    - Example: Inserting an order requires the CustomerID to exist in the Customer table.

```sql
INSERT INTO Orders (OrderID, CustomerID, Product, Quantity) 
    VALUES (102, 1, ‘Smartphone’, 3);
```

## Updating Data with UPDATE Statements

- **Usage:** Modify existing records in a table.

- **Basic Syntax:**

```sql
UPDATE tableName 
	SET columnA = ‘newValue’ 
	WHERE condition;

-- Important: Without a WHERE clause, all rows will be updated.
```

- **Single Record Update Example:**

```sql
UPDATE MusicCatalog 
	SET Genre = ‘Rock’ 
    WHERE songTitle = ‘Hey Jude’;
```

- **Multiple Records Update Example:**

```sql
UPDATE MusicCatalog 
	SET popularity = ‘Viral’ 
    WHERE weeklyListeners > 1000000;
```


---
## Deleting Data with DELETE Statements

- **Usage:** Remove records from a table while maintaining table structure.

- **Basic Syntax:**

```sql
DELETE FROM tableName 
	WHERE condition;
 
-- Important: Without a WHERE clause, all rows will be deleted.
```

- **Single Record Delete Example:**

```sql
DELETE FROM MusicCatalog 
    WHERE songTitle = ‘Obsolete Song’;
```

- **Delete All Records Example:**

```sql
DELETE FROM tableName;

-- Leaves the table structure intact.
```


---
## Best Practices for Data Manipulation

- **Always use WHERE clauses** for UPDATE and DELETE to avoid unintended changes.
- **Verify with SELECT first:** 
  Run a SELECT query with the same WHERE condition before executing updates or deletes.
- **Backup Before Modifying:** 
  Always back up your database before bulk modifications or deletions.
