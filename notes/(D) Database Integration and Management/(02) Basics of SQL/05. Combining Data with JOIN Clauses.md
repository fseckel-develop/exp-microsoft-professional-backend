### Purpose of JOIN Clauses

- **Usage:** Combine rows from two or more tables based on a related column.

- **Why it’s needed:** Data is often split across multiple tables to reduce redundancy and keep databases organised.

- **Key Concept:** Primary and foreign keys define relationships between tables.
    - **Primary Key:** Unique identifier for each row in a table.
    - **Foreign Key:** Column in another table that refers to the primary key.

- **Example Scenario:** Retrieve all orders made by specific customers by linking Customers and Orders tables on CustomerID.

---
### Types of JOIN Clauses

- **INNER JOIN:**
    - Retrieves only rows with matches in both tables.
    - Excludes rows where no match exists.
    - **Use case:** List all customers who have made at least one order.

- **LEFT JOIN:**
    - Returns all rows from the left table and matching rows from the right table.
    - Non-matching right table columns appear as NULL.
    - **Use case:** Include all customers, even those without orders.

- **RIGHT JOIN:**
    - Returns all rows from the right table and matching rows from the left table.
    - Non-matching left table columns appear as NULL.
    - **Use case:** Include all orders, even if some have no corresponding customer.

- **FULL OUTER JOIN:**
    - Combines all rows from both tables.
    - Non-matching columns on either side appear as NULL.
    - **Use case:** Get a complete view of all customers and orders, even if they aren’t related.

---
### Syntax of JOIN Clauses

- **Basic Structure:**

```sql
SELECT columns 
	FROM table1
	JOIN table2 ON table1.column = table2.column;
```

- **Notes:**
    - table1 is typically the left table; table2 is the right table.
    - ON clause specifies the relationship (often primary key = foreign key).

---
### Practical Examples for JOIN Clauses

- **Tables:**
    - Customers: CustomerID, Name, DateOfFirstOrder
    - Orders: OrderID, CustomerID, OrderAmount

- **INNER JOIN Example:**

```sql
SELECT Customers.Name, Orders.OrderID 
	FROM Customers
	INNER JOIN Orders
		ON Customers.CustomerID = Orders.CustomerID;

-- Returns only customers who have placed orders.
```

- **LEFT JOIN Example**:

```sql
SELECT Customers.Name, Orders.OrderID
	FROM Customers
	LEFT JOIN Orders
		ON Customers.CustomerID = Orders.CustomerID;

-- Includes all customers; orders appear as NULL if not placed.
```

- **RIGHT JOIN Example**:

```sql
SELECT Customers.Name, Orders.OrderID
	FROM Customers
	RIGHT JOIN Orders
		ON Customers.CustomerID = Orders.CustomerID;

-- Includes all orders; customers appear as NULL if no match.
```

- **FULL OUTER JOIN Example**:
```sql
SELECT Customers.Name, Orders.OrderID
	FROM Customers
	FULL OUTER JOIN Orders
		ON Customers.CustomerID = Orders.CustomerID;

-- Includes all customers and all orders; unmatched columns are NULL.
```

---
### Key Takeaways

- **INNER JOIN:** Focused on matched data only.
- **LEFT JOIN:** Prioritises left table, includes all rows.
- **RIGHT JOIN:** Prioritises right table, includes all rows.
- **FULL OUTER JOIN:** Complete data view from both tables.
- **Primary & Foreign Keys:** Essential for JOINs to function accurately.
