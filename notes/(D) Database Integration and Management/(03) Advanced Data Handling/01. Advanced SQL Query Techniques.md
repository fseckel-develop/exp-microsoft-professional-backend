### Overview

Advanced SQL filtering and query techniques allow you to refine results, manage complex logic, and extract meaningful insights from large datasets. By combining filtering operators, conditional logic, subqueries, Common Table Expressions (CTEs), and optimization strategies, SQL becomes a powerful tool for real-world data challenges.

---
### Advanced Filtering Techniques

Advanced SQL filtering builds on basic conditions and expands into more complex logic.

- **Comparison operators**
    Used to compare values:
    - Equals (=)
    - Greater than (>)
    - Less than (<)
    - Not equal (<>)
    - Greater than or equal (>=)
    - Less than or equal (<=)

- **Logical operators**
    - AND: All conditions must be true
    - OR: At least one condition must be true

- **Pattern matching**
    - LIKE operator with percent (%) wildcard
    - Used for partial text matches

- **List matching**
    - IN operator
    - Matches values from a predefined list

---
### Conditional Logic with CASE

- **CASE statements** introduce decision-making logic into SQL
- Used to classify or label data dynamically
- Common use cases include:
    - Categorising experience levels
    - Assigning performance tiers
    - Creating readable classifications in query results

---
### Aggregate Functions and Group Filtering

Aggregate functions summarise data across multiple rows:

- **COUNT**: Counts rows
- **AVG**: Calculates averages
- **MIN**: Finds smallest values
- **MAX**: Finds largest values
- **SUM**: Totals numeric values

---
### GROUP BY and HAVING

- **GROUP BY**
    - Groups rows with shared values
    - Enables category-level summaries

- **HAVING**
    - Filters results _after_ aggregation
    - Used with aggregate functions
    - WHERE filters rows before aggregation; HAVING filters after

---
### Subqueries

- A **subquery** is a query nested inside another query
- Executes first and passes results to the outer query
- Commonly used in:
    - WHERE clauses for filtering
    - SELECT clauses for calculated fields
    - FROM clauses as virtual tables

- Subqueries are ideal for:
    - Single-use logic
    - Simple filtering
    - List comparisons using IN, ANY, or ALL

---
### Common Table Expressions (CTEs)

- A **CTE** is a temporary result set defined using the WITH clause
- Exists only for the duration of the query    
- Can be referenced multiple times

- **Benefits of CTEs**
	- Improved readability
	- Better organisation of complex logic
	- Reusability of intermediate results
	- Easier debugging and maintenance

- **Subqueries vs CTEs**
	- Subqueries are best for simple, one-time operations
	- CTEs are better for:
	    - Complex queries
	    - Multi-step logic
	    - Queries that reuse the same results

---
### Complex Joins and Query Design

- Advanced joins combine data from multiple tables
- Often involve:
    - Three or more tables
    - Subqueries for pre-filtering
    - CTEs for step-by-step logic
- Subqueries can filter data _before_ joining
- CTEs help manage intermediate calculations prior to joins

---
### Performance Tuning and Optimisation

Optimising SQL queries ensures performance and scalability as data grows.

- Apply WHERE and HAVING clauses early to reduce dataset size
- Avoid deeply nested subqueries
- Use CTEs to simplify complex queries
- Write efficient join conditions
- Create and use indexes effectively

Efficient queries reduce server load and improve response time.

---
### Stored Procedures and Functions

Stored procedures and functions allow SQL logic to be centralised and reused.

- Improved performance through precompiled execution
- Reduced duplication of SQL logic
- Better security through controlled access
- Cleaner and more maintainable code
- **Stored procedures** handle multiple operations
- **Functions** typically return a single computed value

---
### SQL Security Best Practices

- Parameterised queries to prevent SQL injection
- Least privilege access control
- Encryption of sensitive data at rest and in transit
- Regular auditing and monitoring of database activity

Strong security practices help protect data integrity and ensure compliance.

---
### Key Takeaways

- Advanced filtering enables precise data retrieval
- Subqueries and CTEs simplify complex logic
- Aggregate functions and HAVING support grouped analysis
- Optimisation techniques ensure scalable performance
- Security and reusability are essential for production systems

Mastering these advanced SQL techniques significantly enhances your ability to write efficient, maintainable, and secure queries.
