
- Efficient SQL queries focus on smooth, resource-friendly data retrieval rather than speed.
- Small optimisations can significantly improve database responsiveness
- This process is known as SQL performance tuning

---
### Understanding Query Performance and the SQL Optimiser

- Inefficient queries slow databases by retrieving unnecessary data or lacking proper filtering
- The SQL Optimiser determines the most efficient way to execute a query
- It generates an execution plan outlining data access, joins, and filtering steps
- Execution plans act as diagnostic tools for identifying inefficiencies
- Common issues include full table scans, inefficient joins, and unnecessary sorting

---
### Basic Query Optimisation Techniques

- Select only required columns to reduce data processing and memory usage
- Avoid retrieving unnecessary fields to minimize workload
- Use WHERE clauses to filter rows early in query execution
- Reduce processing by eliminating irrelevant data
- Apply DISTINCT to remove duplicate rows when appropriate
- Use LIMIT to restrict result size, especially with large datasets

---
### Key Performance Tuning Techniques

- Query optimisation improves execution by refining structure, joins, and syntax
- Indexing speeds up data access but must be balanced to avoid update overhead
- Execution plan analysis reveals inefficient operations
- Updated database statistics help the optimiser make better decisions
- Limiting result sets reduces memory and processing requirements

---
### Indexing for Performance Optimisation

- Indexes allow faster data retrieval by reducing full table scans
- Single-column indexes optimise searches on individual fields
- Composite indexes improve performance for multi-column filters
- Over-indexing can degrade write performance and should be avoided

---
### Query Planning and Execution Efficiency

- Execution plans show how the database scans tables and performs joins
- Index scans are more efficient than full table scans
- Join efficiency impacts overall query cost
- Reordering joins and processing smaller tables first can improve performance

---
### Advanced Optimization Strategies

- Query rewriting can reduce execution time
- Joins may perform better than nested subqueries in many cases
- EXISTS can be faster than IN for related table checks
- Removing unnecessary computations improves efficiency
- Pagination limits retrieved data and improves responsiveness

---
### Efficient Query Design

- Use SELECT statements with only necessary columns
- Design joins carefully to avoid redundant data processing
- Prefer clear, readable query structures for easier optimisation

---
### Data Types, Caching, and Memory Management

- Choosing smaller appropriate data types reduces storage and improves performance
- Caching frequently accessed data minimises repeated database calls
- Proper memory allocation supports efficient execution for large queries

---
### Batch Processing for Large Operations

- Processing data in batches prevents memory overload
- Batch operations improve stability and transaction handling
- Suitable for bulk updates, imports, and large transformations

---
### Summary

- SQL performance tuning improves efficiency, scalability, and responsiveness
- Indexing, filtering, and execution plan analysis are core optimization tools
- Efficient query design and controlled data retrieval reduce system load
- Applying these techniques ensures databases perform reliably as data grows
