## Introduction

When working with databases, developers often repeat the same queries and calculations across applications. This repetition increases development time and the risk of errors. SQL stored procedures and functions solve this problem by allowing reusable, centralized logic that improves consistency, performance, and security. These tools automate recurring tasks and ensure reliable database operations.

---
## Stored Procedures

Stored procedures are collections of SQL statements stored in the database and executed as a single unit. Instead of rewriting complex logic repeatedly, developers can call a stored procedure whenever needed, ensuring consistent results and reduced errors.

### Types of Stored Procedures

- **User-Defined Stored Procedures**
	  - Custom procedures created by developers
	  - Used for tasks like updating order statuses or generating reports
	  - Promote reusable, maintainable, and consistent database logic

- **Temporary Stored Procedures**
	  - Created for use within a single database session
	  - Automatically removed when the session ends
	  - Ideal for short-term calculations or temporary data processing

---
## Functions in SQL

Functions are stored SQL routines designed to perform calculations or retrieve data and return a result. Unlike stored procedures, functions are typically used within queries and always return a value.

### Types of Functions

- **Scalar Functions**
	  - Return a single value such as a number or string
	  - Commonly used for calculations and data transformations
	  - Example use: calculating sales tax from a product price

- **Table-Valued Functions**
	  - Return a table instead of a single value
	  - Useful for complex, reusable queries
	  - Can be treated like regular tables in the FROM clause

---
## Benefits of Stored Procedures and Functions

- **Improved Code Organization**
	  - Centralises SQL logic for easier maintenance
	  - Reduces duplication across applications

- **Performance Optimisation**
	  - SQL logic is pre-compiled on the database server
	  - Reduces parsing and execution overhead

- **Reduced Network Traffic**
	  - Executes logic on the server rather than sending multiple queries
	  - Improves application efficiency

- **Enhanced Security**
	  - Allows controlled access to database logic
	  - Users can execute procedures without direct access to tables

---
## Creating and Executing Stored Procedures

1. **Define the Procedure**
	   - Use `CREATE PROCEDURE` followed by a unique name
	   - Declare input parameters with names and data types

2. **Add SQL Logic**
	   - Include statements such as `INSERT`, `UPDATE`, or `SELECT`

3. **Declare Output Parameters**
	   - Use `OUTPUT` to return values from the procedure
	   - Useful for returning generated IDs or status values

4. **Complete the Definition**
	   - End the procedure using `END`

5. **Execute the Procedure**
	   - Use `EXEC` and pass required parameters
	   - Capture output values using variables if needed

---
## Creating and Executing Functions

1. **Define the Function**
	   - Use `CREATE FUNCTION` followed by a descriptive name
	   - Declare input parameters with data types

2. **Specify Return Type**
	   - Use `RETURNS` followed by a data type for scalar functions
	   - Use `RETURNS TABLE` for table-valued functions

3. **Add Function Logic**
	   - Scalar functions use `RETURN` for calculations
	   - Table-valued functions return a `SELECT` query

4. **End the Function**
	   - Use `END` to complete the definition

5. **Execute the Function**
	   - Scalar functions are used in `SELECT` statements
	   - Table-valued functions are used in the `FROM` clause

---
## Best Practices

- **Parameter Validation**
	  - Always validate inputs to prevent invalid data and logic errors

- **Error Handling**
	  - Use TRY CATCH blocks to handle errors gracefully
	  - Log errors or return meaningful messages

- **Version Control**
	  - Track changes using version control systems
	  - Essential for collaboration and maintaining stable code

---
## Summary

Stored procedures and functions are essential SQL tools for automating tasks, improving performance, and enhancing security. By centralizing logic within the database, they simplify maintenance, reduce errors, and support scalable, multi-user environments. Mastering these tools allows developers to build efficient, reliable, and secure database-driven applications.
