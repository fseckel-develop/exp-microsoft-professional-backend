## Introduction

- Securing SQL databases is critical because they store sensitive business data, personal information, and intellectual property  
- Weak security can lead to data breaches, financial loss, and reputational damage  
- Strong database security builds trust, ensures compliance, and protects data integrity  

---
## Importance of Database Security

- Protects against data breaches by reducing the risk of unauthorised access  
- Maintains data confidentiality and integrity by preventing tampering and exposure  
- Supports regulatory compliance with legal and industry data protection standards  

---
## Access Control Principles

- Access control defines who can access the database and what actions they can perform  

- **Authentication**
	  - Verifies user identity before granting access  
	  - Best practices include strong passwords, multi-factor authentication (MFA), and single sign-on (SSO)  

- **Authorization**
	  - Assigns permissions based on user roles  
	  - Follows the Principle of Least Privilege (PoLP), granting only necessary access  
	  - Uses role-based access control (RBAC) and regular permission audits  

---
## Encryption for Data Protection

- Encryption converts data into an unreadable format for unauthorized users  

- **Data at Rest**
	  - Protects stored database files using encryption  
	  - Prevents readable data access even if files are compromised  

- **Data in Transit**
	  - Secures data traveling across networks  
	  - Uses protocols like SSL and TLS to prevent interception or tampering  

---
## Preventing SQL Injection Attacks

- SQL Injection occurs when attackers insert malicious SQL through user input  

- **Parameterised Queries**
	  - Use placeholders to treat user input strictly as data  
	  - Prevent execution of malicious SQL code  

- **Input Validation**
	  - Restricts inputs to expected types and formats  
	  - Blocks invalid or potentially harmful data  

- **Stored Procedures**
	  - Limit direct SQL execution  
	  - Allow only predefined and approved database operations  

---
## Preventing Privilege Escalation

- Privilege escalation happens when users gain higher access than intended  

- Apply the Principle of Least Privilege (PoLP)  
- Assign permissions carefully and review them regularly  
- Use role-based access to limit unnecessary exposure  

---
## Additional Security Layers

- **Regular Secure Backups**
	  - Encrypted and stored offsite  
	  - Enable recovery from breaches, corruption, or accidental loss  

- **Monitoring and Logging**
	  - Tracks database activity in real time  
	  - Detects suspicious behaviour and unauthorised access attempts  

- **Firewalls**
	  - Restrict database access to trusted IP addresses  
	  - Act as a barrier against unauthorised internal and external access  

---
## Summary

- SQL database security protects sensitive data, ensures compliance, and preserves trust  
- Key practices include access control, encryption, injection prevention, and layered defenses  
- A strong, multi-layered security strategy keeps databases reliable, resilient, and secure in evolving threat environments
