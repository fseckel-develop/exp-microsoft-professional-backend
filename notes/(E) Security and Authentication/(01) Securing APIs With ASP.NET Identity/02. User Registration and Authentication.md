## Overview

- ASP.NET Identity provides secure mechanisms for user registration and authentication
- Focuses on protecting user credentials and maintaining secure access
- Uses best practices such as:
	- Password hashing
	- Email verification
	- Session and cookie management

---
## User Registration Process

### Form Submission

- User enters registration details (e.g., email and password)
- Registration form is submitted to the server
- Data is transmitted securely for processing
- Server prepares the account for creation

### Password Hashing

- Plain-text passwords are never stored
- Submitted passwords are converted into hashed values
- Hashing ensures:
	- Passwords are unreadable if data is exposed
	- Original passwords cannot be reverse-engineered
- Protects user credentials even if the database is compromised

### Email Confirmation

- System sends a verification email to the registered address
- Email contains a confirmation link
- User confirms ownership by selecting the link
- Prevents fake or unauthorized account creation

### Secure Data Storage

- Confirmed user data is stored securely
- ASP.NET Identity saves:
	- Hashed passwords
	- Email addresses
	- Email confirmation status
- Data is persisted using IdentityDbContext with Entity Framework

---
## User Authentication Process

### Credential Submission

- User enters username/email and password into a login form
- Credentials are sent to the server for validation
- Initiates the authentication workflow

### Password Verification

- Server hashes the submitted password
- Hashed value is compared with the stored hash
- Authentication succeeds only if hashes match
- Ensures plain-text passwords are never exposed

### Session Management

- A session is created after successful authentication
- Session maintains the user's logged-in state
- Sessions:
	- Are temporary
	- Expire after inactivity
- Enhances security while improving user experience

### Persistent Login with Cookies

- Optional “Remember Me” feature
- Uses cookies to persist login state across visits
- Differences from sessions:
	- Cookies remain after browser closure
	- Sessions end after expiration or inactivity
- Balances convenience with controlled security

---
## Summary

- ASP.NET Identity ensures secure registration and authentication workflows
- Combines hashing, verification, and session handling for strong security
- Enables developers to build reliable, user-friendly, and secure web applications
