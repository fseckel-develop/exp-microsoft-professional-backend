## Overview

- Role-Based Access Control (RBAC) manages permissions through predefined user roles
- Ensures users access only the resources relevant to their responsibilities
- Scales effectively from small teams to large organizations
- Reduces complexity in permission management

---
## Definition of RBAC

- RBAC is a security model that restricts access based on user roles
- Users are assigned roles instead of individual permissions
- Each role contains a predefined set of permissions
- Common roles include:
	- Admin
	- Manager
	- Customer
- Simplifies access control by grouping responsibilities

---
## Core RBAC Concepts

### Role Assignment

- Users are grouped into roles with shared responsibilities
- Permissions are assigned at the role level
- New users receive permissions automatically by role assignment
- Prevents inconsistent or incorrect permission allocation

### Permission Management

- Roles define what actions users can perform
- Centralised permission updates affect all assigned users
- Easier auditing and maintenance of access rules

---
## RBAC and JSON Web Tokens (JWT)

### JWT Usage in RBAC

- JWTs securely transmit user identity and role information
- Generated after successful authentication
- Typically contain:
	- User ID
	- Assigned roles
	- Metadata
- Stored client-side and sent with each request

### Benefits of JWT Integration

- Enables stateless authentication
- Improves performance and scalability
- Ensures role verification without server-side session storage

---
## RBAC vs Claims-Based Access Control

### RBAC Characteristics

- Permissions are bundled into roles
- Easier to manage and understand
- Best suited for structured access scenarios
- Less flexible for unique or dynamic permissions

### Claims-Based Access Characteristics

- Permissions are assigned as individual claims
- Provides granular and user-specific access control
- Better suited for complex or evolving requirements

---
## RoleManager in ASP.NET Core

### Role Management

- RoleManager is used to create and manage roles
- Assigns roles to users during authentication
- Automates permission distribution

### Role-Based Token Generation

- Upon login, RoleManager assigns roles
- JWT is generated containing role information
- Access is granted immediately based on token contents
- Reduces manual permission configuration

---
## Common User Roles

### Admin

- Full system access
- Can manage:
	- Users
	- Products
	- Orders
- JWT includes admin role with all permissions

### Manager

- Oversees operations and workflows
- Permissions include:
	- Viewing products
	- Accessing order lists
- Elevated access without full control

### Customer

- Limited access to essential features
- Can:
	- Browse products
	- Place orders
- JWT restricts permissions to customer-level actions

---
## Summary

- RBAC simplifies access management by grouping permissions into roles
- JWTs securely enforce role-based permissions
- RoleManager streamlines role assignment in ASP.NET Core
- RBAC is ideal for applications with clear role hierarchies
- Complements claims-based models for more advanced scenarios
