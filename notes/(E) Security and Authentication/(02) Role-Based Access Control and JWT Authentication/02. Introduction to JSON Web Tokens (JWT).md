## Overview

- JSON Web Tokens (JWTs) are compact, self-contained tokens used to securely transmit information between a client and a server
- Commonly used for authentication and authorization in modern web applications
- Ensure data integrity, authenticity, and trust without maintaining server-side session state

---
## Structure of JWTs

- A JWT consists of three encoded components combined into a single string
- Each part plays a specific role in security and data transmission

### Header

- Describes metadata about the token
	- Token type (JWT)
	- Signing algorithm (e.g., HS256)
- Allows the server to determine how to verify the token’s authenticity

### Payload

- Contains **claims**, which are pieces of information about the user
	- Examples:
		- User ID
		- Username
		- Roles (e.g., admin)
		- Expiration time (`exp`)
- Enables the server to identify the user and determine their permissions
- Expiration claim limits how long the token remains valid

### Signature

- Ensures the integrity of the token
	- Created by encoding the header and payload
	- Signed using a secret key
- Prevents tampering
	- Any modification to header or payload invalidates the signature

---
## JWTs in Authentication and Authorisation

- JWTs are generated after successful user authentication
	- Token includes identity and role-based claims
- Used by the application to control access
	- Example:
		- Admin role → access to administrative features
- Eliminates the need for server-side session storage
	- Enables stateless authentication

---
## Signing and Verification

- JWTs are signed to prevent unauthorised modification
	- Server uses a secure secret key
- Verification process:
	- Server checks the token’s signature
	- Confirms the token has not expired
	- Validates claims before granting access
- Expired or altered tokens are automatically rejected

---
## Typical JWT Workflow

- User submits credentials to the server
- Server authenticates credentials
- Server generates a JWT
	- Includes user ID, roles, and expiration
	- Signs token with a secret key
- Token is returned to the client
- Client stores the token securely
	- Commonly in cookies or local storage
- Client includes JWT in requests to protected resources
- Server verifies the token on each request
	- Grants or denies access based on claims

---
## Real-World Applications of JWTs

- Web applications
	- Role-based access control (e.g., admin, editor, user)
- API security
	- Protects endpoints by requiring valid JWTs
- Session management
	- Stateless authentication reduces server load and complexity

---
## Key Takeaways

- JWTs securely transmit user data using a compact format
- Structure (header, payload, signature) ensures integrity and trust
- Signing and expiration protect against tampering and misuse
- Widely adopted for scalable, secure authentication and authorization
