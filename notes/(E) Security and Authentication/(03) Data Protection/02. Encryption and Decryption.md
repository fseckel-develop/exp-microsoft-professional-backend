## Introduction

Sensitive information such as passwords, credit card numbers, and personal messages travels across the internet every day. Without protection, this data could be intercepted and misused. Encryption makes digital security possible by transforming readable data into an unreadable form that only authorised users can access. This guide explains how encryption works, the main types of encryption, common algorithms, and best practices for secure implementation.

---
## What Is Encryption?

Encryption is the process of converting readable data, known as **plain text**, into an unreadable format called **ciphertext**. This transformation protects sensitive information from unauthorised access.

Encryption works like a **lock-and-key system**:

- An **algorithm** scrambles the data.
- A **key** unlocks the encrypted data and restores it to readable form.
- Without the correct key, the data remains secure and unreadable.

---
## Types of Encryption

### Symmetric Encryption

Symmetric encryption uses the **same key** for both encryption and decryption.

**Key Characteristics:**
- Fast and efficient
- Best for large amounts of data
- Requires secure key sharing

**Example:**  
A company encrypts files on a shared drive using a single password known only to authorised employees.

### Asymmetric Encryption

Asymmetric encryption uses **two different keys**:
- **Public key**: Used to encrypt data (shared openly)
- **Private key**: Used to decrypt data (kept secret)

**Key Characteristics:**
- More secure for data exchange over the internet
- Slower than symmetric encryption
- Ideal for secure communications

**Example:**  
Secure email communication, where the sender encrypts a message using the recipient’s public key, and only the recipient can decrypt it with their private key.

---
## Common Encryption Algorithms

### Advanced Encryption Standard (AES)

- Type: Symmetric encryption
- Known for speed and strong security
- Commonly used for data at rest

**Use Cases:**
- File storage
- Databases
- Online banking transactions

### RSA Encryption

- Type: Asymmetric encryption
- Widely used for secure data exchange
- Common in SSL/TLS certificates

**Use Cases:**
- Secure website connections (HTTPS)
- Secure key exchange

**How AES and RSA Work Together:**
- RSA securely exchanges a secret key
- AES uses that key to encrypt data efficiently during the session

---
## Implementing Symmetric Encryption (AES)

1. **Select an Algorithm**  
   Use AES for secure and efficient encryption.

2. **Generate a Key**  
   Create a strong random key (e.g., 256-bit).

3. **Initialise the Cipher**  
   Configure AES with the key and an Initialisation Vector (IV).

4. **Encrypt Data**  
   - Convert plain text to bytes  
   - Encrypt to produce cipher text

5. **Store Securely**  
   Protect the encryption key using secure storage mechanisms.

---
## Implementing Asymmetric Encryption (RSA)

1. **Generate a Key Pair**  
   Create a public/private key pair (e.g., 2048-bit RSA).

2. **Distribute the Public Key**  
   Share it with users who need to encrypt data.

3. **Encrypt Data**  
   Use the public key to encrypt information.

4. **Decrypt Data**  
   Use the private key to restore the original plain text.

5. **Key Management**  
   Protect the private key with strict access controls.

---
## Testing Encryption and Decryption

- Encrypt sample data
- Decrypt it to confirm accuracy
- Verify that data integrity is preserved

---
## Best Practices for Key Management

- Rotate keys regularly
- Store keys securely (e.g., Hardware Security Modules)
- Restrict key access
- Backup keys securely
- Use strong key sizes (AES-256, RSA-2048 or higher)

---
## Data Security Best Practices

- Encrypt data in transit using TLS
- Encrypt data at rest using AES
- Never hardcode encryption keys
- Audit encryption systems regularly
- Monitor for unauthorised access attempts

---
## Conclusion

Encryption is a foundational element of data protection. By understanding symmetric and asymmetric encryption, using secure algorithms like AES and RSA, and following strong key management practices, organisations can protect sensitive information and maintain trust in a connected digital world.
