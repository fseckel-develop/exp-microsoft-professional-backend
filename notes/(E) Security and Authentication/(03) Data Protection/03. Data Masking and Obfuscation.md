## Overview

- Businesses handle large volumes of sensitive data (e.g. payment details, personal identifiers)
- Data masking and data obfuscation are key techniques to protect this information
- Both approaches reduce exposure of sensitive data but serve different purposes / use cases

---
## Data Masking

### Definition and Purpose

- Data masking alters sensitive data elements by replacing them with fictional but realistic values
- Purpose is to protect confidentiality while preserving data usability
- Commonly used in testing, development, analytics, and training environments

### Types of Data Masking

- Static Data Masking
	- Creates a masked copy of an existing dataset
	- Original data is replaced with fictitious values
	- Structure and format are preserved for realistic testing
	- Example:
		- Bank account number `0123456` → `8304652`

- Dynamic Data Masking
	- Masks data in real time without altering the stored data
	- Displayed values depend on user role or authorization level
	- Example:
		- Customer view: `OliviaDIDXXXX1234`
		- Authorized staff view: full identifier

- Personal Data Masking
	- Focuses on concealing personal identifiers
	- Common in customer service and testing environments
	- Example:
		- Phone number displayed as `XXXXXX7890`

- Social Security Number Masking
	- Restricts visibility to partial identifiers
	- Example:
		- `XXXXX6789` shown to unauthorised users

### Benefits of Data Masking

- Protects sensitive and confidential information
- Supports compliance with privacy regulations
- Enables safe use of realistic data in non-production environments
- Reduces risk of data exposure during testing and development

---
## Data Obfuscation

### Definition and Purpose

- Data obfuscation deliberately makes data difficult or impossible to interpret
- Protects information even if unauthorised access occurs
- Ideal when data usability is not required for humans

### Data Obfuscation Techniques

- Tokenisation
	- Replaces sensitive data with meaningless tokens
	- Tokens have no value outside a secure mapping system
	- Example:
		- Credit card number replaced with token like `9XJ67P`

- Data Scrambling
	- Alters data so the original value is unrecognizable
	- Maintains structure but destroys readability
	- Often used during transmission or storage
	- Example:
		- Scrambled customer IDs sent over unsecured networks

- Code Obfuscation
	- Modifies software code to hide logic and intent
	- Preserves functionality while preventing reverse engineering
	- Techniques include:
		- Renaming variables
		- Adding misleading or redundant code
		- Restructuring logic flow

---
## Key Differences Between Masking and Obfuscation

- Data Masking
	- Substitutes real data with realistic fake data
	- Data remains usable and readable
	- Best for testing, training, and analytics

- Data Obfuscation
	- Transforms data into unreadable or unintelligible form
	- Focuses on preventing interpretation entirely
	- Best for protecting data in storage, transmission, or development systems

---
## Conclusion

- Data masking preserves usability while protecting sensitive information
- Data obfuscation prioritises security by making data unusable to unauthorized users
- Both techniques are essential components of a comprehensive data protection strategy
- Choosing the right approach depends on whether data realism or maximum concealment is required
