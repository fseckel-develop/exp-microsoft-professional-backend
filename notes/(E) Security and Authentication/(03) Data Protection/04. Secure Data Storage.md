## Overview

- Learn best practices for securely storing data
- Understand encryption, access controls, and data redundancy
- Apply practical steps to protect sensitive information at rest

---
## Encryption

- **Definition:** 
	  Converting readable data (plain text) into unreadable text (cipher text) to protect sensitive information.
- **Purpose:** 
	  Protects data at rest from unauthorised access.
- **Example:** 
	  AES-256 encryption ensures that customer data remains unreadable without the correct key.
- **Implementation Steps:**
	  1. Choose an encryption algorithm (AES-256 recommended).
	  2. Encrypt sensitive data in databases, file systems, or storage devices.
	  3. Secure encryption keys using a Key Management System (KMS).
	  4. Restrict access to keys to authorised personnel.
	  5. Test encryption to verify data is inaccessible without keys.

---
## Access Controls

- **Definition:** 
	  Mechanisms that define who can access or modify stored data.
- **Purpose:** 
	  Ensures only authorised individuals can view or edit sensitive information.
- **Example:** 
	  Role-Based Access Control (RBAC) restricts customer support agents to basic customer information while giving admins full access.
- **Implementation Steps:**
	  1. Identify user roles (Admin, Manager, Support Agent, etc.).
	  2. Set role-based access controls defining permissions for each role.
	  3. Implement attribute-based access controls (ABAC) for finer-grained access.
	  4. Assign minimum required permissions for each role (least privilege).
	  5. Regularly audit access and revoke permissions when no longer needed.

---
## Data Redundancy

- **Definition:** 
	  Creating backup copies of data to ensure availability if one copy is lost or damaged.
- **Purpose:** 
	  Maintains access to critical information during failures or data loss.
- **Example:** 
	  A business keeps daily transaction data on a local server and in the cloud; if local storage fails, the cloud backup ensures access.
- **Implementation Steps:**
	  1. Create backup copies in multiple locations (local and cloud).
	  2. Automate backup schedules (daily, weekly, or as needed).
	  3. Verify backup integrity to ensure successful restoration.
	  4. Implement redundant storage systems (e.g., RAID configurations) for continuous availability.

---
## Summary

- **Encryption** secures data at rest by making it unreadable without proper keys.
- **Access Controls** limit data access to authorised users based on roles and attributes.
- **Data Redundancy** ensures data availability even during failures or loss.

By combining these, organisations can maintain secure, reliable, and accessible data storage.
