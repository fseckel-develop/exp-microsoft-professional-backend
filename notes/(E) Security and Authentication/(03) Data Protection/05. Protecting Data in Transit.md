---
### Data in Transit

- **Definition:** 
	  Data actively moving from one place to another, such as login credentials, messages, or financial transactions.

- **Risk:** 
	  Vulnerable to interception, unauthorised access, or tampering during transmission.

- **Goal:** 
	  Ensure privacy, integrity, and security while data is being transmitted.

---
### Encryption Protocols

#### Transport Layer Security (TLS)

- **Purpose:** 
	  Encrypts data between a client (e.g., web browser) and a server (e.g., website).

- **Function:** 
	  Protects sensitive information such as login credentials and payment details.

- **Example:** 
	  Entering credit card information on an online shopping site; TLS encrypts it to prevent interception.

- **TLS vs. SSL:** 
	  TLS replaced SSL due to stronger encryption and improved security.

- **Implementation Notes:**
	- Establishes a secure connection using a TLS handshake.
	- Verifies server identity to prevent man-in-the-middle attacks.

#### Virtual Private Networks (VPNs)

- **Purpose:** 
	  Creates a secure, encrypted tunnel between a user’s device and a network.

- **Function:** 
	  Protects all network traffic, including sensitive communications, over public networks.

- **Use Cases:**
	- Remote employees accessing company resources securely.
	- Protecting privacy by masking IP addresses.
	- Geo-spoofing to access restricted content safely.

- **Example:** 
	  Employee logs in from public Wi-Fi using a VPN to ensure data is encrypted and private.

---
### Network-Based Protections

#### Firewalls

- **Purpose:** 
	  Monitors and controls incoming and outgoing network traffic based on predefined security rules.

- **Function:** 
	  Blocks unauthorised or malicious traffic while allowing legitimate data to pass.

- **Example:** 
	  Filtering network traffic to prevent malware or suspicious access.

#### Intrusion Detection Systems (IDS)

- **Purpose:** 
	  Continuously monitors network traffic for unusual or suspicious behaviour.

- **Function:** 
	  Alerts system administrators to potential threats in real-time.

- **Example:** 
	  Like a security camera, IDS detects anomalies and flags potential breaches immediately.

---
## Layered Security Approach

- **TLS:** 
	  Protects individual connections, ensuring sensitive data like payment info remains encrypted.

- **VPN:** 
	  Secures all network traffic, masking source and destination IPs and adding privacy for remote connections.

- **Combined Approach:** 
	  TLS + VPN + Firewalls + IDS = robust, multi-layered defense for data in transit.

---
## Summary

- **TLS:** Encrypts specific communications between client and server.
- **VPNs:** Secures entire network traffic and protects privacy.
- **Firewalls & IDS:** Add network-level monitoring and threat prevention.
- **Outcome:** Together, these protocols maintain privacy, integrity, and trust in digital communications.
