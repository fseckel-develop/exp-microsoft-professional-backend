## Importance of Secure Coding

- Every vulnerability can become an entry point for attackers
- Secure coding is foundational for building trustworthy systems
- Prevents data theft, service disruption, and loss of user trust
- Focuses on mitigating common vulnerabilities like SQL Injection and XSS

---
## Core Secure Coding Principles

- Proactively prevent vulnerabilities rather than reacting to breaches
- Validate and sanitise all user inputs
- Rely on established security standards and trusted tools
- Continuously review, test, and assess application security

---
## Input Validation

- Ensures user input is safe, correct, and expected
- Blocks harmful or malformed data before processing
- Reduces attack surface for injection-based vulnerabilities
- Common use cases include:
	- Validating form inputs (e.g., email, usernames)
	- Rejecting malicious scripts or unexpected characters

---
## Common Vulnerabilities Addressed

- SQL Injection
	- Occurs when unvalidated input alters database queries
	- Can lead to data leakage or database manipulation
- Cross-Site Scripting (XSS)
	- Attackers inject malicious scripts into user-facing content
	- Scripts execute in victims’ browsers without their knowledge
	- Can result in session hijacking, data theft, or unauthorised actions

---
## Secure Coding Standards and Resources

- OWASP Top 10
	- Industry-recognised list of critical web application risks
	- Provides actionable mitigation guidance
	- Covers issues like injection, misconfiguration, and authentication flaws
- Trusted Libraries
	- Use proven solutions (e.g., Bcrypt for password hashing)
	- Avoids insecure custom implementations
	- Protects sensitive data even if systems are compromised

---
## Code Reviews and Automated Testing

- Code Reviews
	- Help catch missed validations or logic flaws early
	- Encourage shared responsibility and knowledge
- Automated Security Testing
	- Scans for outdated dependencies and known vulnerabilities
	- Provides fast, repeatable security checks
	- Improves long-term application resilience

---
## Penetration Testing

- Simulates real-world attack scenarios
- Identifies weaknesses before attackers do
- Examples include:
	- Brute-force login vulnerabilities
	- Authentication and authorisation flaws

---
## Using Microsoft Copilot for Secure Coding

- Assists in generating secure code patterns
- Helps create:
	- Input validation logic
	- Parameterised database queries
	- XSS detection and mitigation routines
- Can refine code to align with OWASP Top 10
- Supports best practices such as:
	- Modular design
	- Clear naming conventions
	- Documentation and maintainability

---
## Conclusion

- Secure coding is an ongoing process, not a one-time task
- Combining validation, standards, testing, and tooling strengthens defenses
- Microsoft Copilot can accelerate secure development when guided properly
- Well-secured applications are more resilient, reliable, and user-centric
