## Introduction

- Purpose of in-memory caching
	- Improves application performance by storing frequently accessed data in memory
	- Reduces response times and system load
- Role of IMemoryCache in .NET / ASP.NET Core
	- Interface for temporary, in-memory data storage
	- Suitable for frequently accessed or computationally expensive data
- Importance of cache management
	- Prevents excessive memory usage
	- Ensures scalability and reliability under heavy load

---
## Core Concept: What Is IMemoryCache?

- Definition
	- In-memory caching interface provided by ASP.NET Core
	- Stores data in server RAM for fast access
- Key benefits
	- Faster data retrieval compared to database or API calls
	- Reduced backend load
	- Improved overall user experience
- Typical use cases
	- Product lists
	- User profiles
	- Configuration data
	- Read-heavy, infrequently changing data

---
## Configuring In-Memory Caching

- Enabling caching
	- Add IMemoryCache service in `Startup.cs`
	- Reference `Microsoft.Extensions.Caching.Memory`
- Creating the cache
	- Use `MemoryCacheOptions` to configure behavior
	- Define a size limit to control memory usage
- Cache size management
	- Example size limit: 1024 units
	- Size units are arbitrary and defined by the developer
	- Prevents uncontrolled memory growth

---
## Managing Cache Entries

### Step 1: Storing Data in the Cache

- Using the `Set` method
	- Stores key-value pairs in memory
- Configuration options
	- Size
		- Each entry contributes to the total cache size
	- Expiration
		- `AbsoluteExpirationRelativeToNow` defines lifespan
- Benefits
	- Keeps cache efficient and organised
	- Automatically removes outdated entries

### Step 2: Retrieving Data from the Cache

- Using `TryGetValue`
	- Checks whether data exists in the cache
	- Returns false if data is missing
- Cache hit
	- Data retrieved instantly from memory
- Cache miss
	- Data not found in cache
	- Triggers fallback logic to retrieve data from the database or service

---
## Handling Cache Misses

- Definition of a cache miss
	- Requested data is not present in the cache
- Miss handling strategy
	- Fetch data from the database asynchronously
	- Store retrieved data in cache for future requests
- Performance impact
	- Prevents blocking operations
	- Improves responsiveness during data retrieval

---
## Cache Expiration and Invalidation

- Why expiration is necessary
	- Prevents stale or outdated data
	- Maintains data accuracy in dynamic systems
- Expiration strategies
	- Time-based expiration
		- Data removed after a defined duration
	- Event-based refresh
		- Refresh cache when underlying data changes
- Manual invalidation
	- Use `Remove` method with cache key
	- Forces fresh data retrieval on next request

---
## Cache Management Best Practices

- Managing memory usage
	- Always define cache size limits
	- Assign sizes to individual entries
- Clearing stale data
	- Remove outdated or irrelevant cache entries
- Choosing what to cache
	- Data accessed frequently
	- Data that changes infrequently
- Avoid caching
	- Highly volatile or real-time critical data
