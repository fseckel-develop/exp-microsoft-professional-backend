## Overview

- Efficient systems require both performance and data accuracy
- Cache expiration policies ensure cached data remains relevant and trustworthy
- Proper expiration balances speed, memory usage, and reliability
- Common expiration strategies include absolute, sliding, and dependent expiration

---
## Key Cache Expiration Policies

### Absolute Expiration

- Cached data expires after a fixed time period
- Expiration occurs regardless of how often the data is accessed
- Best suited for data with a predictable lifespan
	- Example: session tokens
		- A session token expires after 30 minutes of inactivity
		- Enhances security by preventing unauthorised access

### Sliding Expiration

- Expiration timer resets every time the data is accessed
- Cached data remains active as long as it is frequently used
- Ideal for user-specific, interaction-driven data
	- Example: shopping carts
		- Each user action resets the expiration timer
		- Prevents cart data from expiring while browsing

### Dependent Expiration

- Cached entries are invalidated when related data changes
- Ensures consistency between cached data and source data
- Commonly used for linked or hierarchical data
	- Example: inventory systems
		- Product stock updates invalidate related cache entries
		- Prevents users from seeing outdated availability

## Implementing Cache Expiration in .NET

### Using IMemoryCache

- Supports absolute and sliding expiration policies
- Configuration is done through `MemoryCacheEntryOptions`
- Absolute expiration example:
	- Data expires after a fixed time (e.g., 5 seconds)
	- Cache entry is removed without reset
- Sliding expiration example:
	- Expiration resets on every access
	- Data persists while actively used

### Using Redis for Distributed Caching

- Enables dependent expiration across multiple servers
- Cache keys can be logically linked
- Updating a parent key invalidates dependent child keys
	- Example:
		- Updating product data removes related inventory cache
		- Ensures consistency across distributed systems

## Best Practices for Cache Management

### Balancing Performance and Memory

- Use absolute expiration for predictable data lifecycles
- Apply sliding expiration to frequently accessed user data
- Avoid excessive cache retention to prevent memory waste

### Ensuring Data Consistency

- Implement dependent expiration for related data sets
- Use centralised caching systems like Redis for synchronisation
- Prevent stale or inconsistent data in distributed environments

### Testing and Monitoring Expiration

- Verify cache expiration behaviour regularly
- Ensure expired data is reloaded correctly
- Monitor cache behaviour to detect inefficiencies or misconfigurations

## Conclusion

- Cache expiration policies are critical for maintaining fast and reliable systems
- Absolute, sliding, and dependent expiration each serve distinct use cases
- Combining multiple strategies provides flexible and efficient cache management
- Proper expiration ensures fresh data, optimal performance, and a better user experience
