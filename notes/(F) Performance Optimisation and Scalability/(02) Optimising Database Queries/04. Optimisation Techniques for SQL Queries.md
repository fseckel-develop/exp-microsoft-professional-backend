## Purpose of SQL Query Optimisation

- Ensure queries use database resources efficiently
- Reduce execution time and server load
- Improve application responsiveness and scalability
- Produce cleaner, more maintainable SQL code

---
## Core Optimisation Techniques

### Rewriting Queries for Simplicity

- Simplifying query structure improves performance
- Reduces unnecessary computation and server strain
- Enhances readability and maintainability
- Common approach:
	- Remove or rewrite subqueries
	- Replace subqueries with WHERE clauses or JOINs
- Example benefit:
	- Direct filtering with WHERE is often more efficient than nested subqueries

### Managing Subqueries

- Subqueries are nested SQL statements inside a main query
- Used for intermediate calculations or data filtering
- Performance drawbacks:
	- Require separate execution
	- Increase computational overhead
- Optimisation strategy:
	- Rewrite logic to filter directly in the main query
	- Avoid matching results via nested queries when possible

---
## Index Usage for Performance

### Using Indexes Effectively

- Indexes enable fast data lookups
- Prevent full table scans
- Created using CREATE INDEX
- Key benefits:
	- Faster searches on frequently queried columns
	- Improved filtering and sorting performance
	- Reduced I/O operations

---
## Avoiding Inefficient Temporary Storage

### Temporary Tables

- Store intermediate query results
- Consume additional memory
- Increase query execution time

### Common Table Expressions (CTEs)

- Defined using the WITH keyword
- Exist only during query execution
- Advantages:
	- Avoid physical temporary table creation
	- Improve readability and modularity
	- Allow reusable, named subqueries within a single query
- Ideal for:
	- Complex calculations
	- Intermediate result reuse

---
## Optimising Joins

### Efficient Join Strategies

- Joins combine data from multiple tables
- Can become resource-intensive on large datasets
- Optimisation techniques:
	- Filter data before performing joins
	- Use indexed columns for join conditions
	- Apply WHERE conditions early to reduce dataset size
- Additional improvements:
	- Use table aliases for clarity and concise syntax

---
## Measuring Performance Improvements

### Performance Analysis Best Practices

- Use EXPLAIN PLAN to inspect query execution steps
- Identify inefficiencies such as:
	- Full table scans
	- Missing index usage
- Benchmark queries:
	- Before optimisation
	- After optimisation
- Limit retrieved data by:
	- Selecting only required columns
	- Filtering with indexed conditions

---
## Key Takeaways

- Small optimisations can lead to significant performance gains
- Effective techniques include:
	- Query rewriting
	- Strategic indexing
	- Eliminating unnecessary temporary tables
	- Optimising join logic
- Continuous analysis ensures long-term efficiency and scalability
