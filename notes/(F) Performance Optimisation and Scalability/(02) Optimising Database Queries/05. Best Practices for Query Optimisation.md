## Overview

- Efficient SQL queries improve database performance, scalability, and responsiveness
- Poorly written queries increase resource usage and slow down applications
- Query optimisation combines good query design, indexing, and continuous monitoring
- Optimisation is an ongoing process, especially in production environments

---
## Best Practices for Writing Efficient SQL Queries

### (1) Avoid SELECT *

- Retrieving all columns increases memory usage and I/O
- Limits performance, especially on large tables
- Best practice:
	- Select only required columns
- Benefit:
	- Reduces unnecessary data transfer
	- Keeps queries lean and efficient

### (2) Use WHERE Clauses Effectively

- Filters data early during query execution
- Prevents full table scans
- Improves performance by narrowing result sets
- Example use cases:
	- Filtering active users
	- Limiting results to recent records using date ranges

### (3) Limit the Use of Subqueries

- Subqueries execute additional processing steps
- Can slow down performance on large datasets
- Optimisation approach:
	- Replace subqueries with JOINs when possible
- Benefits:
	- Reduced overhead
	- Improved readability and execution efficiency

### (4) Use JOINs Instead of Subqueries

- JOINs connect related tables directly
- More efficient than nested queries
- Best practices:
	- Join only necessary tables
	- Retrieve only required columns
- Optimisation tip:
	- Apply WHERE conditions to limit joined data early

### (5) Optimise JOIN Operations

- JOINs can become expensive if poorly designed
- Optimisation strategies:
	- Use appropriate join types
	- Ensure join columns are indexed
	- Avoid pulling unnecessary rows
- Example:
	- Filtering recent orders in a customer dashboard

### (6) Use Pagination for Large Datasets

- Prevents loading large result sets at once
- Improves application responsiveness
- Techniques:
	- LIMIT and OFFSET
	- ORDER BY for consistent results
- Common use cases:
	- E-commerce product listings
	- News or article feeds

---
## Indexing Strategies for Optimised Queries

- Create indexes on frequently searched columns
	- Columns used in WHERE, JOIN, and ORDER BY clauses
- Use covering indexes
	- Include all required columns to reduce lookups
- Monitor and adjust indexes
	- Add or remove indexes based on usage patterns
- Balance performance
	- Avoid over-indexing to prevent write slowdowns

---
## Continuous Query Monitoring and Optimisation

### Monitoring Query Performance

- Track execution times and resource usage
- Identify slow-running or resource-heavy queries
- Common tools:
	- SQL Profiler
	- Dynamic Management Views (DMVs)

### Analysing Query Workloads

- Focus on queries that:
	- Run frequently
	- Consume the most CPU or memory
- Use workload analysis to prioritise optimisation efforts
- Example:
	- Queries consuming high CPU over repeated executions

### Dynamic Index Optimisation

- Adjust indexes as workloads evolve
- Add indexes for new query patterns
- Remove unused or low-value indexes
- Performance benefit:
	- Significant reductions in query execution time

### Setting Up Alerts for Performance Issues

- Proactively detect slow or inefficient queries
- Configure alerts for performance thresholds
- Benefits:
	- Faster response to issues
	- Reduced user impact
- Enables proactive database management

---
## Key Takeaways

- Efficient query writing reduces resource usage and improves performance
- Indexing and JOIN optimisation are critical for scalable systems
- Pagination helps manage large datasets effectively
- Query optimisation is continuous, not a one-time task
- Regular monitoring, analysis, and adjustment ensure long-term database health
