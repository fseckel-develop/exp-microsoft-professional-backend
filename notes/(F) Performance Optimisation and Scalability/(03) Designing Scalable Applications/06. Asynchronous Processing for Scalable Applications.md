## Introduction

-	Modern applications must handle multiple operations simultaneously without slowing down.
-	Asynchronous processing allows tasks to run independently from the main thread.
-	Improves:
	-	Responsiveness
	-	Scalability
	-	Resource utilisation
-	Prevents blocking behaviour during time-consuming operations.

---
## Core Concepts of Asynchronous Processing

### Asynchronous Tasks

-	Run independently from the main application thread.
-	Execute in the background without blocking user interaction.
-	Ideal for:
	-	Email sending
	-	Data fetching
	-	File uploads
	-	Long-running computations
-	Example:
	-	Email applications load messages in the background while users navigate the interface.

### Queues

-	Provide temporary storage for background tasks.
-	Ensure tasks are processed in an orderly (FIFO) manner.
-	Prevent system overload by controlling execution flow.
-	Example:
	-	Ride-sharing apps queue ride requests until a driver is available.
-	Benefits:
	-	Improved reliability
	-	Better workload distribution
	-	Controlled execution order

### Concurrency

-	Allows multiple tasks to run simultaneously.
-	Maximises CPU and resource utilisation.
-	Improves performance in high-traffic systems.
-	Example:
	-	Streaming platforms preload upcoming scenes while playing the current one.

### Non-Blocking I/O

-	Handles input/output operations without pausing other processes.
-	Keeps applications responsive during:
	-	File uploads
	-	API calls
	-	Database queries
-	Example:
	-	A file upload continues while users interact with other parts of the application.

## Implementing Queues in a .NET Application

### Queue Setup

-	Use .NET’s built-in `Queue` class.
-	Tasks are added using:
	-	Enqueue()
-	Tasks are processed using:
	-	Dequeue()
-	Follows FIFO (First-In, First-Out) principle.

### Background Worker Processing

-	A background worker:
	-	Receives the queue
	-	Processes tasks asynchronously
	-	Executes while the main thread remains responsive
-	Workflow:
	-	Check if queue contains items
	-	Dequeue task
	-	Process task
	-	Mark task as complete

### Cancellation Tokens

-	Allow graceful termination of background processing.
-	Used to:
	-	Stop tasks when errors occur
	-	Allow users to cancel operations
-	Improves:
	-	System safety
	-	User control
	-	Error handling

### Benefits of Queue-Based Async Processing

-	Prevents UI blocking.
-	Allows user interaction during processing.
-	Maintains task order.
-	Improves application resilience.
-	Supports scalable background job handling.
