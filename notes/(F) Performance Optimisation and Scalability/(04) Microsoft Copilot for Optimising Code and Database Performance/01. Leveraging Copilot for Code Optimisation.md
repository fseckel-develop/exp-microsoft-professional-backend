## Performance Enhancements

- Purpose:
	- Improve execution speed.
	- Optimise resource usage.
- How Copilot Helps:
	- Suggests refining loops and algorithms.
	- Recommends adding exit conditions to reduce unnecessary iterations.
	- Encourages use of efficient built-in methods.

### Example 1: Optimising a Loop in Node.js

- Initial Approach:
	- Manual loop processing array data.
- Copilot Improvement:
	- Replace loop with higher-order functions such as `filter()` and `map()`.
- Result:
	- More concise and readable code.
	- Improved maintainability.
	- Clearer intent and easier debugging.

### Example 2: Reducing Multiple Database Calls

- Initial Approach:
	- Multiple asynchronous database calls for individual users.
- Issue:
	- Increased latency and unnecessary repetition.
- Copilot Refactoring:
	- Retrieve data in a single query.
	- Use in-memory methods like `find()` to access specific users.
- Result:
	- Fewer database calls.
	- Reduced overhead.
	- Improved application performance.

## Code Patterns & Best Practices

- Purpose:
	- Create scalable and maintainable architecture.
- How Copilot Helps:
	- Suggests design patterns such as dependency injection.
	- Encourages consistent naming conventions.
	- Promotes modular, decoupled code structures.

### Example: Applying Dependency Injection

- Initial Structure:
	- Direct database calls within application logic.
- Copilot Suggestion:
	- Create a `DatabaseService` class.
	- Inject dependencies rather than hard-coding them.
- Result:
	- Improved modularity.
	- Better testability.
	- Enhanced scalability for larger systems.
