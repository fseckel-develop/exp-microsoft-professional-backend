## Detecting Inefficient Loops

One of the most common performance bottlenecks comes from inefficient loops.

### Problem: Nested Loops

- Nested loops often cause unnecessary iterations.
- Each outer loop iteration may trigger a full scan of another collection.
- This significantly increases time complexity (e.g., O(n²)).
- Performance degrades rapidly as datasets grow.

### Example Scenario

A program calculates total sales per product:

- Outer loop → iterates through products.
- Inner loop → scans all sales records for matches.
- This causes repeated full scans of the dataset.

### Copilot’s Suggestion

- Replace nested loops with direct dictionary lookups.
- Use methods like `ContainsKey`.
- Retrieve values directly instead of iterating.

### Resulting Benefits

- Reduced complexity
- Fewer comparisons
- Faster execution
- Cleaner, more maintainable logic

## Reducing Excessive I/O Operations

Excessive file or database access can severely impact performance.

### Problem: Multiple File Reads

An inefficient implementation:

- Calls `File.ReadAllText()` separately for each file.
- Performs multiple blocking file system operations.
- Scales poorly as file count increases.

### Copilot’s Recommendation

- Process files within a single loop.
- Batch operations where possible.
- Reduce repeated file system access.

### Benefits

- Fewer blocking I/O operations
- Reduced overhead
- Improved responsiveness
- Cleaner structure

Copilot’s answers improve significantly when prompts are specific. 
The clearer the question, the more targeted the optimisation suggestion.

## Simplifying Redundant Conditional Logic

Redundant condition checks can clutter code and reduce clarity.

### Problem: Repeated If Statements

Example scenario:

- Multiple `if` conditions checking price thresholds.
- Repeated logic across conditions.
- Harder to maintain and extend.

### Copilot Suggestions

- Use `if-else` to simplify structure.
- Replace chained conditionals with a `switch` expression.
- Combine related logic into cleaner structures.

### Outcome

- More readable logic
- Less duplication
- Easier future modifications
- Same functionality, improved clarity

- Recommends removing unnecessary imports.
