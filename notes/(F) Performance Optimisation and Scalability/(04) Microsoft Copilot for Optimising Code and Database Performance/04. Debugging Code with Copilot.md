## Overview

Debugging is like navigating through a dense forest. Some paths may be blocked, but with the right strategies, you can find a clear path forward. With Microsoft Copilot, the debugging process becomes streamlined, helping you identify and resolve errors efficiently. 

**Goal:** Use Copilot to detect and fix syntax errors, logic flaws, null references, and unhandled exceptions in application code.

---
## Common Debugging Strategies

### 1. Syntax Errors

- **Definition:** Errors that break the rules of the programming language, preventing code execution.
- **Example:** Missing semicolons in C#.
- **Copilot Role:** Detects the location of syntax errors and suggests corrections.

### 2. Debugging Tools

- **Print Statements:** Verify variable updates during execution.
- **Breakpoints:** Pause execution to inspect variable values.
- **Copilot Role:** Suggests specific checks, highlighting potential problematic variables and conditions.

### 3. Logic Flaws

- **Definition:** Mistakes in reasoning causing incorrect program results without crashing.
- **Example:** A shopping cart applies a 10% discount instead of 20%.
- **Copilot Role:** Analyses the code and suggests adjustments to align logic with intended behaviour.

### 4. Null Reference Errors

- **Definition:** Occurs when a program accesses an uninitialised object or variable.
- **Example:** Accessing a product’s price before creating the product object.
- **Copilot Role:** Detects uninitialised variables and recommends proper initialisation.

### 5. Unhandled Exceptions

- **Definition:** Errors triggered by unforeseen events that are not managed by code.
- **Example:** Accessing a file that does not exist.
- **Copilot Role:** Suggests error handling using `try-catch` blocks or pre-condition checks (e.g., `File.Exists`) to prevent crashes.

## Copilot-Assisted Debugging in Action

### Sample Scenario

A console application contains four methods with deliberate errors:

- Null Reference Error
- Unhandled File Exception
- Logic Flaw in Discount Calculation
- Division by Zero

### Using Copilot for Debugging

#### 1. Null Reference Error

**Prompt:** "This method throws a null reference exception. Can you suggest a fix?"  

**Copilot Suggestion:** Check for null before accessing the object.  

```csharp
if (message != null)
{
    Console.WriteLine(message.Length);
}
else
{
    Console.WriteLine("Message is null");
}
```

**Result:** Prevents crash and outputs meaningful information.

---
#### 2. Unhandled File Exception

**Prompt:** "The code throws a file not found exception. How can I fix it?" 

**Copilot Suggestion:** Use `File.Exists` to check if the file exists before reading.  

```csharp
if (File.Exists(filePath))
{
    Console.WriteLine(File.ReadAllText(filePath));
}
else
{
    Console.WriteLine("File does not exist");
}
```

**Result:** Avoids unhandled exceptions and provides informative output.

---
#### 3. Logic Flaw in Discount Calculation

**Prompt:** "This method should apply a 20% discount for prices over $100 but only applies 10%. Fix the logic."  

**Copilot Suggestion:** Adjust discount logic to the correct threshold.  

```csharp
decimal price = 135M;
if (price > 100)
{
    price *= 0.8M; // Apply 20% discount
}
```

**Result:** Correct discount applied.

---
#### 4. Division by Zero

**Prompt:** "This method throws a division by zero exception. How can I prevent it?"  

**Copilot Suggestion:** Validate the divisor before performing division.  

```csharp
if (divisor != 0)
{
    var result = dividend / divisor;
}
else
{
    Console.WriteLine("Error: Division by zero is not allowed");
}
```

**Result:** Program safely handles edge cases.

## Best Practices When Using Copilot for Debugging

1. **Prompt Clearly:** Provide context about the error, including error messages when available.
2. **Validate Suggestions:** Always review Copilot’s suggestions to ensure logic aligns with requirements.
3. **Test Thoroughly:** Run all possible execution paths after applying fixes.
4. **Iterative Debugging:** Continue using Copilot to address remaining errors until the application runs without issues.
