## Introduction

This guide provides step-by-step instructions for configuring Redis and SQL Server caching in a .NET Core environment and troubleshooting tips.

---
## A. Setting Up Redis Caching

#### Step 1: Install Redis

- Download and install Redis from [https://redis.io/download](https://redis.io/download).
- Start Redis using the command:

```Terminal
redis-server
```

#### Step 2: Add Redis Package in .NET Core

- Install the required package via NuGet:

```Terminal
dotnet add package Microsoft.Extensions.Caching.StackExchangeRedis
```

#### Step 3: Configure Redis in appsettings.json

- Add the Redis connection settings:

```JSON
{
  "Redis": {
    "ConnectionString": "localhost:6379"
  }
}
```

#### Step 4: Configure Redis in Program.cs

- Register Redis caching in the dependency injection container:

```C#
builder.Services.AddStackExchangeRedisCache(options =>
{
    options.Configuration = builder.Configuration["Redis:ConnectionString"];
});
```

#### Step 5: Use Redis Caching in Your Application

- Example of setting and retrieving cache data:

```C#
public class CacheService
{
    private readonly IDistributedCache _cache;
    
    public CacheService(IDistributedCache cache)
    {
        _cache = cache;
    }

    public async Task 
    SetCacheAsync(string key, string value, TimeSpan expiration)
    {
        var options = new DistributedCacheEntryOptions
        {
            AbsoluteExpirationRelativeToNow = expiration
        };
        await _cache.SetStringAsync(key, value, options);
    }

    public async Task<string> GetCacheAsync(string key)
    {
        return await _cache.GetStringAsync(key);
    }
}
```

## B. Setting Up SQL Server Caching

#### Step 1: Install SQL Server Caching Package

- Install the necessary NuGet package:

```Terminal
dotnet add package Microsoft.Extensions.Caching.SqlServer
```

#### Step 2: Configure SQL Server Caching in appsettings.json

- Add SQL Server connection settings:

```JSON
{
  "SqlServerCache": {
    "ConnectionString": "Server=your_server;Database=your_db;User Id=your_user;Password=your_password;"
  }
}
```

#### Step 3: Register SQL Server Caching in Program.cs

- Add SQL Server caching service:

```C#
builder.Services.AddDistributedSqlServerCache(options =>
{
    options.ConnectionString = 
	    builder.Configuration["SqlServerCache:ConnectionString"];
    options.SchemaName = "dbo";
    options.TableName = "Cache";
});
```

#### Step 4: Use SQL Server Caching in Your Application

- Example of setting and retrieving cache data:

```C#
public async Task 
SetSqlCacheAsync(string key, string value, TimeSpan expiration)
{
    var options = new DistributedCacheEntryOptions
    {
        AbsoluteExpirationRelativeToNow = expiration
    };
    await _cache.SetStringAsync(key, value, options);
}

public async Task<string> GetSqlCacheAsync(string key)
{
    return await _cache.GetStringAsync(key);
}
```

## Troubleshooting Common Issues

### Redis Issues

- Redis is not starting 
  → Ensure the port (default 6379) is not in use or change it in the 'redis.conf' file
- Connection refused error 
  → Make sure Redis is running by executing redis-server.

### SQL Server Caching Issues

- Cache table is missing 
  → Ensure the SQL Server cache table is created before using caching.
- Timeout expired error 
  → Check that SQL Server is running, and verify the connection string credentials.

## Conclusion

Redis and SQL Server caching in .NET Core helps improve application performance by reducing database load and speeding up data retrieval. Redis is ideal for frequently changing data, like session storage, while SQL Server caching is better suited for long-lived data, like product catalogs. Proper configuration and troubleshooting ensure smooth integration and optimal performance.
