## Main Characteristics

- Definition  
	- Simple comparison-based sorting algorithm  
	- Repeatedly compares adjacent elements  
	- Swaps elements if they are in the wrong order  
	- Continues until no swaps are required  

- Core Idea  
	- Larger elements “bubble” toward the end of the list each pass  
	- Each iteration places one element in its final position  

---
## Mechanics

- Use nested loops  
- Start from the beginning of the list  
- Compare element`[i]` with element`[i+1]`
- Swap if out of order  
- Repeat full passes  
- Stop early if no swaps occur  
- Operates in place  
- No recursion required  

---
## Complexity

- Time Complexity  
	- Worst case: O(n²)  
	- Average case: O(n²)  
	- Best case (already sorted): O(n)  

- Space Complexity  
	- O(1)  
	- Constant extra memory  
	- In-place sorting  

---
## Stability & Characteristics

- Stable sorting algorithm  
- Very simple to implement  
- Performance degrades rapidly as input grows  
- Inefficient for medium to large datasets  

---
## Suitable Use Cases

- Very small datasets  
- Nearly sorted datasets  
- Educational demonstrations  
- Memory-constrained environments  
- Situations where simplicity is more important than speed  

---
## Trade-offs

- Advantages  
	- Easy to understand  
	- Minimal memory usage  
	- Can detect already sorted data  

- Disadvantages  
	- Extremely slow at scale  
	- Rarely used in production systems  
	- Poor scalability
