## Main Characteristics

- Definition  
	- Divide-and-conquer sorting algorithm  
	- Recursively splits list into halves  
	- Merges sorted halves  

- Core Idea  
	- Divide → Sort → Merge  

---
## Mechanics

- Recursively split array into halves  
- Continue until single-element lists  
- Merge sublists in sorted order  
	- Compare elements sequentially  
	- Build new sorted array  
- Uses recursion  
- Requires temporary arrays  

---
## Complexity

- Time Complexity  
	- Best case: O(n log n)  
	- Average case: O(n log n)  
	- Worst case: O(n log n)  

- Space Complexity  
	- O(n)  
	- Requires additional memory for merging  

---
## Stability & Characteristics

- Stable sorting algorithm  
- Predictable performance  
- Not in-place (standard implementation)  
- Performs well on linked lists  

---
## Suitable Use Cases

- Large datasets  
- Stable sorting requirements  
- External sorting (disk-based data)  
- Linked lists  
- When predictable performance is critical  

---
## Trade-offs

- Advantages  
	- Guaranteed O(n log n)  
	- Stable  
	- Reliable performance  

- Disadvantages  
	- Requires extra memory  
	- Slower than QuickSort in many in-memory cases
