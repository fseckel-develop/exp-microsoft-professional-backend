## Main Characteristics

- Definition  
	- Divide-and-conquer sorting algorithm  
	- Selects a pivot element from the list  
	- Partitions elements into two groups  
	- Recursively sorts the partitions  

- Core Idea  
	- Partition → Recursively sort subarrays  
	- Elements smaller than pivot move left  
	- Elements larger than pivot move right  

---
## Mechanics

- Choose a pivot element  
	- Common strategies  
		- First element  
		- Last element  
		- Random element  
		- Median-of-three  

- Partition the array  
	- Compare each element with pivot  
	- Move smaller elements before pivot  
	- Move larger elements after pivot  

- Place pivot in its correct final position  

- Recursively apply Quick Sort to  
	- Left partition  
	- Right partition  

- Recursion stops when subarray size ≤ 1  

- Operates mostly in place  

---
## Complexity

- Time Complexity  
	- Best case: O(n log n)  
	- Average case: O(n log n)  
	- Worst case: O(n²)  
		- Occurs with poor pivot selection (e.g., already sorted data with naive pivot)

- Space Complexity  
	- O(log n) average (due to recursion stack)  
	- O(n) worst case recursion depth  

---
## Stability & Characteristics

- Not a stable sorting algorithm (standard implementation)  
- In-place sorting algorithm  
- Very cache-efficient  
- Performance depends on pivot choice  
- One of the fastest general-purpose sorting algorithms in practice  

---
## Suitable Use Cases

- Large in-memory datasets  
- General-purpose sorting  
- Situations where average-case performance matters most  
- Systems requiring fast practical performance  

---
## Trade-offs

- Advantages  
	- Very fast in practice  
	- Average O(n log n) performance  
	- In-place sorting (low memory overhead)  
	- Excellent cache performance  

- Disadvantages  
	- Worst case O(n²)  
	- Not stable  
	- Performance sensitive to pivot selection  
	- Recursive implementation can cause deep recursion in worst cases
