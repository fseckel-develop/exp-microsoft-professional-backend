## Characteristics

- Definition  
	- Sequential search algorithm  
	- Checks each element one by one  
	- Stops when target is found or list ends  

- Core Principle  
	- Start at index 0  
	- Compare current element with target  
	- Move to next element  
	- Repeat until match or end of list  

---
## Implementation Logic

- Iterate through array using loop  
	- Compare element at index i with target  
	- If match → return index  
	- If loop completes → return -1 (not found)  
- Zero-based indexing  
	- First element at index 0  
	- Position reflects array index  

---
## Complexity

- Time Complexity: O(n)  
	- Worst case: target is last element or not present  
	- Requires checking each element once  
- Space Complexity: O(1)  
	- No additional data structures  
	- Constant extra memory usage  

---
## Suitable Use Cases

- Small datasets  
- Unsorted lists  
- Situations where sorting is unnecessary  
- Structured data with limited variability  
- Memory-constrained environments  

---
## Advantages & Trade-offs

- Advantages  
	- Very simple to implement  
	- Works on unsorted data  
	- Minimal memory usage  
- Disadvantages  
	- Inefficient for large datasets  
	- Performance decreases linearly as data grows  

---
## Key Insight

- Performance scales directly with input size  
- Best choice when simplicity and low overhead are more important than speed  
- Often used as foundational example before introducing more advanced search algorithms
