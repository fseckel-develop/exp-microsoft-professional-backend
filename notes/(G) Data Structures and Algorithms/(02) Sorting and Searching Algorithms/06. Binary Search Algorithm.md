## Characteristics

- Definition  
	- Divide-and-conquer search algorithm  
	- Repeatedly splits a sorted dataset in half  
	- Narrows search range until target is found or eliminated  

- Core Principle  
	- Requires sorted data  
	- Compare target with middle element  
	- Discard half of remaining elements each step  
	- Repeat until match or search range is empty  

---
## Preconditions

- Dataset must be sorted  
- Random access structure required (e.g., array)  
- Not suitable for unsorted data without prior sorting  

---
## Algorithm Steps

- Initialise boundaries  
	- left = 0  
	- right = array length - 1  
- While left ≤ right  
	- Calculate midpoint  
		- mid = left + (right - left) / 2  
	- If array[mid] == target → return mid  
	- If array[mid] < target → search right half  
		- left = mid + 1  
	- If array[mid] > target → search left half  
		- right = mid - 1  
- If not found → return -1  

---
## Complexity

- Time Complexity: O(log n)  
	- Search space halves each iteration  
	- Efficiency increases significantly for large datasets  
- Space Complexity: O(1)  
	- Uses only a few pointer variables (left, right, mid)  
	- Memory usage remains constant regardless of input size  

---
## Suitable Use Cases

- Large datasets  
- Sorted arrays or lists  
- High-performance search requirements  
- Systems where speed is critical  

---
## Advantages & Trade-offs

- Advantages  
	- Extremely fast for large datasets  
	- Predictable logarithmic performance  
	- Minimal memory usage  
- Disadvantages  
	- Requires sorted data  
	- Sorting step adds additional computational cost  
	- Less flexible than linear search for unsorted data  

---
## Comparison Insight

- Compared to Linear Search  
	- Linear search checks every element sequentially  
	- Binary search eliminates half the dataset per step  
	- Performance gap widens as dataset size increases  

---
## Key Insight

- Performance scales logarithmically with input size  
- Ideal when working with large, pre-sorted datasets  
- Significantly outperforms linear search at scale
