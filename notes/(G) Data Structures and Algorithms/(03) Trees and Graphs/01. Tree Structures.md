
- Tree structures are **hierarchical data structures** designed for efficient storage and retrieval   
- They create **logical pathways** through complex data
- Enable scalability, fast access, and structured organisation
- Key types:
    - Binary Trees
    - Binary Search Trees (BSTs)
    - AVL Trees (self-balancing BSTs)

---
## Tree Data Structure Fundamentals

- Hierarchical structure with **parent–child relationships**
- Consists of:
    - Root node (top-level starting point)
    - Parent nodes (connect downward)
    - Child nodes (descend from parents)
- Each level connects logically to the next
- Supports structured navigation instead of sequential scanning

---
## Binary Trees

- Definition:
    - A tree where each node has **at most two children**
    - Children are referred to as **left** and **right**
- Key characteristics:
    - Simple branching structure
    - Clear two-way decision model

- Advantages:
    - Simplifies hierarchical organisation
    - Ideal for binary decisions (e.g., yes/no logic)

- Backend applications:
    - Expression parsing in compilers (correct operator precedence)
    - File system hierarchies in operating systems
    - Structured navigation of directories

- Limitation:
    - No inherent ordering → does not guarantee fast searching

---
## Binary Search Trees (BST)

- Definition:
    - A binary tree with a **strict ordering rule**
- Ordering property:
    - Left subtree values < parent node
    - Right subtree values > parent node
- Structural consequence:
    - Data remains **sorted implicitly**

- Advantages:
    - Efficient searching
    - Efficient insertion and deletion
    - Narrowed search space at each step

- Real-world applications:
    - Search engine indexing
    - Alphabetically ordered contact lists
    - Ordered data management systems

- Limitation:
    - Can become **unbalanced**
    - Unbalanced growth reduces performance

---
## AVL Trees (Self-Balancing BSTs)

- Definition:
    - A self-balancing Binary Search Tree
- Balance condition:
    - Height difference between left and right subtrees ≤ 1
- Mechanism:
    - Performs **rotations** after insertions or deletions
    - Automatically restores structural balance

- Advantages:
    - Prevents one-sided growth
    - Maintains consistent search time
    - Guarantees predictable performance

- Backend applications:
    - Database indexing
    - Memory allocation management in operating systems    
    - High-scale systems (e-commerce platforms, large datasets)

---
## Conceptual Comparison

- Binary Tree:
    - Basic hierarchical structure
    - No ordering guarantee

- Binary Search Tree:
    - Adds ordering rule
    - Enables efficient search
    - May degrade if unbalanced

- AVL Tree:
    - Adds automatic balancing
    - Ensures stable and consistent performance

---
## Core Insight

- Tree structures transform data storage from **linear scanning** to **structured navigation**
- Binary trees provide structure
- BSTs provide ordered efficiency
- AVL trees provide sustained performance under growth
- Essential for scalable backend system design
