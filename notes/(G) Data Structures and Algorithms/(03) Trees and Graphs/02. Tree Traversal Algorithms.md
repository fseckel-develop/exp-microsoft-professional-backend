## Core Traversal Types

### Preorder Traversal (Root → Left → Right)

-	Visit order:
	-	Process root node first  
	-	Traverse left subtree  
	-	Traverse right subtree  
-	Priority:
	-	Root node is handled before its children  
-	Use cases:
	-	Creating backups of folder structures  
	-	Copying hierarchical systems  
	-	Scenarios where parent processing must occur first  
-	Conceptual benefit:
	-	Preserves structural hierarchy from top down  

### Inorder Traversal (Left → Root → Right)

-	Visit order:
	-	Traverse left subtree  
	-	Process root node  
	-	Traverse right subtree  
-	Key property in BST:
	-	Returns values in ascending (sorted) order  
-	Use cases:
	-	Retrieving sorted data from Binary Search Trees  
	-	Sequential data processing without extra sorting  
-	Conceptual benefit:
	-	Leverages inherent ordering of BSTs for efficiency  

### Postorder Traversal (Left → Right → Root)

-	Visit order:
	-	Traverse left subtree  
	-	Traverse right subtree  
	-	Process root node last  
-	Priority:
	-	Children are processed before their parent  
-	Use cases:
	-	Deleting directory structures  
	-	Handling dependencies before parent removal  
-	Conceptual benefit:
	-	Ensures safe bottom-up processing  

### Level-Order Traversal (Breadth-First Search – BFS)

-	Visit order:
	-	Process nodes level by level  
	-	Move from top to bottom  
-	Traversal type:
	-	Breadth-first approach  
-	Use cases:
	-	Organizational hierarchies  
	-	Network broadcasting  
	-	Layer-based processing systems  
-	Conceptual benefit:
	-	Preserves structural depth order

### Traversal Implementation

-	Depth-first traversals:
	-	Preorder  
	-	Inorder  
	-	Postorder  
	-	Typically implemented using recursion  
-	Level-order traversal:
	-	Implemented using a queue  
	-	Processes nodes iteratively

	-	Parent-first processing

	-	Sorted output in BST  
-	Postorder:
	-	Child-first processing  
-	Level-order:
	-	Layer-by-layer processing
