
-	Graphs are **data structures** used to represent connections between items  
-	Components:
	-	Nodes (points representing items like tasks or people)  
	-	Edges (lines representing relationships between nodes)  
-	Graphs help organise relationships in:
	-	Project task sequences  
	-	Social networks  
	-	Backend data models  

---
## Types of Graphs

### Directed Graphs

-	Edges have a **specific direction**, representing one-way relationships  
-	Visit order flows from source node to target node  
-	Use cases:
	-	Project management task sequencing  
	-	Workflow processes  
	-	One-way relationships on social platforms (e.g., following on X)  
-	Conceptual benefit:
	-	Clearly shows logical progression or dependencies  

### Undirected Graphs

-	Edges have **no specific direction**, representing mutual relationships  
-	Connections are **bidirectional**  
-	Use cases:
	-	Friendship networks  
	-	Shared interests or mutual connections  
	-	Mutual relationships on social platforms (e.g., Facebook friendships)  
-	Conceptual benefit:
	-	Represents equality and reciprocity in relationships  

---
## Real-World Applications

-	Social media platforms:
	-	Directed graphs: one-sided follows  
	-	Undirected graphs: mutual friendships  
-	Project planning:
	-	Directed graphs: ordered task dependencies  
-	General data modelling:
	-	Graphs provide clear, logical structure for connections  
-	Combining directed and undirected graphs:
	-	Models complex interactions in real-world systems  

---
## Core Insight

-	Understanding **graph types** allows accurate modelling of relationships  
-	Directed graphs for **ordered or dependent connections**  
-	Undirected graphs for **mutual or equal connections**  
-	Essential for organising information efficiently and logically
