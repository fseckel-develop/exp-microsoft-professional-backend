
-	Graph traversal algorithms are methods to **visit nodes in a graph systematically**  
-	Two fundamental techniques:
	-	Depth-First Search (DFS)  
	-	Breadth-First Search (BFS)  
-	Applications include:
	-	Pathfinding  
	-	Network analysis  
	-	Solving puzzles and graph-based problems  

---
## Depth-First Search (DFS)

-	Traversal strategy:
	-	Explore as far as possible along each branch before backtracking  
-	Implementation:
	-	Recursive DFS using a hash set to track visited nodes  
	-	Iterative DFS using a stack  
-	Algorithm steps:
	-	Check if the current node is visited  
	-	If not, process the node  
	-	Add node to visited set  
	-	Recurse through or push neighbours for further exploration  
-	Use cases:
	-	Pathfinding (e.g., mazes)  
	-	Topological sorting in Directed Acyclic Graphs (DAGs)  
	-	Finding connected components in a graph  

---
## Breadth-First Search (BFS)

-	Traversal strategy:
	-	Visit all neighbours at the current level before moving deeper  
-	Implementation:
	-	Use a queue to track nodes at the current level  
	-	Hash set to track visited nodes  
-	Algorithm steps:
	-	Enqueue the starting node  
	-	While the queue is not empty:
		-	Dequeue node and process it  
		-	Enqueue unvisited neighbours  
-	Use cases:
	-	Finding shortest path in unweighted graphs  
	-	Web crawling and search engine indexing  
	-	Network broadcasting and spreading updates efficiently

## Implementation Concepts (.NET Core)

### Graph Setup

-	Define a `Graph` class with an **AdjacencyList dictionary**  
-	Methods:
	-	`AddEdge` → Adds connections between nodes  
	-	`GetGraph` → Retrieves the current adjacency list  

### DFS Implementation

-	`DFS(graph, start, visited)`:
	-	Recursive call through neighbours  
	-	Add nodes to visited hash set  
	-	Console output shows traversal order  
-	Iterative DFS:
	-	Uses a stack to simulate recursion  

### BFS Implementation

-	`BFS(graph, start)`:
	-	Queue maintains nodes at the current level  
	-	Hash set tracks visited nodes  
	-	Process nodes level by level  

---
## Core Insight

-	DFS is ideal for **deep exploration** and ordering problems  
-	BFS is ideal for **shortest-path calculations** and level-wise processing  
-	Choosing the right traversal method **depends on the use case**  
-	Mastering both techniques enables efficient graph navigation and problem-solving
