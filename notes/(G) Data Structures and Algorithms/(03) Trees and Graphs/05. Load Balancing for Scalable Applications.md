
-	Load balancing distributes **incoming requests across multiple servers**  
-	Ensures **stability, efficiency, and reliability** in scalable systems  
-	Key purpose:
	-	Prevent server overload  
	-	Enhance system availability  
	-	Maintain consistent user experience under high traffic  

---
## Key Concepts of Load Balancing

### Load Balancing Definition

-	Technique for **evenly distributing network traffic** across servers  
-	Optimizes:
	-	System stability  
	-	Performance  
	-	Responsiveness  

### Server Pool

-	A **group of servers** sharing incoming requests  
-	Requests are directed to the **best available server**  
-	Analogy: Restaurant chefs receiving orders efficiently  
-	Ensures no single server becomes a bottleneck  

### Single Point of Failure

-	A component whose failure **can crash the entire system**  
-	Load balancing **minimizes risk** by distributing traffic  
-	Analogy: Multiple airport security checkpoints prevent passenger delays  

### Health Checks

-	Regular tests to confirm servers are **ready to handle requests**  
-	Only healthy servers receive traffic  
-	Analogy: Checking athletes’ readiness before a game  

### Latency and Throughput

-	**Latency:** Time delay between user request and system response  
	-	Low latency → faster response  
	-	High latency → slower system and reduced throughput  
-	**Throughput:** Number of requests a system processes in a given time  
	-	Optimized when latency is minimized and workload is evenly distributed  
-	Load balancing **reduces latency** and **maximises throughput** by directing traffic efficiently

## Load Balancing Algorithms

### Round Robin

-	Requests are assigned to **servers in a fixed loop**  
-	Suitable when servers have **similar capacity and performance**  

-	Implementation:
	-	Maintain an index for the current server  
	-	Assign request to the server at the index  
	-	Increment index modulo the number of servers  
-	Example behaviour:
	-	Server A → Server B → Server C → Server A → …
	  
-	Pros: Simple, evenly distributes load  
-	Cons: Does not account for server capacity or current load  

### Least Connections

-	Requests are assigned to the **server with the fewest active connections**  
-	Optimal for servers with **variable workloads**  

-	Implementation:
	-	Track number of active connections per server in a dictionary  
	-	Select server with minimum connections for new request  
	-	Increment connection count upon assignment  
-	Example behaviour:
	-	Servers preloaded with connections: A=7, B=9, C=5  
	-	Request routed first to C (least connections), then A, then C again 
 
-	Pros: Balances load dynamically based on real-time usage  
-	Cons: Slightly more complex than round robin  

### Weighted Round Robin

-	Similar to round robin but assigns **weights to servers** based on capacity  
-	Higher weight servers receive **more requests**  

-	Implementation:
	-	Assign weight to each server  
	-	Loop through servers according to their weights  
-	Example behaviour:
	-	Server weights: A=2, B=1, C=3  
	-	Requests favour higher-weight servers (more powerful servers get more load)  

-	Pros: Accounts for differences in server performance  
-	Cons: Requires proper weight configuration  

### IP Hashing

-	Assigns requests to servers **based on a hash of client IP**  
-	Ensures a **consistent server assignment** for the same client  

-	Implementation:
	-	Compute hash of IP address  
	-	Modulo hash with number of servers to get index  
	-	Assign request to server at that index  
-	Example behaviour:
	-	IP 192.168.1.1 → Server A consistently  
	-	IP 192.168.1.2 → Server B consistently  

-	Pros: Improves cache locality and performance for repeated clients  
-	Cons: Less flexible when server pool changes dynamically  

---
## Core Insights

Load balancing is **essential for scalable, high-traffic systems**. Algorithms vary in **complexity and suitability** depending on server capacity, workload, and client behaviour:  

-	Round Robin → Simple, equal distribution  
-	Least Connections → Dynamic load management  
-	Weighted Round Robin → Performance-aware distribution  
-	IP Hashing → Consistent server assignment  

Proper selection of algorithms **reduces latency, maximises throughput, and ensures reliability.**
By intelligently distributing workload, load balancing prevents overload and mitigates single points of failure
