## Introduction

Asynchronous processing allows applications to run tasks in the background without blocking the main execution thread. This improves responsiveness and scalability, ensuring applications remain fluid and efficient even under heavy workloads.

---
## Key Concepts

- **Asynchronous Processing:** 
  Enables apps to handle multiple tasks simultaneously, maintaining user interaction.

- **Thread Blocking vs. Non-blocking:** 
	- synchronous tasks force processes to wait, blocking the main execution flow
	- asynchronous tasks (non-blocking operations) allow continued execution

- **Use Cases:** Ideal for background tasks like API calls, file uploads, and data synchronisation.

- **Asynchronous Frameworks:** 
  Tools like C#’s `async`/`await` and Task Parallel Library (TPL) help manage tasks efficiently.

---
## Implementing Async Tasks in .NET Core

### Example 1: Interacting with Main while Async Task Runs

Simulate fetching data from an API while allowing user input. Users can interact with the app while the background task runs. Task completion triggers continuation without blocking the interface.

**Implementation:**

```csharp
using System;
using System.Threading.Tasks;

class Program
{
    static async Task<string> FetchDataAsync()
    {
        await Task.Delay(5000);
        return "Data retrieved successfully";
    }

    static async Task Main()
    {
        var fetchTask = FetchDataAsync();
        Console.Write("Enter your name: ");
        string name = Console.ReadLine();
        string result = await fetchTask;
        Console.WriteLine(result);
        Console.WriteLine($"Hello {name}");
    }
}
```

### Example 2: Async with Return Values and Continuations

Execution of a background operation can be started and the result can be handled later on after completion. Continuations act like callbacks, running after the async task completes.

**Implementation:**

```csharp
using System;
using System.Threading.Tasks;

class Program
{
    static async Task<string> GetMessageAsync()
    {
        await Task.Delay(1500);
        return "Hello from async world";
    }

    static void Main()
    {
        var messageTask = GetMessageAsync();
        Console.WriteLine("Doing other work...");
        messageTask.ContinueWith(t => Console.WriteLine($"Message received: {t.Result}"));
        Console.ReadLine();
    }
}
```

### Example 3: Async Error Handling

Async tasks can be wrapped in try-catch blocks for error handling similar to synchronous code.

**Implementation:**

```csharp
using System;
using System.Threading.Tasks;

class Program
{
    static async Task FetchDataWithErrorAsync()
    {
        await Task.Delay(2000);
        throw new Exception("Network failure");
    }

    static async Task Main()
    {
        try
        {
            await FetchDataWithErrorAsync();
        }
        catch (Exception ex)
        {
            Console.WriteLine($"Error: {ex.Message}");
        }
        Console.WriteLine("Execution continued after handling error");
    }
}
```

## Benefits for Scalability

- **Increased Responsiveness:** 
  Users can continue interacting with an app while tasks execute in the background.

- **Efficient Resource Utilisation:** 
  Reduces idle time, optimising CPU and memory usage.

- **Better Scalability:** 
  Supports high user loads without slowing down application performance.

## Core Insigths

Asynchronous processing is essential for building scalable applications. Developers can create fast, responsive, and high-performing systems by minimizing thread blocking and optimizing resource use. Using async frameworks like async/await and TPL enables seamless background operations, user interaction, and reliable error handling.
