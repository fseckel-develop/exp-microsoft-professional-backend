## Overview

The Knapsack problem is a classic optimisation problem used to determine the best combination of items under a fixed capacity constraint.

It models real-world backend scenarios where:

- Resources are limited (CPU, memory, bandwidth)
- Tasks have different values (priority, profit, importance)
- We must maximise total value without exceeding capacity

---
## Problem Definition (0/1 Knapsack)

- Given:
	- An array of values  
	- An array of weights  
	- A maximum capacity  
	- A number of items `n`

- Goal: Maximise total value without exceeding the capacity.

- Each item can either be:
	- Included (1)
	- Excluded (0)
- Items cannot be split.

---
## Recursive Approach

- **Base Case**:
	-  If `n == 0` or `capacity == 0` 
	  then return `0`

- **Decision Step**:
	- If the weight of the current item (index `n - 1`) is less than or equal to capacity:
	  then we choose the maximum of:
		1. Include the item  
			   - Add its value  
			   - Reduce capacity  
			   - Call function with `n - 1`
		2. Exclude the item  
			   - Call function with `n - 1`
		
	- If the weight exceeds capacity:
		- Skip the item
		- Call function with `n - 1`

### Problem with Pure Recursion

- The recursive solution recalculates the same subproblems multiple times.
- This leads to:
	- Exponential time complexity  
	- Poor performance for larger inputs  
	- Time complexity: **O(2^n)**

---
## Dynamic Programming Solution: Memoization

- To improve performance, we store results of subproblems.
- Each subproblem is defined by: `(n, capacity)`
- Before computing:
	- Check if the result already exists in a dictionary
	- If yes → return stored value
	- If no → compute and store it

### Memoization Strategy

1. Create a dictionary: 
	- `Dictionary<string, int> memo`
2. Generate a key:
	- `key = n + “-” + capacity`
3. If key exists:
	- Return stored result
4. Otherwise:
	- Compute result
	- Store it in dictionary
	- Return result

### Performance Improvement

- With memoization:
	- Time complexity:   `O(n × capacity)`
	- Space complexity: `O(n × capacity)`
- Each subproblem is solved only once.

---
## Why Knapsack Is Important in Backend Systems

The Knapsack problem models real backend resource allocation problems:

- Selecting tasks under limited CPU capacity  
- Allocating memory efficiently  
- Budget-constrained feature selection  
- Job scheduling with limited resources  

It demonstrates how dynamic programming:

- Stores intermediate results  
- Avoids redundant computation  
- Optimizes constrained resources  

---
## Key Takeaways

- Knapsack is a constrained optimization problem.
- Pure recursion is inefficient due to repeated work.
- Memoization stores computed states to improve performance.
- Dynamic programming guarantees optimal results for 0/1 Knapsack.
- The concept directly applies to backend resource scheduling and optimisation.
