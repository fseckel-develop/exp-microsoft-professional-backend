## Introduction

- Hashing improves search efficiency in back-end applications  
	- Converts data into fixed-size, unique hash values  
	- Enables constant-time data retrieval  

- Hash tables leverage hashes for fast lookups  
	- Optimise performance in authentication, caching, and session management  
	- Enhance security and scalability  

---
## Key Concepts of Hashing

- **Hashing**  
	- Converts data (e.g., passwords, session IDs, keys) into unique hash values  
	- Uses hash functions to produce fixed-size codes for quick identification  
	- Enables fast retrieval, indexing, and comparison  

- **Hash Tables**  
	- Store data as key-value pairs  
	- Use hash values as keys to map data locations  
	- Allow near-constant time lookup, insertion, and deletion  

- **Collision Handling**  
	- Resolves cases where different inputs generate the same hash value  
	- Techniques include:  
		- **Chaining** – store colliding entries in a linked list at the same slot  
		- **Open Addressing** – find an alternative empty slot using linear or quadratic probing 
		- **Double Hashing** – use a secondary hash function to calculate the next slot for more even distribution  

---
## Applications in Back-End Systems

- **Session Management**  
	- Hash session IDs for secure storage and retrieval  
	- Prevents tampering and hijacking  
	- Ensures fast lookup of user session data (preferences, authentication tokens)  

- **Caching**  
	- Hash keys map cached items for rapid access  
	- Reduces server load and latency  
	- Supports high-performance retrieval in large datasets  

- **Password Hashing**  
	- Converts passwords into irreversible hash values  
	- Stored hashed values prevent exposure of plain-text passwords  
	- Login verification done by comparing hashed input to stored hash  

---
## Benefits of Hashing in Back-End

- Enables fast and efficient searches  
- Organises data for quick access and retrieval  
- Protects sensitive data from direct exposure  
- Enhances scalability and performance under high load  
- Reduces computational overhead for frequent lookups

## Conclusion

- Hash tables are essential for back-end optimisation  
	- Facilitate constant-time access to data  
	- Improve efficiency in searching, indexing, and data retrieval  
	- Enhance security through irreversible hashing of sensitive information  

- Leveraging hashing improves:  
	- System performance  
	- Reliability  
	- Scalability in modern applications
