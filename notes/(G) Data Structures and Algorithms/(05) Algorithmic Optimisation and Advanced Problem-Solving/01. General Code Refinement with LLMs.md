## Overview

This note explains how developers can use large language models (LLMs), such as GitHub Copilot, to analyse existing implementations and improve the efficiency of data structures and algorithms. AI-assisted refactoring can help identify inefficiencies and guide improvements that lead to more maintainable and scalable backend code.

---
## Workflow for AI-Assisted Code Improvement

### 1. Locate Performance Issues

Start by identifying areas in the code that may reduce overall performance.

Typical indicators include:

- Inefficient loops or repeated computations
- Suboptimal data structures
- High memory usage
- Slow algorithmic operations

These sections of code are good candidates for optimisation.

### 2. Query the LLM for Improvement Ideas

Use focused prompts when interacting with the AI.

Examples:

- "How could this search function be optimised?"
- "Is there a more efficient data structure for this task?"

Clear and specific prompts generally produce more useful suggestions.

### 3. Evaluate the Proposed Changes

AI-generated recommendations should always be reviewed carefully.

Consider:

- Whether the suggestion actually improves performance
- If the code remains readable and maintainable
- Whether the change introduces new complexity or potential errors

Human validation remains essential.

### 4. Implement Refactoring Gradually

Apply improvements step by step rather than replacing large portions of code at once.

Best practices include:

- Preserving the existing behaviour of the program
- Keeping changes small and traceable
- Using version control to track modifications

This approach makes debugging and validation easier.

### 5. Verify Results Through Testing

After implementing changes, confirm that the optimisation produced measurable improvements.

Validation methods include:

- Running automated tests
- Measuring execution time
- Observing memory consumption
- Comparing results before and after the refactor

The goal is to ensure that functionality remains correct while performance improves.

---
## Example Scenario: Improving a Priority Queue

A practical example involves optimising a priority queue implementation.

Initial setup:

- Open an existing priority queue implementation in a development environment such as Visual Studio Code.
- Assume the implementation works correctly but may not scale efficiently with large datasets.

Optimisation process:

1. Request suggestions from Copilot on improving performance.
2. The AI may recommend replacing a list-based structure with a more efficient alternative such as a binary heap.
3. Update the implementation to incorporate the new structure.
4. Adjust insertion and removal logic accordingly.

Final step:

- Test the updated implementation to verify correct ordering behaviour.
- Benchmark the performance to confirm improvements.
- Document the reasoning behind the refactor.

---
## Key Takeaways

- LLM tools can assist in analysing and improving existing code.
- Targeted prompts help generate more relevant optimisation suggestions.
- Developers must review and validate AI-generated recommendations.
- Performance improvements should always be verified through testing and benchmarking.

Combining developer expertise with AI assistance can significantly improve code quality and long-term maintainability.
