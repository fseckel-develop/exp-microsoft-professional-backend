## Core Infrastructure and Application Services

### Virtual Machines (VMs)

- Software-based simulations of physical computers  
	- Run operating systems and applications independently of host hardware  
- Provide flexibility and scalability  
	- Suitable for cloud computing, software testing, and disaster recovery  
	- Allow easy replication, backup, and migration  
- Enable cost-efficient environment testing without physical hardware investment  

### App Services (PaaS)

- Fully managed Platform-as-a-Service (PaaS) for web apps and APIs  
- Simplifies building, deploying, and scaling applications  
- Automatically adjusts resources based on demand  
	- Ensures smooth performance during traffic spikes  
	- Reduces operational complexity  

### Azure Functions (Serverless Computing)

- Event-driven, server-less compute service  
- Executes code only when triggered  
	- No continuous hosting required  
	- No server management necessary  
- Automatically scales based on demand  
- Ideal for automation tasks  
	- Data transformation  
	- Validation  
	- Service integrations  

### Resource Groups

- Logical containers for organizing related Azure resources  
	- Virtual machines  
	- Storage accounts  
	- Databases  
- Enable unified management of resources as a single unit  
- Simplify tracking, updating, and lifecycle management

## Scalability and Reliability Mechanisms

### Auto-Scaling

- Automatically adjusts computing resources based on real-time demand  
	- Scales up during peak usage  
	- Scales down during low demand  
- Maintains performance while optimizing costs  

### Azure Virtual Machine Scale Sets

- Deploy and manage groups of identical, load-balanced virtual machines  
- Support automatic scaling aligned with application demand  
- Ensure consistent performance during predictable traffic spikes  

### Load Balancing

- Distributes incoming network traffic across multiple servers  
- Prevents individual server overload  
- Routes traffic to healthy instances automatically  
- Enhances availability and performance  

### Service Level Agreements (SLAs)

- Formal guarantees of service availability and performance  
- Typical uptime commitments between 99.9% and 99.99%  
- Provide service credits if availability targets are not met  
- Offer reliability assurance for business-critical applications
