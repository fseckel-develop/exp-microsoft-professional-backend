## Local Application Preparation with Visual Studio Code

### Environment Variables

- Key-value pairs storing configuration settings  
	- Prevent hard-coding sensitive data (e.g., API keys)  
	- Support environment-specific configurations  
- Use `.env` files during development  
	- Store sensitive credentials separately from source code  
- Install required libraries (e.g., `.NET` env package)  
- Load variables at runtime using `Env.Load()`  
	- Makes variables accessible to the application  
	- Reduces exposure risk in repositories and logs  

### Dependency Management

- Dependencies are pre-built libraries that extend functionality  
	- Example: JSON handling libraries  
- Managed using tools like NuGet  
	- Add packages with `dotnet add package`  
	- Ensure consistency using `dotnet restore`  
- Ensures compatibility, maintainability, and stable builds  

### Build Configuration

- Defines how the application is compiled and packaged  
	- Target environment  
	- Optimisation settings  
	- Dependencies and scripts  
- Compile application using `dotnet build`  
	- Generates output in `/bin` directory  
- Review build output for errors or warnings  
- Enables local testing before deployment

## Cloud-Specific Configuration for Azure

### Application Settings in Azure App Services

- Key-value pairs used to configure app behaviour in the cloud  
- Managed securely via the Azure portal  
- Accessible as environment variables at runtime  
- Allow environment-specific behaviour  
	- Detailed errors in development  
	- Restricted output in production  
- Local `.env` files can simulate cloud settings during development  

### Connection Strings

- Define how applications connect to external services  
	- Databases  
	- Storage accounts  
	- Caching services (e.g., Redis)  
	- Message queues  
- Contain server address, database name, and credentials  
- Stored securely in Azure App Service settings  
	- Prevents embedding credentials in source code  
	- Improves security and manageability

## Containerisation for Consistent Deployment

### Containers

- Lightweight, self-contained units  
	- Include application, dependencies, libraries, and configuration  
- Rely on host system kernel  
	- Efficient and fast execution  
- Ensure consistent runtime behaviour across environments  

### Containerisation with Docker

- Packages application and dependencies into a single image  
- Enables identical behaviour across development, testing, and production  
- Workflow:  
	- Build container locally  
	- Test container  
	- Deploy to cloud  

### Deployment to Azure

- Deploy custom containers to Azure App Services  
- Use container registries such as:  
	- Azure Container Registry  
	- Docker Hub  
- Guarantees consistent execution from local machine to cloud
