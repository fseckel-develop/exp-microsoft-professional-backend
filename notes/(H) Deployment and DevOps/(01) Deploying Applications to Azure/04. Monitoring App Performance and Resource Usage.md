## Azure Monitor

- Central monitoring service in Azure  
- Collects, analyses, and visualises telemetry data from:  
	- Applications  
	- Virtual machines  
	- Infrastructure resources  
- Provides:  
	- Metrics (CPU, memory, request rate)  
	- Logs  
	- Alerts  
	- Dashboards  
- Enables proactive issue detection and resolution  

#### Common Azure CLI Commands

- Enable diagnostics for a web app:  
  `az webapp log config --name <app-name> --resource-group <resource-group> --application-logging true`  

- Stream live logs:  
  `az webapp log tail --name <app-name> --resource-group <resource-group>`  

- List available metrics for a resource:  
  `az monitor metrics list-definitions --resource <resource-id>`  

- Query metrics (e.g., CPU usage):  
  `az monitor metrics list --resource <resource-id> --metric "CpuPercentage"`

## Application Insights

- Feature of Azure Monitor for application-level telemetry  
- Provides:  
	- Real-time performance monitoring  
	- Dependency tracking  
	- Failure diagnostics  
	- Transaction tracing  
	- Anomaly detection  
- Helps identify bottlenecks and optimise application behaviour  
- Improves user experience through rapid diagnostics  

#### Common Azure CLI Commands

- Create an Application Insights resource:  
  `az monitor app-insights component create --app <app-insights-name> --location <location> --resource-group <resource-group>`  

- Connect App Service to Application Insights:  
  `az webapp config appsettings set --name <app-name> --resource-group <resource-group> --settings APPINSIGHTS_INSTRUMENTATIONKEY=<instrumentation-key>`  

- View Application Insights components:  
  `az monitor app-insights component show --app <app-insights-name> --resource-group <resource-group>`

## Autoscaling with Azure Monitor Autoscale

- Automatically adjusts computing resources based on demand  
- Scales out (add instances) during high load  
- Scales in (remove instances) during low usage  
- Optimises both performance and cost efficiency  
- Uses metrics such as:  
	- CPU usage  
	- Memory utilisation  
	- Request count  

#### Common Azure CLI Commands

- Create an autoscale rule (example based on CPU):  
  `az monitor autoscale create --resource-group <resource-group> --resource <resource-id> --min-count 1 --max-count 5 --count 1`  

- Add scale-out rule (CPU > 70%):  
  `az monitor autoscale rule create --resource-group <resource-group> --autoscale-name <autoscale-name> --condition "Percentage CPU > 70 avg 5m" --scale out 1`  

- Add scale-in rule (CPU < 30%):  
  `az monitor autoscale rule create --resource-group <resource-group> --autoscale-name <autoscale-name> --condition "Percentage CPU < 30 avg 5m" --scale in 1`                 
- List autoscale settings:  
  `az monitor autoscale list --resource-group <resource-group>`
