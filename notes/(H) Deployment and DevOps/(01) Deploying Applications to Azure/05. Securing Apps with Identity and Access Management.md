## Azure Active Directory (Microsoft Entra ID)

- Cloud-based identity and access management (IAM) service  
- Centralises user authentication and authorisation  
- Enables Single Sign-On (SSO) across multiple applications  
- Enforces authentication policies and security controls  
- Protects against unauthorised access  

#### Key Capabilities

- User and group management  
- Enterprise application registration  
- Conditional access policies  
- Integration with Azure resources and third-party apps  

#### Common Azure CLI Commands

- Log in to Azure (AAD-backed authentication):  
  `az login`  

- List Azure AD users:  
  `az ad user list --output table`  

- Create a new user:  
  `az ad user create --display-name <name> --user-principal-name <user@domain> --password <password>`  

- Create a group:  
  `az ad group create --display-name <group-name> --mail-nickname <nickname>`  

- Add user to group:  
  `az ad group member add --group <group-name> --member-id <user-object-id>`

## Multi-Factor Authentication (MFA)

- Adds a second verification factor beyond password  
	- SMS code  
	- Authenticator app  
	- Push notification  
- Reduces risk of credential compromise  
- Enforced through Azure Active Directory policies  

#### Security Benefits

- Protects against phishing and password theft  
- Strengthens account security  
- Required for high-privilege or sensitive roles  

#### Common Administrative Actions (Conceptual via CLI & Policy)

- View Conditional Access policies (via Microsoft Graph integration):  
  `az rest --method get --url https://graph.microsoft.com/v1.0/identity/conditionalAccess/policies`  

- Enable security defaults (recommended baseline protection):  
  Configured in Azure AD properties (typically via portal)

## Role-Based Access Control (RBAC)

- Authorisation model controlling what users can do  
- Access granted based on assigned role  
- Built-in roles include:  
	- Owner (full control)  
	- Contributor (manage resources, no role assignment)  
	- Reader (view-only access)  
- Custom roles can define granular permissions  

- RBAC Scope Levels:
	- Management group  
	- Subscription  
	- Resource group  
	- Individual resource  

#### Common Azure CLI Commands

- List available role definitions:  
  `az role definition list --output table`  

- Assign a role to a user:  
  `az role assignment create --assignee <user-object-id> --role <role-name> --scope <resource-id>`  

- List role assignments:  
  `az role assignment list --assignee <user-object-id> --output table`  

- Remove a role assignment:  
  `az role assignment delete --assignee <user-object-id> --role <role-name> --scope <resource-id>`
