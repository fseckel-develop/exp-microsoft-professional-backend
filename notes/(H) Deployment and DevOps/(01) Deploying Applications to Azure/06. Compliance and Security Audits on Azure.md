## Introduction

This job aid presents a practical checklist designed to help maintain the security and compliance of applications hosted on Microsoft Azure. It highlights key practices for safeguarding data, securing network infrastructure, meeting regulatory requirements, 
and using Azure CLI commands to monitor systems, perform security audits, and address potential vulnerabilities.

---
## Data Encryption

### Encrypt Data at Rest

- Use **AES-256 encryption** to prevent unauthorised access to stored data
- Apply encryption to databases, cloud storage, and file systems
- Ensure encryption keys are stored securely
- Rotate encryption keys regularly (e.g., every 6 months) to reduce the risk of compromise

### Encrypt Data in Transit

- Use **TLS 1.3** to secure communication between systems
- Enforce **HTTPS-only connections** to prevent unsecured data transmission
- Protect login credentials, payment details, and sensitive API communications

---
## Network Security

### Implement Network Security Groups (NSGs)

- Use NSGs to control inbound and outbound traffic to Azure resources
- Define explicit rules specifying:
	- Allowed IP addresses
	- Ports
	- Protocols
- Follow the principle of least privilege

### Audit NSG Rules

- Review rules regularly
- Remove outdated or unnecessary permissions
- Ensure no unintended public exposure of critical resources (e.g., databases)

---
## Compliance Monitoring and Auditing

### Use Azure Security Center (Microsoft Defender for Cloud)

- Monitor security posture across cloud and on-premises environments
- Receive real-time threat detection alerts
- Get actionable security recommendations
- Continuously enforce security policies

### Use Compliance Manager

- Track compliance with standards such as:
	- GDPR
	- HIPAA
	- ISO frameworks
- Access prebuilt assessment templates
- Use automated risk tracking
- Generate detailed compliance reports

### Conduct Security Audits

- Use Microsoft Defender for Cloud to:
	- Identify vulnerabilities
	- Detect misconfigurations
	- Review access controls

- Use Compliance Manager to:
	- Validate regulatory alignment
	- Assess compliance posture
	- Track remediation progress

## Azure CLI Commands for Security Audits and Compliance

Once your application is deployed to Azure, you can use the Azure CLI to monitor, audit, 
and remediate security issues directly from the terminal (PowerShell, Bash, or VS Code)

### A. List Security Assessments

- Command:  `az security assessment list`
- Purpose:
	- Retrieves all security assessments for your subscription
	- Outputs results in JSON format
	- Identifies resources marked as _Healthy_ or _Unhealthy_
	- Provides remediation recommendations

- You can pipe the output into tools or scripts for automation:
  `az security assessment list --output json`

### B. List Security Tasks (Remediation Items)

- Command: `az security task list`
- Purpose:
	- Displays actionable remediation tasks
	- Includes severity levels (Low, Medium, High)

- Example issue: Web app not restricted to HTTPS-only connections
- Output is returned in JSON format for automation or dashboard integration

### C. Check Regulatory Compliance Standards

- Command: `az security regulatory-compliance-standards list`
- Purpose:
	- Lists regulatory standards applied to your subscription.
	- Shows compliance status per framework.
	- Highlights resources that are out of compliance.
	- Useful for regulated industries such as healthcare and finance.

### D. Transparent Data Encryption (TDE) on Azure SQL Database

- If an audit shows that Transparent Data Encryption is not enabled:
  `az sql db tde set --database <database-name> --resource-group <resource-group-name> --server <sql-server-name> --status Enabled`
- Purpose:
	- Enables Transparent Data Encryption on a SQL database
	- Protects data at rest
	- Can be automated in security remediation scripts

## Automating Security Audits

- Azure CLI commands can be:
	- Scheduled using automation tools
	- Integrated into CI/CD pipelines
	- Used in scripts to:
	    - Run daily assessments
	    - Identify non-compliant resources
	    - Automatically remediate security gaps

- Regular automated audits reduce risk and ensure continuous compliance.
