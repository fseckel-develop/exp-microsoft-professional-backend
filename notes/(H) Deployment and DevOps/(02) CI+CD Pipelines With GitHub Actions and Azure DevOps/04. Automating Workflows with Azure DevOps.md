## Introduction  

- Automation and streamlined collaboration are essential in modern software development  
- Azure DevOps provides an integrated DevOps environment beyond basic CI/CD  
- Combines automation, version control, artifact management, and project tracking  
- Supports scalable and flexible cross-platform CI/CD workflows  
- Integrates seamlessly with GitHub repositories  

---
## Core Azure DevOps Tools  

### Azure Pipelines  

- Cloud-based service for automating build, test, and deployment processes  
- Supports Continuous Integration and Continuous Delivery (CI/CD)  
- Integrates with multiple version control systems, including GitHub  
- Scales across different project sizes and programming languages  
- Automatically triggers builds and deployments on repository events  
- Supports YAML-based configuration  

### Azure Repos  

- Version control system supporting Git and TFVC  
- Enables collaborative development and code reviews  
- Tracks changes and maintains version history  
- Ensures consistent, version-controlled code across teams  

### Azure Artefacts  

- Centralised package repository service  
- Stores and manages shared dependencies and libraries  
- Integrates with Azure Pipelines for automated builds  
- Ensures consistent package versions across projects  

### Azure Boards  

- Agile project management and tracking tool  
- Provides backlogs, sprint planning, and work item tracking  
- Supports task assignment and progress monitoring  
- Enhances team collaboration and transparency

## Connecting GitHub to Azure DevOps  

### Establish Service Connection  

- Create Azure DevOps organization and project  
- Navigate to Project Settings → Service Connections  
- Connect GitHub repository to Azure DevOps  
- Authorise repository access  
- Enable pipeline triggers based on GitHub activity  

### Configure Azure Pipeline (YAML-Based)  

- Define trigger for main branch updates  
- Select agent pool (Azure-hosted or self-hosted agent)  
- Define build configuration (e.g., Release)  
- Add tasks for:  
	- Restore dependencies (.NET restore)  
	- Build application (.NET build)  
	- Publish artefacts  
	- Deploy to Azure Web App  
- Specify:  
	- Azure subscription  
	- Web app type  
	- Application name  
	- Package path  

---
## Pipeline Execution and Monitoring  

- Automatically triggered by GitHub events (push to main)  
- Can be manually triggered from Azure DevOps interface  
- Agent executes restore, build, package, and deployment tasks  
- Publishes application directly to Azure Web App  
- Provides real-time logs and execution status  
- Green check mark indicates successful deployment  

---
## Best Practices for Cross-Platform CI/CD  

### Setup and Configuration  

- Create and push application to GitHub  
- Ensure Azure Web App is preconfigured  
- Use YAML pipelines for version-controlled automation  
- Keep pipeline configuration modular and readable  

### Monitoring and Maintenance  

- Review logs under Pipelines → Runs  
- Validate triggers in YAML configuration  
- Verify GitHub service connection authorization  
- Ensure correct project paths and dependency setup  
- Confirm deployment credentials and package paths  

---
## Troubleshooting  

### Pipeline Not Triggering  

- Check trigger configuration in YAML  
- Confirm GitHub repository connection  
- Verify branch configuration  

### Build Failures  

- Inspect logs for syntax or dependency errors  
- Validate tool versions and paths  
- Confirm agent pool configuration  

### Deployment Failures  

- Verify Azure Web App name  
- Check Azure subscription and service connection  
- Confirm artifact/package location
