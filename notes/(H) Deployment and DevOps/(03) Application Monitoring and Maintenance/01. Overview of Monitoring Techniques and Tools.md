
- Monitoring ensures apps remain reliable, responsive, and fast in production environments  
- Enables early detection of issues before users are affected  
- Combines real-time tracking, log analysis, and performance insights for full system visibility  

---
## Real-Time Monitoring

- Continuous tracking of live server-side metrics and system health  
- Focuses on immediate issue detection and rapid response  

- Implemented using Azure Monitor
	- Monitors live metrics such as CPU usage, memory consumption, request rate, uptime  
	- Provides dashboards for metric visualization  
	- Supports configurable alerts for threshold breaches (e.g., high CPU or memory usage)  

- Example use case  
	- Sudden CPU spike triggers alert  
	- Immediate action prevents slow response times and user disruption  

- Live Metrics capabilities  
	- Real-time request rate monitoring  
	- Duration tracking (response times)  
	- Failure rate and exception tracking  
	- Resource monitoring (CPU, committed memory)  

- Enables investigation of anomalies  
	- High CPU: traffic surge vs. potential attack  
	- Repeated failures (e.g., 500 errors): application logic or resource exhaustion  
	- Supports scaling decisions (e.g., upgrading service plan)  

---
## Log Monitoring

- Collection and analysis of historical and real-time log data  
- Sources include applications, servers, databases, and network devices  
- Used to identify patterns, root causes, and recurring errors  

- Log Analytics within Azure Monitor 
	- Centralises logs for efficient querying  
	- Supports editing and running log queries  
	- Detects recurring issues (e.g., timeout errors during peak hours)  

- Practical insights  
	- Track API request logs  
	- Identify slow response trends  
	- Trigger automated alerts for repeated failures  
	- Analyze traffic sources, operating systems, and browser usage  
	- Review session data over selected timeframes  

- Helps uncover:  
	- Performance degradation patterns  
	- Traffic spikes  
	- Security-relevant anomalies

## Application Performance Monitoring (APM)

- Focuses on user-experience-oriented performance measurement  
- Monitors both availability and performance  

- Key performance metrics  
	- Response times  
	- Error rates  
	- Transaction traces  
	- Resource usage  

- Implemented using Application Insights within Azure Monitor
	- Real-time data collection and visualisation  
	- Configurable alerts for performance thresholds  
	- Deep insights into endpoints and dependencies  

- Example (ASP.NET API scenario)  
	- Weather forecast endpoint simulates response delays and memory usage  
	- Application Insights tracks:  
		- Request rate  
		- Duration  
		- Failure rate  
		- Exceptions  
		- CPU and memory consumption  
	- Error spikes indicate need for troubleshooting  

- Supports:  
	- Root cause identification  
	- Performance optimization  
	- Capacity planning and scaling decisions  

---
## Integrated Monitoring Strategy

- Real-time monitoring → Detects immediate operational issues  
- Log monitoring → Identifies patterns and root causes over time  
- APM → Ensures optimal user experience and system performance  

- Combined approach provides:  
	- Full visibility into backend health  
	- Faster troubleshooting  
	- Data-driven infrastructure improvements  
	- Reliable, high-performing production systems
