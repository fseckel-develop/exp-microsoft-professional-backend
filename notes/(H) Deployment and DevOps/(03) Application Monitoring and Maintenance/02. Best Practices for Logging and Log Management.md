
- Logging captures key system events and interactions to maintain reliability and performance  
- Raw logs become valuable only when properly structured, stored, and monitored  
- Effective log management transforms scattered data into actionable operational insights  

---
## Centralised Logging

- Collects and aggregates logs from multiple sources into a single platform  
	- Applications  
	- Servers  
	- Cloud services  
	- Infrastructure components  

- Enhances:  
	- Troubleshooting efficiency  
	- Cross-system visibility  
	- Security monitoring  
	- Operational consistency  

- Example implementation  
	- Forward logs to Azure Monitor
	- Search errors across infrastructure from one central interface  
	- Eliminate need to manually inspect individual servers  

- Benefits  
	- Faster root cause analysis  
	- Unified infrastructure insights  
	- Improved incident response  

---
## Structured Logging

- Formats log entries as key-value pairs (commonly JSON format)  
- Makes logs machine-readable, searchable, and queryable  

- Typical structured log fields  
	- Level (info, error, warning)  
	- Message  
	- Timestamp  
	- HTTP method  
	- Status code  
	- Route  
	- User ID (when applicable)  

- Practical example (Node.js application)  
	- Logging framework such as Winston
		- Outputs logs in JSON format  
		- Logs requests, responses, and errors  
		- Records status codes (e.g., 404, 500)  
		- Captures timestamps for compliance  

- Advantages  
	- Easy filtering and querying  
	- Pattern and anomaly detection  
	- Improved debugging  
	- Compatibility with analytics and monitoring tools  

---
## Log Storage and Retention Policies

- Define how long logs remain active before archiving or deletion  
- Balance operational needs with storage costs and compliance requirements  

- Retention policy objectives  
	- Maintain recent logs for active debugging  
	- Archive older logs for regulatory compliance  
	- Prevent unnecessary storage consumption  

- Example policy  
	- Retain error logs for 90 days  
	- Archive after threshold period  
	- Ensure secure long-term storage  

- Ensures  
	- Regulatory compliance  
	- Cost-effective storage management  
	- Access to relevant historical data  

---
## Monitoring and Auditing Logging Systems

- Logging systems must be continuously monitored to ensure effectiveness  
- Prevent silent failures (e.g., server stops sending logs)  

- Monitoring practices  
	- Use alerts to detect logging interruptions  
	- Track unusual error spikes  
	- Verify log completeness  

- Auditing practices  
	- Confirm timestamps are included  
	- Verify severity levels are recorded  
	- Ensure required compliance fields are present  

- Example  
	- Alerts in Azure Monitor notify teams of logging issues  
	- Scheduled audits validate structure and compliance
