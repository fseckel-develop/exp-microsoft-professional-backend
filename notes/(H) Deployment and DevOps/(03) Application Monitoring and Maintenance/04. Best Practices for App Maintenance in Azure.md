## Security Through Updates and Patching

- Regular updates and patching:
	- Protect against vulnerabilities
	- Reduce attack surface
	- Strengthen application resilience

### Automated Patching

- Definition:
	- Automatic application of software updates, security patches, and bug fixes
- Benefits:
	- Eliminates manual intervention
	- Reduces risk of unpatched vulnerabilities
	- Minimizes operational disruption
- Azure capabilities:
	- Schedule recurring updates
	- Apply patches during off-peak hours
	- Rapidly remediate emerging threats
- Practical example:
	- Weekly automated patching for an e-commerce platform
	- Updates applied during low-traffic windows to avoid customer impact

### Service Updates

- Definition:
	- Enhancements, bug fixes, security patches, and new features released by Azure
- Characteristics:
	- Automatically applied to Azure-managed resources
	- Improve functionality, performance, and security
- Examples:
	- Database engine improvements
	- App Service feature enhancements
	- Advanced analytics capabilities
- Impact:
	- Systems remain modern and optimized
	- Improved stability and smoother user experience

### Patch Compliance

- Definition:
	- Ensuring all systems and applications meet required patch levels
- Purpose:
	- Maintain security and regulatory compliance
	- Reduce exploitable vulnerabilities
- Azure support:
	- Compliance dashboards
	- Identification of unpatched resources
	- Security recommendations
- Outcome:
	- Clear visibility into infrastructure health
	- Faster remediation of non-compliant systems

## Performance Monitoring and Optimisation

- Regular performance reviews ensure:
	- Efficient resource utilisation
	- Stable response times
	- Scalable infrastructure

### Monitoring with Azure Tools

- Application insights include:
	- Data in / data out
	- HTTP 5XX errors
	- Response time metrics
	- Request counts
- Metrics provide:
	- CPU usage
	- Memory consumption
	- Thread count
	- Health check status
- Benefit:
	- Increased visibility into application behaviour
	- Data-driven optimisation decisions

### Scaling Strategies

- Scaling Up:
	- Increase CPU, memory, and compute power
	- Suitable for resource saturation scenarios
- Scaling Out:
	- Increase number of instances
	- Suitable for handling higher traffic loads
- Scaling options:
	- Manual scaling
	- Rule-based autoscaling (e.g., CPU > 75%)
	- Elastic scaling for dynamic workloads
- Considerations:
	- Cost per instance
	- Traffic patterns
	- Performance thresholds

### Database Query Optimisation

- Continuous query evaluation improves performance
- Techniques:
	- Use explicit JOINs
	- Avoid correlated subqueries
	- Implement indexing strategies
	- Use SQL EXPLAIN statements to analyse execution plans
- Performance indicators:
	- Table scans vs indexed access
	- Number of rows processed
	- Execution frequency
- Optimisation outcome:
	- Reduced query cost
	- Faster response times
	- Improved scalability for larger datasets
