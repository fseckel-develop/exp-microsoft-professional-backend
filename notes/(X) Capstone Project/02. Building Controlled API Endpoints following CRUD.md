---

## Core API Concepts

- API Controllers:
	- Classes that handle HTTP requests  
	- Return structured HTTP responses  
	- Keep backend logic organised and modular  
- Routing:
	- Maps URL paths to controller methods  
	- Example:
		- `/api/inventory` → `InventoryController`
		- `/api/orders` → `OrderController`
- HTTP Methods:
	- GET → Retrieve data  
	- POST → Create new records  
	- DELETE → Remove records  
- Dependency Injection (DI):
	- Provides services (e.g., `DbContext`) to controllers  
	- Improves modularity and testability  
- Async Error Handling:
	- Return clear responses if an ID is not found  
	- Use asynchronous methods for better performance  
- Microsoft Copilot:
	- Suggests improvements  
	- Generates missing endpoints  
	- Optimises async and error-handling logic

	- Uses dependency injection for `LogiTrackContext`

---
## InventoryController Endpoints

- Route Base: `/api/inventory`  
- Required Endpoints:
	- `[HttpGet]`
		- Return all inventory items  
	- `[HttpPost]`
		- Add new inventory item  
	- `[HttpDelete]` `/api/inventory/{id}`
		- Delete item by ID  
- Requirements:
	- Inject `LogiTrackContext` via constructor  
	- Ensure data is persisted in database  

---
## OrderController Endpoints

- Route Base: `/api/orders`
- Required Endpoints:
	- `[HttpGet]`
		- Return all orders  
	- `[HttpGet]` `/api/orders/{id}`
		- Return single order with items  
	- `[HttpPost]`
		- Create new order  
		- Accept multiple inventory items  
	- `[HttpDelete]` `/api/orders/{id}`
		- Delete order by ID  
-	Important:
	-	Include related items when returning order  
	-	Ensure correct model binding in POST
