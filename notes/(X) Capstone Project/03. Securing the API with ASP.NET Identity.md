---
## Core Security Concepts

- Authentication:
	- Verifies who the user is  
	- Uses login credentials  
	- Returns JWT token for future requests  
- Authorisation:
	- Determines what the user is allowed to access  
	- Based on identity and assigned roles  
	- Implemented with `[Authorize]` attribute  
- ASP.NET Identity:
	- Handles:
		- User registration  
		- Password hashing  
		- Login validation  
		- Role management  
- JWT (JSON Web Token):
	- Issued after successful login  
	- Sent with requests in Authorisation header  
	- Used to validate user identity  
- Microsoft Copilot:
	- Generates authentication setup  
	- Suggests role-based restrictions  
	- Identifies security improvements

## Install and Configure Identity

- Install packages:
	  `dotnet add package Microsoft.AspNetCore.Identity.EntityFrameworkCore`
	  `dotnet add package Microsoft.EntityFrameworkCore.Sqlite`
- Create `ApplicationUser` class:
	- Inherits from `IdentityUser`
- Update DbContext:
	- Inherit from `IdentityDbContext<ApplicationUser>`
- Register Identity in Program.cs:
	- `AddIdentity<ApplicationUser, IdentityRole>()` 
	- `AddEntityFrameworkStores<LogiTrackContext>()`  
- Run migrations:
	  `dotnet ef migrations add AddIdentity`  
	  `dotnet ef database update`  
- Result:
	- Database updated with Identity schema

## Authentication Endpoints

- Create `AuthController`
- Required Endpoints:
	- `[HttpPost] /api/auth/register`
		- Register new user  
		- Store hashed password  
	- `[HttpPost] /api/auth/login`
		- Validate credentials  
		- Return JWT token  
- Use:
	- `UserManager` for user creation  
	- `SignInManager` for authentication  
- Token usage:
	- Included in Authorisation header  
	- Format: Bearer {token}  

---
## Protect API Routes

- Apply `[Authorize]` attribute:
	- Protect `OrderController`
	- Protect `InventoryController`
- Role-based restriction:
	- Assign roles:
		- Manager  
	- Restrict routes:
		- `[Authorize(Roles = "Manager")]`
- Example use cases:
	- Only managers can delete inventory  
	- Only authorised users can access orders

## Testing Security

- Register new user  
- Log in to retrieve JWT token  
- Test:
	- Access protected routes without token → Should fail  
	- Access with valid token → Should succeed  
	- Access role-restricted route without correct role → Should fail  
- Confirm:
	- Token validation works  
	- Roles enforced correctly  
	- Data remains protected
