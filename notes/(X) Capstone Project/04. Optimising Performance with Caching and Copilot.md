## Core Performance Concepts

- In-Memory Caching:
	- Stores frequently accessed data in memory  
	- Reduces repeated database queries  
	- Improves response time for repeated requests  
- Query Optimization:
	- Use `.Include()` to eager-load related data  
	- Use `.AsNoTracking()` for read-only queries  
	- Minimizes N+1 query problems  
- Performance Measurement:
	- Use Stopwatch or profiling tools  
	- Measure before and after caching/query improvements  
- Microsoft Copilot:
	- Suggests performance enhancements  
	- Identifies slow or repetitive queries  
	- Helps refine controller logic  

---
## Enable In-Memory Caching

- Add service in Program.cs:
	- `builder.Services.AddMemoryCache();`  
- Inject `IMemoryCache` in controller:
	- Example in InventoryController:

```C#
private readonly IMemoryCache _cache;  

public InventoryController(LogiTrackContext context, IMemoryCache cache) 
{ 
	_context = context; 
	_cache = cache; 
}  
```
